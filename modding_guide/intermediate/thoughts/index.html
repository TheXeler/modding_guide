Thoughts

Hearthlings 从 base_human.json mixin 获得 happiness 组件和 thoughts 组件。这些组件使他们能够根据当前的 thoughts 变得高兴或悲伤，我们可以在角色表的 Morale 选项卡中看到这些 thoughts。

Thoughts 将从不同的 Lua 脚本中添加，如 AI 行为、组件或 trait 控制器，并且如果它们有有限的持续时间，将自动移除。

添加 Thoughts

要向可用 thoughts 列表中添加新 thought，我们需要在清单中 mixinto stonehearth/data/happiness/thoughts.json：

  "mixintos" : {
     "stonehearth:data:thoughts" : "file(data/happiness/thoughts.json)"
  }

我们的 mixinto 看起来像这样：

  {
     "thoughts": {
        "traits": {
           "mutually_exclusive": true,
           "values": {
              "impatient": {
                 "display_name": "i18n(my_mod:data.happiness.thoughts.traits.values.impatient.display_name)",
                 "tooltip": "i18n(my_mod:data.happiness.thoughts.traits.values.impatient.tooltip)",
                 "happiness_delta": -5,
                 "duration": "12h",
                 "stackable": false
              }
           }
        }
     }
  }

首先有一个 "thoughts" 键。在其中有用于分组 thoughts 的标识符。在这些标识符内部可以有一些属性，然后有一个 "values" 字段，其中包含更多对应于 thoughts 或其他组的自定义标识符。

我们可以在组标识符内找到的一个常见属性是 "mutually_exclusive" : true。这意味着此组中一次只能激活一个 thought。例如，检查 sleepiness thoughts："fatigued" thought 将在添加时替换 "tired" thought，因为它们是互斥的。如果此属性为 false，则此组中的多个 thoughts 可以同时激活。

一些 thoughts 基于"阈值"而不是"值"。但这些仍然从某些 Lua 脚本中查询。例如，关于空间（拥挤度）的 thoughts 除了 "mutually_exclusive" 属性外，还有："time_average": true、"averaging_period": "1m"，然后它们不是 "values" 而是有一个 thoughts 的 "thresholds" 映射。然后 open_space_observer.lua 查询这些属性并根据 thresholds 映射内的字段自动调整 thoughts。

对于 traits，有一个包含一些非互斥 thoughts 的 "values" 条目，但也有与 "values" 同级别的其他标识符，定义互斥 thoughts 的组。这样，我们可以嵌套组并为它们定义不同的排除。

我们可以在 thought 中包含不同的属性：

  • "display_name" -- 将在角色表的 Morale 选项卡中显示的本地化文本。
  • "tooltip" -- 当我们将鼠标悬停在 thought 上时显示的工具提示的本地化文本。
  • "happiness_delta" -- 这是随时间添加到当前幸福感值的值（如果你希望 thought 为负面/降低幸福感，请使用负值）。不要为此字段使用小数值。幸福感范围为 0 到 100（从 50 开始）。
  • "duration" -- 一个时间表达式。如果我们包含此字段，thought 将在持续时间结束后自动移除。持续时间也将在 UI 中显示，随时间减少。其他 thoughts 可能会覆盖此 thought，具体取决于 "mutually_exclusive" 属性，因此它可能持续的时间更短。
  • "stackable" -- 此 thought 是否可以堆叠。
  • "max_stacks" -- 如果 "stackable" 为 true，此字段定义我们可以堆叠 thought 的最大次数（UI 中会出现堆叠指示器）。这意味着如果 thought 在已经激活时再次添加，它将堆叠此 thought 的幸福感增量，而不是覆盖它。
  • "max_threshold" -- 由阈值管理的 thoughts 的值。
  • "hide" -- 一个布尔值。我们对此字段用于一些我们不希望在 UI 中显示的中性 thoughts（happiness_delta 为 0），因为 thoughts 在角色表中分为正面或负面。

要从 trait 控制器（或任何其他 Lua 脚本）添加 thought，我们将调用 radiant/modules/entities.lua 中的 add_thought 函数，并传递我们要添加 thought 的实体、thought 的 URI，以及如果需要的话，thought 工具提示的一些参数：

  radiant.entities.add_thought(self._sv._entity, 'stonehearth:thoughts:traits:impatient', {
              tooltip_args = {
                 target_name = radiant.entities.get_custom_name(target)
              }
           })

要引用 thoughts，我们必须像这样构建 URI：'mod_namespace:thoughts:thought_category:thought'。例如：

  "stonehearth:thoughts:traits:charismatic:inspired",
  "stonehearth:thoughts:traits:plant_lover",
  "stonehearth:thoughts:job:gained_a_level"

注意我们用冒号分隔键，就像 URIs 一样。

Happiness 组件从 stonehearth/data/happiness/thoughts.json 读取 thoughts，因此我们必须对其进行 mixinto 以便将我们的自定义 thoughts 添加到列表中（我们可以重用现有的类别/组或添加新的）。但我们也可以使用我们模组的命名空间来引用我们的自定义 thoughts。例如，上面的示例如果我们像这样引用 thought 也会起作用：'my_mod:thoughts:traits:impatient'。

要在游戏中测试，我们可以使用默认控制台手动添加和移除 thoughts（如果命令未出现，请启用 debugtools 模组）：

  add_thought stonehearth:thoughts:traits:charismatic:inspired
  remove_thought stonehearth:thoughts:traits:charismatic:inspired

选择 Hearthling，打开默认控制台并输入 add_thought 命令。注意我们将 thought URI 作为参数传递时不带引号。
