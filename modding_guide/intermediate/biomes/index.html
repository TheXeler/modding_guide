生物群落

与往常一样，我们可以复制现有的生物群系来更快地创建自定义生物群系。生物群系文件位于 stonehearth/data/biome。

  • 将生物群系添加到列表
  • 生物群系信息
  • 生物群系生成数据

将生物群系添加到列表

首先，让我们在清单中为我们的生物群系创建一个别名：

  "aliases" : {
     "biome:silent_woods" : "file(data/biome/silent_woods.json)"
  }

然后，让我们添加 mixinto，以便我们的生物群系出现在可用生物群系列表中：

  "mixintos" : {
     "stonehearth:biome:index" : "file(data/biome/index.json)"
  }

我们的 index.json mixinto 文件如下所示：

  {
     "biomes": {
        "silent_woods": "silent_woods:biome:silent_woods"
     }
  }

键是我们生物群系的自定义标识符，别名指向我们的生物群系别名，前面加上 mod 的命名空间和冒号。

生物群系信息

生物群系的定义分为两个文件。第一个文件包含基本信息，它是我们在别名中引用的文件（在我们的示例中为 silent_woods.json）。第二个文件仅包含与实际地形生成相关的数据（在我们的示例中，它将是 silent_woods_generation_data.json）。

信息文件包含以下字段：

  • "type": "biome" -- 此文件代表一个生物群系。
  • "alias" -- 我们生物群系的别名，前面加上 mod 的命名空间和冒号。
  • "display_name" -- 我们生物群系的本地化名称，这是将在 UI 的"选择生物群系"列表中显示的标题。
  • "description" -- 将在生物群系列表中显示为我们生物群系的本地化描述。
  • "random_location_names" -- 包含我们生物群系的本地化位置名称的数组。当玩家选择在我们的生物群系中游玩时，将选择一个名称，并出现在登船屏幕的故事描述中（我们可以点击那里更改生物群系）。
  • "image" -- 将在 UI 的生物群系列表中使用的图像路径。通常为 400x300 像素。
  • "story_board_image" -- 当玩家选择我们的生物群系时，将出现在第一个登船屏幕右侧的图像。
  • "game_mode_images" -- 包含游戏模式别名（通常为 "stonehearth:game_mode:normal" 和 "stonehearth:game_mode:hard"）的映射，指向将分层在故事板图像顶部的图像。这允许我们使用半透明图像在玩家选择非和平游戏模式时在图像顶部添加敌人。
  • "generation_file" -- 指向我们生物群系的生成数据 JSON 文件的路径。
  • "mining_loot_table" -- 可选字段。我们可以链接到包含我们生物群系自定义采矿战利品表的文件（如果我们不添加此字段，采矿服务将默认加载 "stonehearth:mining:base_loot_table"）。可以在沙漠生物群系（stonehearth/data/biome/desert.json）中看到一个示例。它 mixin 基础战利品表，以重用不同地形类型的默认战利品，然后调整粘土的权重，以便在沙漠中通过采矿获得更多粘土。
  • "applied_manifests" -- 可选字段。我们可以使用它仅在玩家选择在我们的生物群系中游玩时应用自定义清单。也可以在沙漠生物群系中看到一个示例。（热加载清单并不适用于所有情况。将在指南的另一部分讨论）。
  • "default_starting_season" -- 此字段将指向下面定义的季节的键。它用于在 UI 中自动选择默认季节（在登船期间的地图屏幕中）。
  • "seasons" -- 在这里我们将定义生物群系的季节和天气。有关如何设置它们的信息，请参阅此页面。

关于生物群系不是所选王国的首选生物群系的警告不会出现在自定义 mod 中，仅出现在 stonehearth 中。

生物群系生成数据

地形生成由 stonehearth/services/server/world_generation/world_generation_service.lua 管理，并借助 stonehearth/services/server/world_generation 目录中的所有其他文件，尽管实际工作是在 C++ 端完成的。地形本身由 stonehearth/data/service/game_creation_service.lua 生成，一旦游戏知道它必须使用哪个生物群系/种子等。

地形服务处理领土和可见性（战争迷雾）等事物。水文学服务管理水的行为（何时应该创建瀑布、枯竭速率等）。还有一些相关的组件。

生成数据文件中使用的数据是从主生物群系文件中链接的 "generation_file" 中提取的。在其中，我们可以确定诸如地形高度、水深、哪些植物和树木将出现在我们的自定义生物群系中等事物。一些 mod 覆盖生成地形的 Lua 文件，以允许不同的地形生成（默认基于噪声函数）。例如，Canyon Biome mod 有高大的垂直悬崖和河流。

temperate_generation_data.json 文件比沙漠和北极生物群系的相应文件有更多的注释，因此是一个很好的起点。许多注释有"Def N"，这是该参数的默认值，如果我们犯了错误并需要恢复到合理的值，或者最小/最大值，以便我们不尝试添加无效值。如果我们对 stonehearth/data/terrain/terrain_blocks.json 添加 mixinto 以添加更多地地形类型，我们也能够在我们的生物群系中使用它们。

理想情况下，我们将首先定义地形的形状，然后添加我们想要的树木/植物，最后仔细检查我们希望/不希望我们的生物群系具有的任何静态场景（如兔子雕像、小动物巢穴、地标等）。

如果我们在生成文件中使用了错误的值，在生成地图时游戏中会弹出错误。有时它不会在那里失败，但只在某些种子下失败，因此我们可以尝试重新滚动地图几次，看看是否有任何问题。一些值也可能受其他值影响，因此虽然它们没有错，但可能会根据其他值导致问题。

简而言之，建议逐个更改值并进行大量测试，以便我们确切知道哪个值导致问题，以及我们最近的更改对地形生成了多大影响。

让我们解释一下我们可以在生成文件中找到的不同字段：

  • "type": "biome_generation_data" -- 这是一个生物群系生成数据文件。

  • "ring_tesselation" : {} -- 这些参数用于生成我们可以在草地边界上看到的环，它们具有不同的颜色。此字段内的键是 terrain_blocks.json 内 block_types 部分的键。我们只能包含厚度为 1 个方块的地形类型的键（如草地和泥土）。我们可以更改环的宽度和地形类型、添加更多环，或者如果我们不想要它们则移除它们。它们链接到的地形类型（"name"）是 terrain_blocks.json 内 block_types 部分的任何键。

  • "season" -- 这指向我们在主生物群系文件中定义的季节的键。它将用于选择生成我们生物群系的默认调色板（以便 UI 可以在玩家选择任何季节之前在登船的地图屏幕中加载起始颜色）。

  • "palettes" -- 内部的键对应于我们为生物群系定义的季节。每个键然后将 terrain_blocks.json 中的地形类型映射为十六进制格式的颜色（以 # 开头，按惯例大写）。这样，地形可以根据季节看起来不同。我们还可以为植物定义季节性变体（参见此页面）。

    • 登船地图中的地形颜色将从对应于默认季节的调色板中选取，如上所述。但是，如果我们想更改它们，例如，编辑水或树木的颜色（这些不从调色板中获取），我们可以在 "palettes" 部分内的 "minimap" 部分中进行，在定义任何调色板之前。例如：

      "minimap":{
   "water":"#FF0000",
   "trees":"#AA3800",
   "plains_2":"#00BB00",
   "mountains_1":"#334433"
}

      这将更改登船地图中水、树木和两个地形层的颜色。我们可以通过使用地形类型（平原、山麓或山脉）加上下划线和表示步骤/层的数字来编辑其中显示的任何颜色。

  • "terrain": {} -- 此部分中的参数控制地形的形状。

    • "minimap" : {} -- 可选字段。我们可以将地形类型（水、平原/山麓/山脉加上下划线和步骤编号）映射到自定义名称。当我们将鼠标悬停在地形上时，这些名称会出现在地图屏幕的右侧。例如，当将鼠标悬停在水上时，我们可以使其显示"海洋"或"沼泽"，当将鼠标悬停在平原上时，我们可以使其显示"海滩"等。

      "minimap":{
   "water" : "i18n(my_mod:ui.shell.select_settlement.terrain_codes.water)",
   "plains_2" : "i18n(my_mod:ui.shell.select_settlement.terrain_codes.plains_2)"
}

    • "noise_map_settings" : {} -- 地形使用 simplex 噪声生成器生成。这些是该生成器的参数。查看 temperate_generation_data.json 中的注释以了解它们可以做什么。

    • "plains_percentage" -- 平原的百分比。根据下面的其他值，它可能感觉不正确（例如，如果我们为山麓添加太多步骤，它们将覆盖平原）。

    • "height_base": 36 -- 这是关于图层对齐位置的偏移量。默认为 36，必须是 1（对于基岩，这是我们无法挖掘的底层）加上五的倍数（这与采矿有关：1 个方块用于地板以便炉民可以站在上面，4 个方块以便他们有足够的头部空间）。

    • "plains": {}、"foothills" : {} 和 "mountains" : {} -- 定义地形层数量及其高度（以方块/世界单位为单位）的参数。通常我们不编辑平原的值，它们很容易导致错误。

  • "terrain_detailer": {} -- 这些是定义露头（粘在地形层/步骤侧面的地形块）的参数。

    • "mountains" : {}、"foothills" : {} 和 "plains" : {} -- 再次，查看 temperate_generation_data.json 文件中的注释以了解每个参数的作用。例如，如果我们将"layer_count"设置为 0，那些地形步骤的侧面将完全平坦，因此地形看起来像巨大的矩形形状。

  • "landscape": {} -- 这些是定义其余景观的参数：

    • "placement_table": {} -- 在这里我们定义所有想要在地形中生成的实体（树木、植物、灌木和巨石）。键是我们实体的 URI（我们可以重用 stonehearth mod 中的树木等，或使用我们的自定义实体，或两者兼用）。对于每个键，我们定义以下属性：

      • "placement_type" -- 可以是 "single"、"dense" 或 "pattern"。

      • "parameters" : {} -- 根据 placement_type，我们将有不同的参数。它们指的是"tiles"，即 16x16x16 的地形块（您可以通过启用调试键/debugtools 并按 F1 来可视化它们）。使用这些字段，我们可以指定一个 tile 中可以生成多少实体，我们应该在实体周围留出多少空间以便没有其他东西在那里生成（大树需要很大的空间），一个半径以防止树木半漂浮在悬崖上生成，以及基于随机几率的网格状图案的值，以便我们可以生成实体组。

        考虑到这些值会影响实体的放置，因为如果它们太聚集，可能没有任何实体会生成，或者实体会在彼此之上生成（由于它们的碰撞盒）。务必确保留出一些空间，否则植物将无法在 tiles 中找到生成的地方，因为树木占据了相当大的空间。

    • "trees": {} -- 树木放置的参数：

      • "sizes" : {} -- 如果我们使用大小作为后缀设置树木的别名，我们在这里定义树木的大小（查看此页面）。如果我们的树木没有不同的大小，我们可以（并且应该，因为"trees"部分是必需的）仅使用它们的别名定义它们，以及一对假大小，如"small": 0, "large": 10。如果我们对别名使用了后缀方法（如果您的树木有不同的大小，推荐使用），古老百分比将用于大树（因此一定比例的大树将作为古老树木生成）。
      • "weights" : {} -- 每个地形海拔的每棵树的权重。这里的条目数量可能会根据我们之前定义了多少地形步骤而变化。
      • "noise_map_parameters": {} -- 这些是生成森林的噪声函数的参数。

    • "berries" : {} -- 放置灌木的参数。此字段是必需的（否则我们会得到错误），但实际上对游戏没有影响，因此我们可以在其中使用我们为生物群系定义的任何树木/植物。

    • "scattered": {} -- 其余实体（如植物）的参数。如果无论上述参数如何都看不到任何树木被放置，请尝试也将它们添加到此部分，就像沙漠生物群系所做的那样）。

      • "plants": {} -- 不同地形海拔的植物的权重，以及它们的噪声图参数，以便它们可以在不同的地形海拔中更分散或更集中地生成（注意树木放置，因为如果树木太多，可能没有足够的空间让植物出现）。
      • "boulders": {} -- 巨石放置的参数。此字段也是必需的。它基于与树木相同的设置，带有后缀，因此对于"namespace"字段，我们应该指定不带后缀的别名。不幸的是，对于巨石，我们必须使用这种大小后缀，否则它们不会出现。但是，我们可以在那里指定任何实体的完整别名，并仅将我们的巨石添加到植物的"scattered"权重集中。

    • "water" : {} -- 水的参数。错误的值将在生成地形时的加载屏幕上抛出错误，不一定在选择地图屏幕上。"deep" 值是从水面到水深部分的方块数（您可以在大湖中看到它们）。"shallow" 值是从水面到湖泊浅水部分的方块数。

      在世界生成时，水将始终比其生成的表面水平低 1.5 个方块，除非我们在 "water" 部分内指定 "water_height_delta" 属性。它不能为负数，并且必须始终低于"shallow"值的水平。
