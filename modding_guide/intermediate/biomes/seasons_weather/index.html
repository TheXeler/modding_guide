季节

季节在生物群系的主 JSON 文件中定义。以下是如何设置季节的小示例：

  "seasons": {
     "spring": {
        "display_name": "i18n(stonehearth:data.biome.temperate.seasons.spring.display_name)",
        "description": "i18n(stonehearth:data.biome.temperate.seasons.spring.description)",
        "start_day": 0,
        "weather": [
           {
              "uri": "stonehearth:weather:sunny",
              "weight": 10
           }
        ]
     },
     "summer": {
        "display_name": "i18n(stonehearth:data.biome.temperate.seasons.summer.display_name)",
        "description": "i18n(stonehearth:data.biome.temperate.seasons.summer.description)",
        "start_day": 21,
        "weather": [
           {
              "uri": "stonehearth:weather:thunderstorm",
              "weight": 4
           }
        ]
     },
     "autumn": {
        "display_name": "i18n(stonehearth:data.biome.temperate.seasons.autumn.display_name)",
        "description": "i18n(stonehearth:data.biome.temperate.seasons.autumn.description)",
        "start_day": 42,
        "weather": [
           {
              "uri": "stonehearth:weather:windy",
              "weight": 6
           }
        ]
     },
     "winter": {
        "display_name": "i18n(stonehearth:data.biome.temperate.seasons.winter.display_name)",
        "description": "i18n(stonehearth:data.biome.temperate.seasons.winter.description)",
        "start_day": 63,
        "weather": [
           {
              "uri": "stonehearth:weather:snow",
              "weight": 6
           }
        ]
     }
  }

我们将使用自定义键作为季节的标识符（例如，"spring" / "summer" / "autumn" / "winter"）。我们可以有任意数量的季节，但需要确保它们适应一年内。

您可以在 stonehearth/data/calendar/calendar_constants.json 中检查不同时间间隔的长度。默认情况下，它们的设置与现实世界的时间相同，除了天数（我们每月只有 7 天）。不建议更改日历常量，因为它们可能会影响其他 mod，并且可能有代码依赖于它们。如果您想更改月份的名称，也可以在日历常量中找到它们。

在每个季节标识符内，我们有一些属性：

  • "display_name" -- 将在 UI 中显示的季节的本地化名称（在地图登船屏幕、作物列表中——因为作物有首选季节——以及当我们将鼠标悬停在太阳小部件下方的月份名称上时）。

  • "description" -- 季节的本地化描述。它将出现在我们将鼠标悬停在地图登船屏幕中的季节名称上以及太阳小部件下方的月份名称上时。

  • "start_day" -- 这决定了季节的持续时间。在内部，我们从第 0 天开始，所以对于上面的示例，春季的开始日是 0，夏季的开始日是 21，因此春季的持续时间是 3 个游戏月（记住，每月 7 天）。

    我们在季节的描述中手动表达此持续时间，但太阳小部件下方月份名称的工具提示也有一个计数器，告诉季节何时结束，因为我们不是从第一个月的第 1 天确切开始游戏，而是从所选季节的开始开始，这发生在相应月份的第一周中间（对于春季将是第一个月，对于夏季是第四个月，等等）。

    我们可以使季节持续不同数量的月份。例如，在北极生物群系中，冬季持续 7 个月，而春季和秋季持续 1 个月，夏季持续 3 个月。

    务必确保您的第一个季节从第 0 天开始。

  • "weather" : [] -- 包含天气 URI 及其权重的对象数组。这样我们可以为每个季节定义不同的天气，并使某些天气类型比其他类型更频繁。有关如何创建自己的天气，请参见下文。

天气

要创建自定义天气类型，我们需要创建几个文件。现有天气可以在 stonehearth/data/weather 中找到，因此我们可以从那里复制一个，重命名并编辑文件。

首先，让我们将别名添加到我们的清单中，以便我们不会忘记它，并且可以在我们的自定义生物群系中使用它或将其添加到 stonehearth mod 的现有生物群系中：

  "aliases" : {
     "weather:sprinkles_rain" : "file(data/weather/sprinkles_rain/sprinkles_rain.json)"
  }

sprinkles_rain.json 文件将具有以下属性：

  • "display_name" -- 天气的本地化名称，当我们将鼠标悬停在太阳小部件下方的天气预报上时，将显示在工具提示中。

  • "description" -- 将在天气工具提示中显示的本地化描述。在这里，我们可以包括天气具有的增益或劣势。我们可以在工具提示文本中内联 HTML 标签以获得更好的格式。

  • "icon" -- 将在预报中显示的我们天气的图标路径。

  • "sky_settings" -- 天空设置文件的 URI。见下文。

  • "thoughts": [] -- 此天气的想法数组。每当天气活跃时，此列表中的随机想法将添加到每个炉民。如果我们在数组中多次包含一个想法，我们将改变其概率（将有更多机会被选中）。

    这些 URI 不是别名，而是由想法索引中的键组成的字符串。有关更多信息，请参阅这里。

  • "ambient_sound_key" -- 这指向一个标识符（例如，"rain"），当此天气活跃时，声音服务将使用该标识符播放环境声音。您可以重用 stonehearth/data/sounds/sound_constants.json 中的现有标识符，或通过 mixinto 在那里注入自定义标识符。

  • "camera_attached_effect" -- 附加到相机的效果的 URI。这是一个可选字段。我们通常将它们用于雨、雪等。这些效果使用发射器具有 CUBOID 表面的 cubemitters，以便它可以覆盖所有相机视图。

  • "subject_matter" -- 此天气的主题事项的 URI（例如，"candyland:subjects:weather:sprinkles_rain"）。对于天气主题，我们通常使用类似于预报的 2D 图标，并将它们保存在与其他天气文件相同的文件夹中。主题事项文件（sprinkles_rain_subject.json）将包含如下内容：

    {
   "type": "entity",
   "components": {
   },
   "entity_data": {
      "stonehearth:thought_bubble_icon":  {
         "image":  "file(sprinkles_rain_subject.png)"
      }
   }
}

    当天气活跃时，主题将添加到炉民的对话主题列表中。

  • "plant_growth_time_multiplier" -- 可选字段。它是植物生长速率的乘数。如果我们包含它，当此天气活跃时，植物生长将受到影响，因此我们可以使植物和作物生长得更快或更慢。

  • "vision_multiplier" -- 可选字段。视野半径的乘数，通常是小数（从 0 到 1）。根据您的图形设置，这可能会有所不同或没有效果，但此字段用于在您缩小定居点时使其不太可见。一个很好的例子是禁用回退渲染器时的雾天。当您缩小时，您会看到它在雾下变得有点模糊，当您放大时，您将能够清晰地看到它。

  • "hide_cloud_shadows" -- 可选布尔值，用于隐藏出现在地面上的云阴影，适用于阴天、晴天等。

  • "snow_accumulation_per_minute" -- 可选字段。0 到 1 之间的非常小的值（它是每游戏分钟）。它将使水平表面的顶部被一层平坦的雪覆盖，随着时间的推移变得更加不透明。其他表面下方的表面不会有雪。如果此值为负，如果有前一天的雪，它将缓慢移除雪层。雪的颜色在着色器中控制，检查 stonehearth/data/horde/materials/fullscreen_quad_material.json 中的 SNOW_COVER 条目。还要检查 stonehearth/services/client/weather_render/weather_render_service.lua 以了解我们如何渲染天气效果。

  • "is_dark_during_daytime" -- 可选字段，默认为 false。当为 true 时，炉民将在白天保持火坑点燃（我们通常将其用于恶劣天气，如雨、雪等）。

  • "unsheltered_debuff"、"unsheltered_animal_debuff" 和 "unsheltered_resistance_buff" -- 指向增益效果 URI 的可选字段。当我们定义它们时，当此天气活跃时，我们的炉民将获得"unsheltered_debuff"，我们的牧场动物将获得"unsheltered_animal_debuff"。如果我们的炉民或动物具有为"unsheltered_resistance_buff"定义的增益效果（例如，通过装备一些冬季服装），那么他们将不会获得无遮蔽减益效果。

  • "controller" -- 可选字段。它是 Lua 脚本的 URI（请记住将其添加到清单的"controllers"部分！），当此天气活跃时将运行该脚本。一些示例包括 "stonehearth:weather_scripts:foggy"，当雾天活跃时生成不死生物，以及 "stonehearth:weather_scripts:thunderstorm"，使雷暴随机出现并伤害炉民和树木。

  • "renderer" -- 可选字段。它是 Lua 脚本的 URI（必须在清单的"client_controllers"部分中声明！）。示例包括 "stonehearth:weather_renderers:titanstorm" 和 "stonehearth:weather_renderers:windy"。我们可以从它们访问客户端服务并在客户端渲染事物（如阵风效果，它们不附加到相机）。

天空设置

天空通过 stonehearth/services/client/sky_renderer/sky_renderer_service.lua 渲染。天气的 JSON 文件中引用的 sky_settings.json 文件包含有关天气天空的数据。

它具有以下字段：

  • "sky_gradient" -- 图像文件的路径（通常是 256x256 像素的 PNG）。从左到右，它代表游戏内时间（从午夜开始），从上到下它代表天空的顶部到底部。

    您可以重用现有的渐变（它们在 stonehearth/data/texture_gradient/skybox 的子文件夹内）或创建自己的渐变。每个渐变的长度取决于您希望天空在一天中的每个时间看起来如何，也许您希望更慢的过渡或更短的过渡。

    以下是这些时间事件发生的天空渐变百分比：

    midnight           0
sunrise_start      10
sunrise_peak_start 20
sunrise_peak_mid   25
sunrise_peak_end   30
sunrise_end        40
midday             50
sunset_start       60
sunset_peak_start  70
sunset_peak_mid    75
sunset_peak_end    80
sunset_end         90
day_length        100

  • "celestials": [] -- 包含天体参数的数组。通常它包含 2 个条目，一个用于太阳，另一个用于月亮，具有以下属性：

    • "name" -- 天体的标识符（例如，"sun"）。

    • "shadows_enabled" -- 启用/禁用此天体对象阴影的布尔值（通常设置为 true）。

    • "light_colors" : [] -- 包含 "time"（指向一天中的时间的标识符）和 "value"（包含十进制格式的 RGB 颜色的数组）映射的数组。

      如果光色为 [0,0,0]，光将被禁用（例如，我们在夜间禁用太阳，在白天禁用月亮）。这里使用的一天中的时间在 sky_renderer 服务中定义，但您可以只复制现有的 sky_settings 文件来获取它们。

    • "ambient_colors" -- 类似于光色，也对低质量渲染器有用。

    • "angles" -- 定向光的 x、y、z 角度。只有最重要的光可以投射阴影。

    • "depth_offset_values" -- 光的深度偏移。

  • "params" -- 雾的参数。

    • "height_fog" -- 在一天中的每个时间，应该有什么雾高度（在世界空间中）、雾厚度因子、雾噪声因子和雾距离因子。
    • "height_fog2" -- 在一天中的每个时间，应该有什么 X 轴雾噪声缩放、Z 轴雾噪声缩放和雾噪声速度。
    • "height_fog_color_mult" -- 一天中每个时间的雾的颜色。
    • "scattering" -- 在一天中的每个时间，应该有怎样的色调偏移、饱和度因子、值因子和太阳亮度用于光的散射。

  • "colors" -- 更多参数。

    • "starfield_brightness" -- 星星的颜色。通常我们在白天将它们设置为 [0,0,0] 以便禁用它们。

我们可以使用时间滑块调试工具测试天空渐变。
