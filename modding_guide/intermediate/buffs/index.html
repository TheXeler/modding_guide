增益效果

增益效果（包括正面和负面）是对实体属性的临时更改。它们是对某种情况或事件的响应而应用的。例如，睡在地上可能会给炉民添加昏昏沉沉的增益效果，导致他们在醒来后的几个小时内行动变慢。

增益效果组件用于添加、移除和跟踪炉民当前拥有的所有增益/减益效果。我们在 JSON 文件中声明的增益效果数据由 stonehearth/components/buffs/buff.lua 控制器管理。

现有的增益效果可以在 stonehearth/data/buffs 中找到。stonehearth/jobs 的子文件夹中可能还有一些。

创建增益效果

我们可以轻松地仅使用 JSON 创建修改炉民某些属性的增益效果，或者使用小型 Lua 脚本执行更复杂的操作。不要忘记在清单中为您的增益效果添加别名！

以下是我们可以在增益效果的 JSON 文件中找到的属性：

  • "type" : "buff" -- 我们声明此文件代表一个增益效果。

  • "axis" -- "buff"、"neutral" 或 "debuff" 之一。这将在我们选择炉民时将增益效果图标定位在单位框架的不同行中。

  • "display_name" -- 增益效果的本地化名称，显示在 UI 的工具提示中。

  • "description" -- 增益效果工具提示的本地化描述。

  • "icon" -- 增益效果的图标。

  • "duration" -- 增益效果持续时间的时间表达式（游戏内时间）。一旦此持续时间结束，增益效果将自动移除。如果不包含此字段，增益效果将是永久性的。

  • "persistent" -- 此增益效果是否应在保存游戏时存储；如果加载时由其他脚本运行添加增益效果的逻辑，则将其保留为 false。当为 true 时，它将在加载时由增益效果控制器（buff.lua）自动重新应用。

  • "repeat_add_action" -- 如果在增益效果尚未完成其持续时间时重新添加该效果，该怎么办。它可以是 "renew_duration"（将重新启动持续时间）、"extend_duration"（将增益效果的持续时间与当前剩余时间相加）或 "stack_and_refresh"（为增益效果添加一层堆叠并重新启动其持续时间）。

    如果不包含此字段，增益效果将继续其当前持续时间直到过期。

  • "max_stacks" -- 仅在"repeat_add_action"为"stack_and_refresh"时使用。这是增益效果可以堆叠的最大次数（每层堆叠将在当前值之上重新应用修饰符）。

  • "cooldown_buff" -- 可选字段，指向在此增益效果持续时间过期时将应用的增益效果的 URI。当冷却增益效果处于活动状态时，您将无法重新应用此增益效果。您可以在易怒增益效果（furious/winded 增益效果）中看到一个示例。

  • "invisible_to_player" -- 可选字段，默认为 false。当为 true 时，当我们选择实体时，增益效果不会出现在单位框架中。如果增益效果的作用不应传达给玩家（例如，如果它只是为其他脚本设置标志等），这很有用。例如，我们使用游泳增益效果来改变炉民在水中的姿势并使他们移动更慢）。

  • "effect" -- 可选字段。添加增益效果时将运行的效果的 URI（例如，耐力药剂在使用时会使橙色粒子云在炉民身上出现一秒钟）。

  • "modifiers" : {} -- 在这里我们可以修改属性和统计数据。它们的定义方式与特性相同，只是这里的键称为 "modifiers"，而不是"attribute_modifiers"。

    例如，如果我们在增益效果的 JSON 文件中添加以下内容：

    "modifiers" : {
   "speed" : {
      "add" : 20
   }
}

    在游戏中，当我们将鼠标悬停在角色表属性选项卡中的属性上时，我们将看到它。这样我们就知道哪个增益效果正在修改属性以及修改了多少： 如果增益效果/特性被移除，属性修饰符和属性将自动撤销。

  • "properties": [] -- 可选字段。我们可以在脚本中使用的属性数组。与特性相同。

  • "script" -- 可选字段。添加和移除增益效果时将运行的脚本的 URI。您也可以重用 stonehearth mod 中的现有脚本。寻找与您想要实现的功能相近的增益效果。脚本结构如下：

    local MyBuffScript = class()

function MyBuffScript:on_buff_added(entity, buff)
   local info = buff._json.script_info
end

function MyBuffScript:on_buff_removed(entity, buff)
   -- 在这里我们可以清理由增益效果创建的内容
   -- 例如，销毁计时器、监听器等
end

return MyBuffScript

    entity 参数是已添加/移除增益效果的实体，buff 参数代表增益效果控制器，因此我们可以如上所示从 JSON 文件中检索信息。

  • "script_info" : {} -- 如果我们为此增益效果声明了脚本，我们可以在此处为其定义数据（实际上我们可以在 JSON 文件中包含任何字段以在增益效果脚本中检索它，因此如果您愿意，此字段可以称为"data"）。

  • "injected_ai": {} -- 我们可以通过增益效果注入 AI。语法与通过装备部件注入 AI时相同。

  • "set_posture": "stonehearth:swimming" -- 我们可以像这样设置炉民的姿势。姿势的键在实体的 rig 的 postures.json 文件中定义。

您可能会在一些现有的增益效果中看到"tags"属性。它不再使用，因此您可以省略它。

应用和移除增益效果

有几种方法可以将增益效果应用到实体：

  • 通过装备部件中的 "injected_buffs" 属性。一旦实体装备了装备部件，这将立即应用增益效果。通常我们在这种情况下使用没有持续时间的增益效果，以便它们仅在卸下装备部件时才过期。

  • 通过实体（例如怪物）或装备部件（通常是武器）的 entity_data 中的 "inflictable_debuffs" 属性。例如，不死生物在战斗时有 50% 的几率施加此减益效果：

    "entity_data": {
   "stonehearth:buffs": {
      "inflictable_debuffs": {
         "infection": {
            "uri": "stonehearth:buffs:undead_infection",
            "chance": 0.5
         }
      }
   }
}

    请记住，这里我们不是将增益效果添加到具有此实体数据的实体，而是添加到战斗中被击中的其他实体。

  • 通过战斗攻击中的 "inflictable_debuffs" 属性。语法与上面相同（链接到增益效果 URI 的标识符和几率）。

  • 通过使用消耗品：一些药剂中有一个命令，当我们点击它时会运行一个函数。此函数查找消耗品的 entity_data 中的以下信息：

    "stonehearth:consumable": {
   "script": "stonehearth:consumables:scripts:buff_town",
   "buff": "stonehearth:buffs:courage_tonic"
}

    脚本会将增益效果应用到所有炉民。您可以重用它或将其用作示例。

  • 在 Lua 脚本中（我们还可以传递带有选项的额外参数）：

    entity:get_component('stonehearth:buffs'):add_buff('silent_woods:buffs:spooked')
-- 也可以：
radiant.entities.add_buff(entity, buff_uri)

增益效果在以下情况下被移除：

  • 其持续时间过期。

  • 应用它们的装备部件被卸下。

  • 在 Lua 脚本中（我们还可以传递一个额外参数作为移除所有堆叠的标志）：

    entity:get_component('stonehearth:buffs'):remove_buff('silent_woods:buffs:spooked')
-- 也可以：
radiant.entities.remove_buff(entity, buff_uri)

测试增益效果

我们可以轻松添加增益效果以检查它们是否按预期工作，甚至在将它们挂钩到装备部件或脚本之前。

一旦我们在清单中为增益效果添加了别名，我们就可以在游戏中选择一个炉民，打开默认控制台并输入：

  add_buff my_mod:buffs:my_buff

然后按 Enter。我们应该能够在炉民的单位框架旁边看到增益效果的图标。如果增益效果对玩家不可见，我们可以打开对象浏览器，点击增益效果组件的值，并检查它是否列在那里。

我们也可以使用 remove_buff 命令以相同的方式移除增益效果。
