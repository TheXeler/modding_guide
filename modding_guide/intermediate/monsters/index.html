生物和怪物

怪物的创建方式与任何其他生物实体（生物、Hearthlings...）相同，但如果我们想将它们用于遭遇战（突袭等），我们需要创建一些额外的文件。我们可以在 stonehearth/services/server/population/data 下找到现有的人口。

如何创建 mob

Stonehearth 中的所有生物实体都有 stonehearth/mixins/mob.json mixin，要么直接有，要么通过其他 mixins 有。此 mixin 添加了一些基本属性，如 mob/渲染标志、AI 组件、基本属性/可消耗资源（生命值），以及它们需要的一些基本 entity_data。

它们的 JSON 文件类型为 "type" : "entity"。怪物有 monster.json mixin，它比 Hearthlings 使用的 base_human.json mixin 具有更少的组件和数据。动物使用 critter.json mixin。你可以为自定义实体重用这些 mixins。怪物 mixin 有一些基本组件，如姿势、视觉传感器、job 组件，以及一些额外的属性和 AI 包，使它们能够战斗。

你可以添加你的实体需要的任何其他内容，就像创建可放置对象等时一样，例如 model_variants 组件等。请注意，复制现有敌人可能涉及许多文件，并且需要一段时间来修复不起作用的内容，所以要慢慢来。

最后，确保在清单中为你的实体添加别名。例如：

  "aliases" : {
     "monsters:vampires:vampire": "file(entities/monsters/vampires/vampire)",
     "monsters:vampires:vampire_female": "file(entities/monsters/vampires/vampire_female)"
  }

AI 和动画

你可以在美术指南中查看如何设置实体的 rig。根据你的生物/怪物可以执行的 AI 行为，你将需要不同的动画。如果游戏尝试运行 AI 行为但找不到其相关的效果或动画文件，将会弹出错误，这样你就知道问题在哪里。

首先，你的实体至少需要一个 idle_breath.json 效果和一个 run.json 效果，以便它们在站立和奔跑时看起来有生气。在实体数据中，你可能会看到某些 mobs 的这一部分，列出了效果：

  "stonehearth:idle_effects": {
     "effects": [
        {
           "name": "idle_breathe",
           "weight": 5
        },
        {
           "name": "idle_look_around",
           "weight": 2
        },
        {
           "name": "idle_sway",
           "weight": 2
        },
        {
           "name": "emote_count",
           "weight": 0.25
        }
     ]
  }

这些是当实体空闲时将基于权重播放的效果。

如果你重用现有的 rig，你将立即拥有其所有现有动画。对于自定义 rigs，根据它们可以执行的 AI 行为来确定你需要哪些动画可能需要一段时间。你可以检查你的实体使用的 AI 包中的 AI 行为的 Lua 文件（在 stonehearth/ai 下）并搜索 execute('stonehearth:run_effect'。或者，复制现有怪物/生物的效果和动画，并逐步更新它们。

设置人口文件

我们必须像实现新王国一样创建人口文件。但是，我们不需要那么多字段，因为我们不会在 UI 中显示有关人口的信息。

这些是我们通常为怪物人口拥有的字段：

  • "type" : "kingdom" -- 此文件表示一个王国。

  • "kingdom_name" -- 王国的描述性名称，例如 "Animals"、"The undead" 等。

  • "kingdom_id" -- 王国的标识符。按照惯例，尽量保持唯一，并仅使用小写字母和下划线。

  • "amenity_to_strangers" -- 对其他王国的友好度。通常将是 "hostile" 或 "neutral"。只有当他们对我们的友好度是 hostile 时，我们才能攻击 mobs。友好度可以通过代码更改，例如在 goblin 战役中，goblins 在某些条件下对我们变得 neutral。

    当我们将鼠标悬停在 hostile mobs 上时，它们将显示红色轮廓（或在整个模型中显示红色，取决于你的图形设置）。Neutral mobs（例如野生动物）将显示橙色/棕色，friendly mobs 显示白色。

    同样的规则也适用于实体（野生树木和植物的 neutral 颜色，属于我们的实体的白色颜色，以及属于敌对派系且我们可以掠夺的实体的红色颜色）。

    在多人游戏中，每个玩家将被分配一种颜色，因此 friendly 实体的颜色会有所不同。

  • "amenity_to" : {} -- 如果我们需要定义对某些其他派系的不同友好度，我们可以包含此字段。goblin_population.json 的示例：

    "amenity_to": {
   "orcs": "neutral",
   "human_npcs": "neutral",
   "orc_npcs": "neutral",
   "rabbit_biped_npcs": "neutral"
}

    这里我们确定 goblins 应该对 orcs 保持 neutral，并且他们不应该攻击 human/orc/rabbit NPCs（例如，在城镇等级庆祝活动期间访问你城镇的 NPCs、旅行者等，因为战役可能需要它们存在）。

    northern_alliance 人口也有此字段，以使野生动物变得 hostile，以便他们可以狩猎它们：

    "amenity_to": {
   "animals": "hostile"
}

  • "roles" : {} -- 这里我们定义此人口的市民看起来如何。通常我们会有几个角色。

    例如，在 animal_population.json 中，我们为每种动物类型都有一个条目：

    "rabbit" : {
   "male" : {
      "uri": [
         "stonehearth:rabbit"
      ]
   }
},
"deer" : {
   "male" : {
      "uri": [
         "stonehearth:deer"
      ]
   }
},
...

    我们只包含 "male" 性别（这在常量中被视为默认值），因为每个角色的所有动物看起来都一样，并在 "uri" 数组中添加相关动物的 URI。

    如果它们将有女性变体，我们可以添加 "female" 条目，如果它们将有自定义名称，我们还可以在条目内添加 "given_names" 数组等，就像可玩王国的人口文件一样。

  • "task_groups" -- 对于类似人形的怪物（goblins、orcs...），这里我们可以使用 "stonehearth:data:humanoid_task_groups"。动物人口没有此字段，因为它们在其实体文件中只有一些基本的 AI 行为，以使它们看起来有生气而不消耗太多计算机资源。

    你可以创建自己的可重用 JSON 文件，其中包含你希望此人口的所有成员拥有的任务组（记住在清单中为其添加别名，以便你可以在人口文件中使用它）：

    {
   "task_groups": [
      "stonehearth:task_groups:placement",
      "stonehearth:task_groups:restock"
   ]
}

不要忘记在清单中为你的种群添加别名！例如：

  "aliases" : {
     "kingdoms:vampires": "file(services/server/population/data/vampire_population.json)"
  }

关于怪物调整文件

我们可以根据游戏难度和玩家城镇的净资产调整怪物的属性和其他属性。

这是通过 monster_tuning 文件完成的。你可以在 stonehearth/data/monster_tuning 下找到它们。这些文件然后在战役中用于根据不同的因素缩放突袭遭遇战。

你可以在 SHED 中比较 monster_tuning 文件，要么在 Manifest 选项卡中单独查看它们，要么转到 Entity Stats Browser 并选择 Killable Entities 选项卡。
