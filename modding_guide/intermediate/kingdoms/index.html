王国

王国通过 population 服务、population 调用处理程序和 population_faction.lua 进行管理。要添加新的可玩王国，我们可以查看 rayyas_children 和 northern_alliance 模组。

添加怪物种群的步骤类似，请查看此页面。

  1. 创建人口文件。

    我们可以复制现有人口 JSON 文件，它们通常位于 stonehearth/services/server/population/data 内（对于 Rayya's Children 和 Northern Alliance 模组，它们位于其 data 文件夹内）。让我们检查这些文件中可以找到的属性：

    • "type": "kingdom" -- 此文件表示一个王国/人口。

    • "display_name" -- 要在 UI 中显示的本地化显示名称。

    • "kingdom_id" -- 王国的标识符。按照惯例，尽量保持唯一，并仅使用小写字母和下划线。

    • "favored_biome" -- 偏爱生物群系的 URI。通常，偏爱的生物群系为你的自定义王国的基础工匠职业提供足够的资源。当玩家选择在非偏爱生物群系中使用此王国游玩时，UI 中将出现警告。

      警告仅在我们选择非偏爱的原版王国时出现。对于模组化生物群系，它不会出现。尽管如此，"(favored)" 文本将出现在生物群系名称旁边，以便玩家知道在该生物群系中他们将拥有此王国职业所需的所有资源。

    • "quote" -- 未使用的遗留字段。

    • "description" -- 将出现在 UI 中可玩王国列表中的本地化描述。

    • "post_selection_description" -- 在选择王国后，将出现在屏幕左侧文本中的本地化描述。你可能需要以空格开头。

    • "image" -- 王国列表中图像的路径。通常为 400x300 像素。

    • "story_board_image" -- 在第一个登船屏幕中选择王国后，将出现在屏幕右侧的图像路径。通常描绘王国的几个市民，具有透明背景（由选择的生物群系填充）。

    • "starting_talisman" -- 你的王国将获得的第一个护身符的 URI。放置旗帜后，此护身符将与所选装备中的其他物品一起自动创建。通常是工匠职业的护身符，以便市民可以制作其他护身符并逐步解锁更多职业。

    • "starting_resource" -- 起始资源的 URI。放置旗帜后，将与此上述物品一起创建两个此资源物品。通常是一种资源，以便初始工匠可以制作基本工作台并启动城镇。

    • "job_index" -- 我们自定义王国的职业索引的 URI。查看如何创建职业以创建你自己的自定义职业。如果你愿意，你也可以重用 stonehearth 的职业。

      我们可以向职业索引添加 "base_job" 字段以指定不同于 "stonehearth:jobs:worker" 的基础职业。你应该只有一个基础职业，因为原版游戏不支持多个。通常这是一个不需要护身符的职业，以防玩家因任何原因无法制作它。查看 Worker 职业的设置方式。

    • "inventory_capacity_equation" -- 计算库存容量应如何随时间增加的方程式。对于原版游戏，它基于 Hearthlings 的数量："math.max(num_citizens * 150, 1200)"。

    • "initial_town_name" -- 默认分配给城镇的本地化名称（游戏也会附加一个随机数）。在原版游戏中，城镇 progression 战役的第一次遭遇允许玩家为他们的城镇设置自定义名称并选择奖励。

    • "camp_standard" -- 当我们为城镇放置旗帜时将出现的营地标准的 URI。在原版游戏中，城镇 progression 战役的第一次遭遇会将此旗帜更改为看起来不同并给予奖励。

    • "camp_hearth" -- 可选字段。我们可以指定自定义 URI，以便放置旗帜后出现的火坑不同。请注意，原版战役中有一些遭遇使用默认火坑 URI，因此你可能想改为创建自己的城镇 progression 战役。

    • "trappable_animals" -- 可捕捉动物索引的 URI。用于 trapping 服务中的 "stonehearth:jobs:trapper" 职业。

    • "default_material" -- 包含默认建筑材料标签的字符串。例如，对于 rayyas_children，当我们在 UI 中打开建造者时，他们将默认选择粘土材料。

    • "ordinal" -- 用于在可玩王国列表中排序王国的序数。

    • "roles": {} -- 这里我们定义新王国的市民看起来如何。

      对于可玩王国，我们只有一个角色，称为 "default"。在其中我们通常有以下条目：

      • "male" -- 这里我们定义将代表我们人口中男性市民的实体。有一个包含实体 URI（例如男性 Hearthlings）的 "uri" 数组，以及一个包含供男性市民随机选择的名称的 "given_names" 数组。

      • "female" -- 与 male 相同的键，但指向女性市民实体和女性名称。

        对于 Hearthlings，male/female 实体基本上继承 base_human.json mixin，其中包含它们拥有的所有通用组件和 entity_data，然后定义它们的动画表（rig）以及一些模型变体和自定义，以便在名册屏幕中使用。

      • "surnames" : [] -- 包含将为你的市民随机选择的姓氏的数组。

      • "attribute_distribution" -- 这里我们可以更改初始市民的属性点分布（新市民将具有完全随机的 stats）。

        • "allocated_points" -- 属性点的 "min" 和 "max" 数量。你的市民将在 "min" 和 "max" 之间拥有随机数量的点数分配给他们的 stats（例如，最少 5 点，在登船期间不可能获得 1 mind / 1 body / 1 spirit 的市民）。

        • "point_limits" -- 这里我们可以修改每个 stat（"mind"、"body"、"spirit"）的点数的 "min" 和 "max" 数量。这些 stats 是在 base_human.json mixin 中添加的属性，所以如果你要重用它们（因为 UI 已为此准备），请确保你的市民也在其 attributes 组件中定义了它们。

          原版王国的示例包括 Ascendancy 人口，它以最小 "spirit" 2 开始，Rayya's Children 以最小 "mind" 2 开始，Northern Alliance 以最小 "body" 2 开始。但请记住，新市民将具有完全随机的属性。

    • "pet_names" : {} -- 此部分包含宠物名称的数组。每个数组对应于宠物 entity_data 的 "stonehearth:species" 部分中定义的 "id"。还有一个 "default" 数组用于默认名称，以防游戏找不到合适的键。

    • "town_pieces" : {} -- 这些是组成命名城镇时我们可以获得的随机城镇名称的部分：

      • "optional_prefix" -- 有时城镇名称将具有此数组中的前缀。
      • "town_name" -- 你的王国的城镇名称数组。
      • "suffix" -- 你的城镇名称的后缀数组。像前缀一样，并不总是使用。

    • "task_groups": "stonehearth:data:player_task_groups" -- 这里我们分配用于可玩王国的任务组。更多内容见指南的高级部分。

  2. 向清单添加人口文件的别名。

    "aliases" : {
   "kingdoms:custom_kingdom" : "file(data/custom_kingdom.json)"
}

  3. 向可玩王国索引添加 mixinto。

    "mixintos" : {
   "stonehearth:playable_kingdom_index": "file(data/playable_kingdom_index.json)"
}

    我们的 mixinto 文件看起来像这样：

    {
   "kingdoms": {
      "my_kingdom_id" : "my_mod:kingdoms:custom_kingdom"
   }
}

    我们在这里使用 "kingdom_id" 作为键，指向我们人口文件的别名。

  4. 确保你拥有所有需要的文件。

    现在王国应该是可玩的。但我们可能想要设置其他内容：

    • 自定义战役 -- 默认情况下，我们将获得所有不受王国限制的战役，具体取决于我们选择的游戏难度。我们应该为我们的王国制作自定义城镇 progression 战役，或使用 mixintos 或 mixintypes 移除任何我们不想要的战役。

    • 自定义音乐 -- 默认情况下，我们将获得 Ascendancy 王国的音乐。如果我们想自定义将为我们的王国播放的音乐，我们需要 mixinto 声音常量：

      "mixintos" : {
   "stonehearth:data:sound_constants": "file(data/sound_constants.json)"
}

      我们的 mixinto 看起来像这样（简化示例）：

      {
   "music": {
      "kingdoms": {
         "my_mod:kingdoms:custom_kingdom": {
            "tier_to_day_music_track": {
               "idle_day_music_tier_1": {
                  "track"  : [
                     "stonehearth:music:lvlmusic_ray_spring_1_day"
                  ],
                  "loop"   : false,
                  "fade_in": 2000,
                  "volume" : 40
               },
               "idle_day_music_tier_2": {
                  "track"  : [
                     "stonehearth:music:lvlmusic_ray_winter_2_day"
                  ],
                  "loop"   : false,
                  "fade_in": 2000,
                  "volume" : 40
               },
               "idle_day_music_tier_3": {
                  "track"  : [
                     "stonehearth:music:lvlmusic_ray_winter_3_day"
                  ],
                  "loop"   : false,
                  "fade_in": 2000,
                  "volume" : 40
               }
            },
            "tier_to_night_music_track": {
               "idle_night_music_tier_1": {
                  "track"  : [
                     "stonehearth:music:lvlmusic_ray_summer_2_night"
                  ],
                  "loop"   : false,
                  "fade_in": 2000,
                  "volume" : 40
               },
               "idle_night_music_tier_2": {
                  "track"  : [
                     "stonehearth:music:lvlmusic_ray_winter_2_night"
                  ],
                  "loop"   : false,
                  "fade_in": 2000,
                  "volume" : 40
               },
               "idle_night_music_tier_3": {
                  "track"  : [
                     "stonehearth:music:lvlmusic_ray_winter_3_night"
                  ],
                  "loop"   : false,
                  "fade_in": 2000,
                  "volume" : 40
               }
            }
         }
      }
   }
}

      我们使用了来自 rayyas_children 王国的现有曲目来显示差异（文件在 stonehearth 模组中，这就是为什么别名以 stonehearth 开头）。

      在 "kingdoms" 内，我们添加一个带有我们王国 URI 的键。然后有不同的白天和夜间音乐部分，在其中，根据不同的城镇等级有不同的组。我们可以添加多个音乐曲目（原版的是 .ogg 格式，以节省空间），其中一个将根据一天中的时间和当前城镇等级随机选择。

      每个组还有一个 "loop" 属性，通常设置为 false，一个以毫秒为单位的 "fade_in"，以及一个 "volume"（从 0 到 100），以防我们需要调整音量，因为音乐文件与其他游戏中的音乐/声音相比太 quiet/loud。

    • 自定义 UI -- 我们可以选择在 UI 中使用不同的颜色和图像，就像 Rayya's Children 和 Northern Alliance 所做的那样，通过在选择我们的王国后应用延迟模组清单。这将在热加载清单部分中介绍。

  5. 最后，我们可以测试我们的新王国。添加新王国是一项大量工作，但我们可以从复制现有王国开始，在所有内容重命名后，逐步调整所有职业、战役等。
