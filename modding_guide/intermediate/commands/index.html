什么是命令？

命令是玩家可以在他们点击的实体上发出的指令。例如，宠物的 "Release Pet"（释放宠物），装饰品的 "Move Item"（移动物品），以及工作台和工匠的 "Workshop"（工作室）。它们通过 commands 组件进行管理。

它还添加了 "Change Jobs"（更换职业）按钮，这不被视为命令，它将直接调出晋升 UI。你也可以通过点击角色表内的职业图标来打开晋升树。

向实体添加命令

要使命令在实体上可用，请转到实体的 .json 文件，并在 "components" 部分添加 "stonehearth:commands"。然后，在 "commands" 数组内引用与你要添加的每个命令相关的文件或别名：

  "stonehearth:commands" : {
     "commands" : [
        "stonehearth:commands:use_consumable"
     ]
  }

如果你想为你的命令创建 URI，请将其添加到清单的 "aliases" 部分下。

在 Lua 中添加命令

你也可以通过 Lua 代码添加命令。一些组件或其他 Lua 脚本手动添加命令，而不是在每个 JSON 文件中：

  local command_component = self._entity:add_component('stonehearth:commands')
  -- add the command you wish
  command_component:add_command('stonehearth:commands:place_item')

  local event_data = { crafter_type = json.job_alias }
  -- can also modify the command's event data
  command_component:set_command_event_data(command_uri, event_data)

在 Lua 中启用/禁用命令

获取 commands 组件，然后在其上调用 set_command_enabled(uri, is_enabled)：

  local commands_component = self._entity:add_component('stonehearth:commands')
  commands_component:set_command_enabled('stonehearth:commands:place_item', true)

你可以在 commands 组件上调用的其他函数在 stonehearth/components/commands/commands_component.lua 中定义。

命令的结构

命令是一个具有 "type" : "command" 的 JSON 文件。它有一个名称、启用/禁用状态、工具提示、图像以及在 UI 中正确显示所需的其他信息。它可以有不同的执行动作，以及基于动作类型的不同参数。

我们可以在 stonehearth/data/commands 中找到现有命令。出现在单位框架顶部的战斗命令有点不同，它们不像其他命令那样处理（它们根据 Hearthling 是否有战斗职业来显示/隐藏）。救援命令也是直接从 Javascript 代码调用，并根据 Hearthling 的状态显示/隐藏。

这些是我们可以在命令中找到的不同字段：

  • "type": "command" -- 此文件表示一个命令。
  • "name" -- 内部使用的自定义标识符。尽量保持唯一。按照惯例，我们使用小写字母和下划线（而不是空格）。
  • "display_name" -- 本地化名称，当我们将鼠标悬停在单位框架中的命令上时将在工具提示中显示。
  • "description" -- 本地化描述，将在命令的工具提示中显示。
  • "disabled_description" -- 可选字段。如果我们添加它，每当命令被禁用时，工具提示上的描述将更改为此处的描述。
  • "icon" -- UI 的图标。
  • "ordinal" -- 可选字段。一个序数，以在 UI 中按特定顺序显示不同的命令。
  • "hotkey_action" -- 可选字段。它是此命令的热键标识符的名称。如果我们通过 mixinto 将其添加到 stonehearth/data/hotkeys.json，玩家将能够在游戏设置的 Controls 选项卡中为此命令绑定键组合。
  • "default_enabled" -- 可选字段（默认为 true）。此命令是否应默认启用（某些命令在 Lua 中手动启用/禁用）。
  • "visible_to_all_players" -- 可选字段（默认为 false）。使用此字段，当其他玩家在多人游戏中选择具有此命令的实体时，即使该实体属于另一个玩家，命令也将对他们可见。
  • "action" -- 命令的动作类型。有两种类型，下面更详细地解释：
    • "fire_event" -- 此命令将触发一个事件，UI（客户端）将监听该事件。我们需要在 "event_name" 字段中声明哪个事件，并在 "event_data" 字段中定义数据（它是一个表。如果事件不需要传递任何内容，我们应该将其声明为空 {}）。
    • "call" -- 此命令将调用一个函数（在 "function" 字段中定义），带有一些参数（在 "args" 数组中定义，如果我们不需要传递任何参数或如果我们需要传递的唯一参数是具有此命令的实体，则可以为空）。

触发事件（Fire Event）

命令的一个常见动作是 "fire_event"。这意味着当我们点击命令时会触发关联的事件名称，UI 将在 .js 文件中接收它并对其进行某些操作。

例如，这是我们添加到工作台以显示工作室 UI 的命令（我们在 Lua 中添加它）：

  {
     "type": "command",
     "name": "show_workshop",
     "display_name": "i18n(stonehearth:data.commands.show_workshop.display_name)",
     "description": "i18n(stonehearth:data.commands.show_workshop.description)",
     "disabled_description": "i18n(stonehearth:data.commands.show_workshop.disabled_description)",
     "default_enabled": true,
     "ordinal": 10,
     "icon": "file(show_workshop.png)",
     "action": "fire_event",
     "event_name": "radiant_show_workshop",
     "event_data": {}
  }

上面的 "radiant_show_workshop" 事件通过来自 stonehearth/ui/game/show_workshop/show_team_workshop.js 的以下代码连接到 javascript：

  $(document).ready(function () {
     App.workshopManager.init();

     // Show the crafting UI from the workshops, and from the crafter.
     $(top).on("radiant_show_workshop", function (_, e) {
        App.workshopManager.toggleWorkshop(e.event_data.crafter_type);
     });
     $(top).on("radiant_show_workshop_from_crafter", function (_, e) {
        App.workshopManager.toggleWorkshop(e.event_data.crafter_type);
     });
  });

在此示例中，事件数据是在游戏为相应工作台初始化时通过 Lua 在工作室组件中添加的，所以我们不在命令的 JSON 文件的 "event_data" 字段中传递任何参数。这样，每个工作台的事件数据可以不同。

如果我们希望此命令始终为 crafter_type 参数传递相同的数据（例如，stonehearth 模组中的 mason），我们就不会通过 Lua 定义它，而是会像这样将其添加到命令的 JSON 文件中：

  "event_data": {
     "crafter_type" : "stonehearth:jobs:mason"
  }

调用（Call）

命令的另一个常见动作是调用函数。例如，要使用某些消耗品，我们有 use_consumable 命令：

  {
     "type": "command",
     "name": "use_consumable",
     "display_name": "i18n(stonehearth:data.commands.use_consumable.display_name)",
     "description": "i18n(stonehearth:data.commands.use_consumable.description)",
     "icon": "file(use_consumable.png)",
     "action": "call",
     "function": "stonehearth:use_consumable_on_town",
     "args": [
     ]
  }

"function" 的值是清单的 "functions" 部分中描述的函数的名称（前面加上模组的命名空间和冒号！）：

  "functions" :{
     "use_consumable_on_town": {
        "controller": "file(call_handlers/town_call_handler.lua)",
        "endpoint": "server"
     }
  }

在 "controller" 字段中，我们定义了 use_consumable_on_town 函数的代码实际所在的文件。通常它在 call handler 中（它们将在指南的高级部分中解释）。我们还必须定义 "endpoint" 是 "client" 还是 "server"，具体取决于函数应该在客户端还是服务器端执行。

在我们命令的 JSON 文件中，我们有 "args" 数组来向函数传递参数（查看 default_arrows.json 和 summon_titan_from_gong.json 命令以获取示例）。

请注意，即使我们将 "args" 数组留空，传递的第一个参数始终是执行此命令的实体。

调用对象（Call object）

如果我们添加 "object" 字段，我们可以在对象（如服务）上调用函数。例如：

  {
     "type": "command",
     "name": "restore_terrain",
     "display_name": "i18n(stonehearth:data.commands.restore_terrain.display_name)",
     "description": "i18n(stonehearth:data.commands.restore_terrain.description)",
     "default_enabled": true,
     "icon": "file(restore_terrain.png)",
     "action": "call",
     "function": "restore_terrain_command",
     "object": "stonehearth.building",
     "args": []
  }

这里 restore_terrain_command 函数的代码（注意它前面没有模组的命名空间！）在 building 服务（stonehearth.building）中定义，这是一个由游戏实例化的对象。请记住，"args" 数组的第一个对象将是执行此命令的实体。

要以这种方式在对象上调用函数，我们必须将它们添加到清单的 "object_commands" 部分（stonehearth 清单的简化示例）：

  "object_commands": {
     "building": [
        "restore_terrain_command"
     ]
  }
