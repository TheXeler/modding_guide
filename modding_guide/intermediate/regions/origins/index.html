原点

你可以使用 mob 组件定义对象的中心（原点）以及它们如何旋转。

一般来说，模型的原点位于它们所附着表面的中心。对于家具，它们通常附着在地面上，所以原点在底部的中心。一些装饰品附着在墙上，所以它们的原点是附着到墙上的方块的中心。其他像武器和盔甲这样的物品有特殊的原点规则，因此这里不讨论。查看此页面了解如何对齐它们。

如果你已启用调试键，你可以按 F11 查看给定对象的轴。它们代表该模型在体素编辑器中的原点 (0,0,0) 的位置，对象在游戏中围绕其旋转的点，以及游戏认为实体所在的位置。这是一个桶图标，你可以看到原点来自底部的中间。

模型原点

model_origin 是一个偏移量（根据世界轴坐标系定义），以便模型与世界良好对齐。它基于模型本身的原点（在 Qubicle/你的体素编辑器中定义）。

通常所有对象（除了可悬挂的、武器和大多数 meant to be animated 的对象）在体素编辑器中以其宽度和长度的 0 为中心并与地面对齐。

我们可以使用 debugtools 来帮助我们确定模型原点。在游戏中，将物品放置在地面上并打开Entity Editor。通过取消选中 "Axis Alignment Flags" 下 X 和 Z 的复选框并点击 "Update" 按钮，确保对象未轴对齐。

现在我们可以点击 "Rotate Entity" 按钮（一个绿色圆形箭头），看看我们的模型是否正确旋转（第一次点击此按钮时，它将把模型重置为默认旋转，然后下次它将按顺序循环遍历其余方向）。

无论面向哪个方向，轴都应该与模型的完全相同部分匹配。

如果我们的模型没有正确旋转，我们需要确定它的 model_origin 应该在哪里。

  • 奇数尺寸模型的模型原点：

    有些模型的 X 和 Z 维度不能正好被 2 整除。设置它们的模型原点很容易，因为有一个单独的体素是模型的中心。

    当这种情况发生时，你将该维度设置为 0.05，以便它校正世界中的对齐。值应始终为 0 或 + 或 - 0.05（鉴于对象的默认缩放为 0.1）。如果模型需要更大量的定位，那么你可以在体素编辑器中进行。

    例如，如果你在 Qubicle 中创建一个在 X 和 Z 方向上为 3x3 方块的对象，你需要将 model_origin 设置为类似 X 为 -0.05，Z 为 0.05。在 Entity editor 中尝试并点击 "Update" 应用更改，然后再次尝试旋转对象并观察模型是否移位。

    你对 X 或 Z 使用 -0.05 取决于你如何在 Qubicle 中围绕原点定向对象本身。在此示例中，中心体素放置在 X 和 Z 的正方向。X 设置为 -0.05，因为这是原点 (0,0,0) 与 Qubicle 中对象的实际中心 (0.05, 0, 0.05) 之间的差异。

    但为什么当 X 为 -0.05 时 Z 为 +0.05？如前所述，Qubicle 的坐标系与 Stonehearth 的坐标系不同。确切的差异是 Z 轴的方向。在 Qubicle 中，正 Z 轴指向对象的背面（或"北"）。在 Stonehearth 中，正 Z 轴指向对象的前面（或"南"）。所以你实际上想在 Z 轴上使用与 Qubicle 中模型位置相反的 adjustment。

  • 偶数尺寸模型的模型原点：

    对于一个侧面之一或两者都是偶数的模型，你必须决定模型将围绕哪个偏离中心的点旋转。

    如果我们有一个宽和长为 2 个世界单位的对象，那么对象的真正中心是 4 个方块连接的地方，而不是一个的中心。例如，如果我们要对齐这个铁砧以占据 2x2 单位，中心将是黄色圆圈所在的位置：

    任何网格方块 A、B、C 或 D 都可以作为旋转点。由于铁砧将是 2x2，让我们选择网格方块 A 作为旋转中心。我们必须手动将铁砧模型在 X 和 Z 方向移动半个方块，以便象限 A（轴所在的位置）在左后角。

    将模型原点设置为 (0.5, 0, 0.5) 会移动铁砧模型，以便在视觉上，模型的左后部在绿线上方。现在铁砧适合 2x2 网格，这正是我们想要的：

如果你在 render_info 组件中修改了物品的缩放，原点的值可能不完全是 0.05 或 0.5，你必须尝试其他值直到感觉正确。

关于轴对齐标志

轴对齐标志是一种将模型对齐到网格的廉价方法，特别是如果它们是偶数边，但如果其中一个长度是奇数，则可能需要在 QB 文件中移动模型。如果对象有期望模型矩阵在特定位置的动画，则可能并不总是可以使用它们。

在 stonehearth 模组中，它们通常用于窗户、门、一些容器和其他一些与世界具有 1:1 缩放的实体。你可以尝试在 Entity editor 中选中一个或两个，看看它们是否像你希望的那样将模型对齐到世界。

请记住，你可能仍然需要在 QB 文件中调整模型或手动调整模型原点。

区域原点

这是 region_collision_shape 和目标区域原点的偏移量，以便它们与世界良好对齐。

它基于模型原点。由于模型的原点在世界单位的中心，通常你会希望将 region_origin 设置为类似

  "mob" : {
     "region_origin" : { "x": 0.5, "y": 0, "z": 0.5 }
  }

这将区域的原点放在模型原点东南方向的方块角落。你可以使用其他值，但一致性有帮助。如果你的 region_origin 不与方块角落对齐，那么在旋转对象进行放置时，你的区域很可能不会在世界中正确对齐。

当对象的真正中心与 4 个方块的角落相同时，我们应该始终将 region_origin 设置为该点。如果真正中心在方块的中心，那么我们应该将 region_origin 设置为该方块的东南角。

对于 2x2 对象，model_origin 和 region_origin 将是

    "mob" : {
       "model_origin" : { "x": 0.5, "y": 0, "z": 0.5 },
       "region_origin" : { "x": 0.5, "y": 0, "z": 0.5 }
    }

如果你的 region_origin 不正确，碰撞区域将不会与模型旋转时的位置匹配。

原点应该放在哪里

对象的原点信息应在可用时放入 ghost.json 文件。常规 JSON 文件从 ghost.json 文件混合信息，所以你不需要重复它。

如果你的对象不能由玩家放置在世界上，那么它将没有 ghost.json 文件，只有在这种情况下你才将原点信息放在常规 json 文件中。如果不这样做会导致 ghost 对象和常规对象没有相同原点的情况，并且在放置物品时对象的位置会跳跃。

一旦你在 Entity editor 中正确对齐了对象并整理了数字，就该将你的数字粘贴到 JSON 文件中了。

要获取 mob JSON，点击 Entity Editor 中 Mob 条目旁边的卷轴图标。它被称为 "Show Json"。然后从弹出的窗口中复制自动生成的 JSON 并将其粘贴到 ghost.json 文件的组件部分。

关于可悬挂物品

对于在其 entity forms 组件中具有 "placeable_in_walls" 标志的物品，模型原点也可能在 Y 轴上移位（并非所有物品都需要）。不过，使用 Entity editor 编辑可悬挂物品可能很困难，所以你可能需要直接在 JSON 中调整值。

例如，挂毯有这个模型原点：

  "model_origin": { "x": 0, "y": -0.5, "z": -0.5 }

在 Qubicle 中，我们需要将将从墙上悬挂的模型部分与 (0,0,0) 对齐。它还应该在负 Z 空间中，以便以后更容易与墙壁对齐。一些可悬挂物品（如窗户的花盆）在 Qubicle 中可能没有像这样对齐。

可悬挂物品的区域原点遵循与任何其他实体相同的规则。
