Traits

Traits 组件保存实体所有 traits 的映射。对于 Hearthlings，它在 base_human.json mixin 中添加。

在游戏过程中，可以通过此组件添加、移除和检索 traits。你可以在 stonehearth/data/traits 中找到所有现有的 traits。

创建 trait

Trait 由一个 JSON 文件组成，该文件描述显示名称、描述和图标等功能。

要创建新 trait，请在 manifest.json 中添加新别名。例如：

  "aliases": {
     "traits:impatient" : "file(data/traits/impatient/impatient_trait.json)"
  }

然后将该别名录入 mixinto 到 stonehearth/data/traits/traits_index.json，以便它可以包含在可以添加到新 Hearthling 的 traits 列表中：

  "mixintos": {
     "stonehearth:traits_index" : "file(data/traits/traits_index.json)"
  }

Mixinto 看起来像这样：

  {
     "traits": {
        "my_mod:traits:impatient": {}
     }
  }

我们的 impatient_trait.json 文件看起来像这样：

  {
     "type": "trait",
     "display_name": "i18n(my_mod:data.traits.impatient.impatient_trait.display_name)",
     "description": "i18n(my_mod:data.traits.impatient.impatient_trait.description)",
     "icon": "file(impatient_trait.png)",
     "script_controller": "my_mod:trait_scripts:impatient",
     "data": {
        "impatient_thought": "stonehearth:thoughts:traits:got_delayed",
        "negative_thought": "stonehearth:thoughts:traits:made_me_hurry"
     },
     "attribute_modifiers": {
        "speed": {
           "add": 30
        },
        "social_loss_override": {
           "min": 0.3
        },
        "diligence": {
           "add": 15
        }
     }
  }

Traits 文件具有以下字段：

  • "type" : "trait" -- 此文件表示一个 trait。

  • "display_name" -- 要在 UI 中显示的 trait 的本地化名称。

  • "description" -- 要在 UI 中显示的 trait 的本地化描述。你可以在 en.json 文件的描述文本中使用 [name(i18n_data.entity)] 来引用具有此 trait 的 Hearthling 的名称。

  • "icon" -- 要在 UI 中显示的图标。

  • "script_controller" -- 可选字段。将使用参数 entity、uri、parent、args 创建的控制器。脚本可用于创建与 trait 关联的自定义行为。

    例如，food_preference trait 脚本（stonehearth/data/traits/food_preference/food_preference_script.lua）从父 trait 获取数据（使用 uri 参数），并使用数据更改实体的消费偏好，并在实体吃不喜欢的食物时添加想法。

    请记住，脚本控制器的别名必须添加到清单的 "controllers" 键中，而不是 "aliases" 键下。

  • "data" -- 可选字段。如果我们为 trait 定义了脚本控制器，我们可以在此字段中添加数据以从控制器中检索它。

  • "attribute_modifiers" -- 可选字段。Hearthling 属性的修饰符。详见下文。

  • "subject_overrides" -- 可选字段。主题事项（Hearthling 对话中使用的话题）的覆盖数组。添加到 Hearthling 的每个主题都有相关的意见。意见是随机选择的，除非有其他内容覆盖它们。例如，角色表 Appeal 选项卡中观察到的物品偏好将为喜爱/喜欢的物品产生正面意见，为不喜欢的物品产生负面意见。

    此字段用于向 Hearthling 添加永久主题事项或覆盖其现有意见。示例可以在 cultist trait 和 passion_job traits 中看到。数组中的每个条目有 4 个键："subject"（主题的 URI）、"sentiment"（-1 表示负面，0 表示中性，1 表示正面）、"permanent"（此主题是否应永久存在或在一段时间后消失）和 "unvarying"（Hearthlings 在对话时可以改变他们对主题的意见，如果此属性为 true，他们不会）。

  • "properties" -- 可选字段。它是一个字符串数组，我们可以像自定义标签一样添加它们，并在 Lua 脚本中检查它们。详见下文。

"magnificent_beard" trait 在 traits 索引中列出了一些属性。

"immigration_only" : true 属性将使此 trait 仅在 Hearthling 是移民时分配（因此它在登船期间的选择名册屏幕上永远不会随机出现）。

"gender" : "male" 属性将使只有男性 Hearthlings 才能获得此 trait。你可以单独使用此属性而不使用 "immigration_only"，但玩家仍然可以在名册屏幕上更改性别（这不会移除 trait）。

此 trait 具有这两个属性，因为女性在自定义列表中没有面部毛发功能，所以为了防止登船期间出现错误，它被设置为 "immigration_only"。

要测试 traits，我们可以在自定义屏幕上多次重新滚动直到获得新 trait，但这需要时间。相反，让我们通过默认控制台手动向 Hearthlings 添加/移除我们的 traits。我们可以选择一个 Hearthling 并在默认控制台中输入：

  add_trait my_mod:traits:impatient
  remove_trait my_mod:traits:impatient

注意 URI 作为参数传递时不带引号。

如果属性高于其正常值，它们将在 Attributes 选项卡中以绿色显示，如果低于正常值则以红色显示。它们的工具提示将显示导致值增加/减少的 buffs，但不会显示影响它们的 traits（不过你可以在 trait 描述中暗示这一点）。

属性修饰符

如果 trait 更改某些数值，请将其添加到 trait JSON 文件中的 "attribute_modifiers" 内，并让管理数值更改行为的任何内容检查属性值。

请注意，属性修饰符不会保存，并且在加载时会重新应用到实体。

这些修饰符在 stonehearth/lib/modifiers/modifiers_lib.lua 中管理。我们也可以以这种方式向实体添加属性（例如，上面示例中的 "social_loss_override" 默认不包含在 Hearthlings 属性中，但有脚本检查它。我们可以注入新的数值并在其他 Lua 文件中检查它们）。

修饰符的可能值是 "add"、"multiply"、"min" 和 "max"。使用 "add" 我们也可以减去（如果我们提供的值是负数），使用 "multiply" 我们也可以除以（如果我们提供小数值；例如，要将属性更改为其当前值的五分之一，我们将乘以 0.2，即 1/5）。 Multiply 在 add 之前完成，然后是 min，最后是 max。因此，如果有多个 buffs 或 traits 影响此属性，它们将按该顺序应用。

Properties

我们可以通过 traits 向 Hearthlings 添加自定义 properties，然后在其他 Lua 脚本中检查它们。例如，社交组件检查注入到具有 loner trait 的 Hearthlings 的此 property：

  if radiant.entities.has_property(self._entity, 'avoids_conversations_with_citizens') then
     return -- entity doesn't initiate conversations with fellow citizens
  end

Trait 排除

如果你查看 stonehearth 模组中的 traits_index.json，你会注意到顶部有一个 "groups" 键。这允许定义相关或相反 traits 的组以创建排除。当滚动 traits 时，如果 Hearthling 被分配了来自其中一个组的 trait，他们将不会被分配同一组内的不同 trait（Hearthlings 默认可以有 1 或 2 个 traits）。

如果我们的自定义 trait 与其他任何 trait 无关，我们可以像上面的示例一样在 "traits" 键下定义它们。

我们还可以像这样在 "groups" 或 "traits" 内定义 traits 的排除：

  "stonehearth:traits:green_thumb": {
     "excludes": {
        "stonehearth:traits:carnivore": true
     }
  }

这样我们就不必在 "groups" 内添加或移动 traits，这可能会影响其他 traits（traits 在整个索引文件中只定义一次），并且可以为不同的 traits 排除相同的 trait。

自定义 traits 索引

基础王国默认从 "stonehearth:data:traits" 索引获取他们的 traits，但我们可以为模组化人口指定不同的 traits 索引。在他们的人口 JSON 文件中，添加此字段：

  "traits_index" : "my_mod:data:traits"

通常 NPC 派系不使用 traits，只有可玩派系使用。
