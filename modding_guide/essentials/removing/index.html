如何使用 mixintypes 移除内容

现在我们知道如何使用 mixintos 和 overrides 添加/修改内容。但是如果我们不想添加内容，而是想移除它怎么办？我们需要覆盖整个文件吗？

如果我们想要移除某些战役、制作配方或 Hearthling 的名称怎么办？我们可以使用 overrides 来做到这一点，但正如我们所解释的，overrides 对模组兼容性不利。

幸运的是，模组 API 有一个系统允许我们仅覆盖 JSON 文件中的某些值。这是一种让模组制作者指定 mixinto 的某个部分将覆盖整个节点的方法。这允许我们替换数组而不是向它们添加内容。

使用 "mixintypes" 在 mixintos 中覆盖 JSON 节点

通常，mixinto 只能向现有数组添加内容，这意味着替换数组中的值很困难。要替换整个数组，你可以要么在清单中将 JSON 指定为 override，这将清除原始文件（可能导致游戏损坏），要么在你的 mixinto 中创建一个特殊的 "mixintypes" 字段，位于与你想要修改的字段相同的 JSON 级别，以仅替换原始文件中的单个节点。

Mixintype "remove"

要从 JSON 中移除节点，语法如下：

  "mixintypes": { "name_of_node" : "remove" }

例如，这是 Rayya's Children 织布工的配方列表如何从可用配方列表中移除 Ascendancy 的冬季工人服装（还将 worker_outfit_2 的配方替换为其自己的配方，并为另一套服装添加额外配方）：

  {
     "mixins": "/stonehearth/jobs/weaver/recipes/recipes.json",
     "craftable_recipes": {
        "clothing_armor": {
           "recipes": {
              "worker_outfit_2": {
                 "recipe": "file(worker_outfit_2_recipe.json)"
              },
              "mixintypes": {
                 "worker_outfit_winter": "remove"
              },
              "outfit_worker_goldthread": {
                 "recipe": "file(outfit_worker_goldthread_recipe.json)"
              }
           }
        }
     }
  }

"mixintypes" 关键字与我们想要移除的键处于同一级别。我们告诉 mixintype 中想要修改哪些节点，然后指示我们要移除它。这可以在 mixinto 中或在其他地方引用的新文件中完成。

上面的示例通过 "mixins" 导入 Ascendancy 织布工的所有配方，然后修改与 Rayya's Children 不匹配的配方。它是一个在其他文件中引用的文件，最终通过 mixinto 添加，但它本身不是 mixinto。它只是借用 stonehearth 模组的数据，这样就不必手动指定所有配方。

我们不能对数组值使用 "mixintypes" : "remove"，因为数组不能包含键值对。

Mixintype "override"

你可以使用 mixintype "override" 来替换现有数组而不是追加到它们。这对于曾经覆盖人口 JSON 文件以获取翻译名称的翻译模组非常重要。

使用 mixintype "override"，你现在可以在模组的清单中将翻译的人口名称作为 mixinto 提供，而不是覆盖整个 population.json 文件。

在目标文件中，声明你想要覆盖的节点，然后提供你的替换版本。
在下面的示例中，你的清单看起来像这样： 

  {
     "info" : {
        "name" : "My Translation",
        "namespace" : "my_namespace"
        "version" : 3
     },
     "mixintos" : {
        "stonehearth/locales/supported_languages.json" : "file(added_languages.json)",
        "stonehearth/services/server/population/data/ascendancy_population.json": "file(translations/stonehearth/population/ascendancy_population.json)",
        "stonehearth/services/server/population/data/goblin_population.json": "file(translations/stonehearth/population/goblin_population.json)",
        "stonehearth/services/server/population/data/undead_population.json": "file(translations/stonehearth/population/undead_population.json)",
        "rayyas_children/data/rayyas_children_population.json": "file(translations/rayyas_children/population/rayyas_children_population.json)"
     },
     "overrides" : {
        "stonehearth/locales/en­XA.json": "file(translations/stonehearth/en­XA.json)",
        "rayyas_children/locales/en­XA.json": "file(translations/rayyas_children/rayyas_children.json)"
     }
  }

（还记得使用 overrides 向 stonehearth 文件夹添加不存在的文件吗？这是另一个示例。） 然后人口的 mixinto 文件看起来像这样（简化示例）： 

  {
     "roles" : {
        "default" : {
           "male" : {
              "mixintypes": {
                 "given_names": "override"
              },
              "given_names" : [
                 "MaleName1",
                 "MaleName2",
                 "MaleName3"
              ]
           },
           "female" : {
              "mixintypes": {
                 "given_names": "override"
              },
              "given_names" : [
                 "FemaleName1",
                 "FemaleName2",
                 "FemaleName3"
              ]
           },
           "mixintypes": {
              "surnames": "override"
           },
           "surnames" : [
              "Surname1",
              "Surname2"
           ]
        }
     },
     "town_pieces" : {
        "mixintypes": {
           "optional_prefix": "override"
        },
        "optional_prefix" : [
           "Prefix1",
           "Prefix2",
           "Prefix3"
        ]
     }
  }

正如你所见，我们以与移除内容的 mixintypes 相同的方式定义 "mixintypes" 节点，即在我们想要覆盖的节点的级别。然后，我们指定节点（例如 "given_names"）加上冒号和 "override"。只有在那之后，我们才能再次声明该节点并包含我们希望它具有的值。

这是覆盖数组而无需覆盖整个文件的方法，如果游戏更新，这可能会导致 JSON 文件中其他节点的冲突。

你也可以使用此方法从数组中移除特定值，通过覆盖节点然后仅声明你想要保留的值。

从 Lua 文件中移除内容

不幸的是，上面解释的系统不适用于 Lua 文件，所以如果你想从它们中移除内容，你将不得不覆盖整个文件，这意味着它与依赖该文件的其他模组不兼容。
