调试技巧

在模组制作过程中，意外破坏某些功能是很常见的，所以当某些东西没有按预期工作时，这里有一些提示：

  1. 查看 stonehearth.log 文件。它位于 Stonehearth 安装的文件夹中，每次运行游戏时都会被覆盖。 如果某些文件无法正确加载，将在日志中提到（有时错误只出现在日志中，而不会在游戏中弹出）。你还可以从模组的 Lua 文件中向 stonehearth.log 输出文本。说明。

  2. 在开始游戏之前，确保你的模组已启用。如果你的清单文件有问题，它可能会在模组菜单中被标记为无效或过时，游戏将不会加载你的模组。

  3. 使用 JsonLint 验证你的 JSON 文件。这将告诉你是否有任何缺失或多余的逗号，这是一个常见的拼写错误，会导致游戏无法正确读取你的文件。你还可以使用 JSON Diff 将你的 JSON 文件与游戏中的其他文件进行比较。即使键值对的顺序不同，此工具也会告诉你差异，这对于检查是否遗漏了某些键或值很有用。

  4. 搜索别名或本地化字符串中的拼写错误。它们可能包含错误的字母，因此引用了不存在的内容，或者你忘记在文件中为别名添加正确的命名空间前缀。你还可以使用 SHED 打开你的模组，它会告诉你是否有指向不存在文件的内容。忘记为 mixintos 和 overrides 添加模组命名空间前缀也很常见。
    如果你从某些文档（如 .pdf 文件或 .doc 文件）复制粘贴文本，可能会有游戏认为无效的不可见字符，所以要小心。

  5. 在新游戏中测试而不是在存档中测试。某些更改不会应用于旧实体，因此你需要创建新实体才能看到它们。

一般来说，当游戏中出现错误时，它将包含失败代码的跟踪信息，因此你应该能够通过查看该代码来找出问题。即使跟踪指向 stonehearth 模组中的文件，错误也可能是由你的模组中的错误数据引起的。
这些错误也将打印在 stonehearth.log 中，所以如果游戏关闭，你仍然可以检查日志，直到你重新打开游戏。

如果你的游戏在测试模组时崩溃并且现在无法打开，请检查任务管理器，看看是否有 Stonehearth.exe 进程在后台运行。如果需要，右键单击它们并选择"结束任务"。
在极端情况下，这可能是由于你的 user_settings.json（位于与 stonehearth.log 相同的文件夹中）。你可以尝试删除它并重新启动游戏（文件将被重新创建）。

调试用户界面

对于用户界面错误，通常出现在屏幕右侧，你必须始终重新加载用户界面（使用 F5 或错误对话框中的按钮）。它可能处于不一致状态，除非重新加载，否则你将无法继续使用它。如果你忘记查看错误对话框中的跟踪信息，你也可以查看 stonehearth.log（前提是你尚未重新启动游戏）。

重新加载用户界面将使任何非用户界面错误重新出现，即使你已经关闭了它们。如果你已经检查了所有错误，只需再次关闭它们即可。

如果你制作与用户界面相关的模组，你可以使用 CEF 调试器，前提是你安装了 Google Chrome：

  1. 在游戏运行时，打开 Chrome。
  2. 在地址栏中输入 localhost:1338 并按 Enter 键。
  3. 点击 "Stonehearth UI" 链接。如果你不在游戏中（例如，如果你在加载游戏），可能会出现其他链接。
  4. 当你需要重新加载调试器时，确保地址仅为 localhost:1338，因为浏览器会在使用时向其附加参数。
  5. 还要确保你只有一个 Stonehearth 实例在运行，否则端口可能会更改并且不响应你想要的实例。

浏览器应用了安全更新，禁止这种形式的调试。你可以使用旧版浏览器，其中尚未实施此类安全措施。已知 Chrome 68 版本是最后一个有效的版本。最好安装不同的浏览器品牌以避免与主浏览器冲突。模组制作者推荐的一款浏览器是：Slimjet，因为它使用 chromium 引擎版本 68

使用示例：

  1. 你可以检查用户界面元素。点击"检查元素"工具，然后转到游戏，并将鼠标悬停在任何用户界面元素上。你可能需要在激活此工具之前打开要检查的用户界面视图。
  2. 点击你要检查的元素，然后转到浏览器。它将在 HTML 中被选中，你可以检查其所有属性，临时调整其 CSS 等。
  3. 你还可以在网络控制台中检查 Javascript 警告和错误。
  4. 你可以使用 CEF 调试器调试你的 Javascript 代码。请注意，有时它可能会无响应，你可能需要重新加载它。

更快地测试你的模组

为了更轻松地测试你的模组，你可以使用游戏附带的 debugtools。如何使用它们的说明在这里。默认控制台的一些命令可以通过在游戏中 shift+点击实体来使用。这比必须在控制台中输入命令更快。

厌倦了每次需要测试新更改时都启动新游戏？ 你可以通过将此内容添加到 user_settings.json 文件中与 "user_id" 节点相同的级别来缩短世界生成步骤： 

  "mods" : {
     "stonehearth" : {
        "world_generation" : {
           "method" : "tiny"
        }
     }
  }

你的设置中可能已经有这些 "mods" : { "stonehearth" : {} } 键，所以只需在相应级别粘贴 "world_generation" 节点（但记得在需要的地方添加逗号）。这将生成一个微型世界，因此加载它将花费更少的时间。

微型世界（Microworld）

你还可以在此模组的帮助下创建自己的测试世界。Microworld 一旦在你的 user_settings.json 中配置，将跳过主菜单并直接加载你的测试世界，这要快得多。

加载存档

如果你想修复自己模组的存档兼容性，有一种更快的方法来加载特定存档进行测试。你可以在 user_settings.json 中与 user_id 相同的级别添加这些键：

  "game" : {
     "load_game" : "test_save"
  }

记得在需要的地方添加逗号，以便 JSON 仍然有效。"load_game" 的值应该是你想要加载的存档文件夹的名称。

每当你想再次运行正常游戏时，不要忘记删除此键或为其添加下划线前缀（"_load_game"）以使其无效。
