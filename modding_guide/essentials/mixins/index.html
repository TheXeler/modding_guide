Mixins

Mixins 允许实体定义从 mixin 文件中引入内容。使用 Mixins，你可以定义通用数据参数并在多个实体定义中重用它们。

以下是 Stonehearth 模组清单文件的大幅简化版本。

  {
     "info" : {
        "name" : "Stonehearth"
     },
     "aliases" : {
        "medium_oak_tree" : "file(entities/trees/oak_tree/medium_oak_tree)",
        "mixins:tree" : "file(mixins/tree)"
     }
  }

以下是 entities/trees/oak_tree/medium_oak_tree/medium_oak_tree.json 的样子（在编写本指南时）：

  {
     "mixins": "stonehearth:mixins:tree",
     "type": "entity",
     "components": {
        "model_variants": {
           "default": {
              "models": [
                 {
                    "type": "one_of",
                    "items": [
                       "file(medium_oak_tree.qb)",
                       "file(medium_oak_tree_2.qb)",
                       "file(medium_oak_tree_3.qb)"
                    ]
                 }
              ]
           }
        },
        "stonehearth:resource_node": {
           "resource": "stonehearth:resources:wood:oak_log",
           "durability": 9
        },
        "mob": {
           "model_origin": { "x": -0.5, "y": 0, "z": 0.5 },
           "region_origin": { "x": 0.5, "y": 0, "z": 0.5 },
           "align_to_grid": [
              "x",
              "z"
           ]
        },
        "destination": {
           "region": [
              {
                 "min": { "x": -2, "y": 0, "z": -2 },
                 "max": { "x": 3, "y": 1, "z": 3 }
              }
           ]
        },
        "region_collision_shape": {
           "region": [
              {
                 "min": { "x": -2, "y": 0, "z": -2 },
                 "max": { "x": 3, "y": 16, "z": 3 }
              }
           ]
        }
     },
     "entity_data": {
        "stonehearth:evolve_data": {
           "next_stage": "stonehearth:trees:oak:large",
           "evolve_time": "5d"
        },
        "stonehearth:catalog": {
           "display_name": "i18n(stonehearth:entities.trees.oak_tree.medium_oak_tree.display_name)",
           "description": "i18n(stonehearth:entities.trees.oak_tree.medium_oak_tree.description)",
           "subject_override": "stonehearth:trees:oak:large"
        }
     }
  }

现在注意文件顶部的 "mixins" 行。我们通过清单中指定的别名引用 mixin，前面加上声明它的模组名称和冒号。

以下是树 mixin 的定义： mixins/tree/tree.json 

  {
     "type": "mixin",
     "mixins": "stonehearth:mixins:placed_object",
     "components": {
        "render_info": {
           "scale": 1
        },
        "stonehearth:resource_node": {
           "harvest_overlay_effect": "stonehearth:effects:chop_overlay_effect",
           "harvester_effect": "chop",
           "description": "chop_tree"
        }
     },
     "entity_data": {
        "stonehearth:catalog": {
           "category": "plants"
        },
        "stonehearth:render_materials": {
           "hud": "materials/ghost_item.json"
        },
        "stonehearth:on_destroy": {
           "effect": "stonehearth:effects:terrain:tree_harvest"
        }
     }
  }

注意类型是 "mixin"。惊喜！它还在其定义中引入了其他 mixins（"stonehearth:mixins:placed_object"）。这是一种重用数据和代码的便捷方式。你不需要在所有文件中都有 "mixins" 节点，只需要在从中受益的文件中使用。你可以为它们使用路径或别名，并且你在 "mixins" 节点中引用的文件不必是 "mixin" 类型。它们可以是任何类型。

你可以使用数组指定多个 mixins，例如，这是 cobblestone_fence_gate.json 的定义：

  {
     "type": "entity",
     "mixins": [
        "file(cobblestone_fence_gate_ghost.json)",
        "stonehearth:mixins:portals:double_gate",
        "stonehearth:mixins:basic_destructable_door"
     ],
     "components": {
        "stonehearth:entity_forms": {
           "iconic_form": "file(cobblestone_fence_gate_iconic.json)",
           "ghost_form": "file(cobblestone_fence_gate_ghost.json)",
           "placeable_on_ground": true
        }
     },
     "entity_data": {
        "stonehearth:net_worth": {
           "value_in_gold": 6,
           "rarity": "common",
           "shop_info": {
              "buyable": true,
              "sellable": true,
              "shopkeeper_level": 1,
              "shopkeeper_type": "caravan"
           }
        },
        "stonehearth:territory": {
           "marks_territory": true
        }
     }
  }

注意 "mixins" 节点的数组括号。第一个 mixin 指定为路径，其他的指定为别名。

关于相对路径

观察上面示例中的 mixin "file(cobblestone_fence_gate_ghost.json)"。由于 cobblestone_fence_gate_ghost.json 文件与 cobblestone_fence_gate.json（引用它的文件）在同一文件夹中，"file()" 将在同一文件夹中搜索 cobblestone_fence_gate_ghost.json：

如果它在另一个文件夹中，我们将使用 "file(other_folder/cobblestone_fence_gate_ghost.json)"。例如，如果像这张图片中那样，我们会说 "file(mixins/cobblestone_fence_gate_ghost.json)"：

如果它在 cobblestone_fence_gate.json 所在文件夹的外部，我们将使用 "file(../cobblestone_fence_gate_ghost.json)"：

如果它在外部的一个文件夹中，我们将使用 "file(../other_folder/cobblestone_fence_gate_ghost.json)"。在这张图片的情况下，它将是 "file(../mixins/cobblestone_fence_gate_ghost.json)"：

正如你所见，对于我们从包含路径引用的文件的文件夹向上的每个目录，我们在路径开头附加一个 ../。

关于绝对路径

当我们没有某个资源的别名时，使用 "file()" 的替代方法是使用"绝对"路径，它将从模组的命名空间开始，并从清单所在的位置开始遍历子目录。如下所示："/stonehearth/entities/construction/cobblestone_fence_gate/cobblestone_fence_gate_ghost.json"
