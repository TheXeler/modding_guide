入门模组示例

模组制作的第一步

我们有几个入门模组可供开始制作 Stonehearth 模组。建议从制作小模组（或修改现有模组）开始，以熟悉结构和模组 API，然后如果你想尝试 Lua 文件，再尝试更复杂的模组制作。

阅读基础指南后，你应该能够从头创建自己的模组，但让我们先从修改现有模组开始以熟悉文件。我们将使用一个名为 startermod_basic 的示例模组，其清单文件比 stonehearth 模组的清单文件简单得多：

  1. 从 Github 下载 startermod_basic 模组。
  2. 将其解压到你的 stonehearth 模组目录中，与 stonehearth.smod 和 radiant.smod 并列。
    某些压缩软件可能会创建一个同名的内部子文件夹。确保解压模组后，模组目录的结构为 mods/startermod_basic/< 模组文件 >，而不是 mods/startermod_basic/startermod_basic/< 模组文件 >。
  3. 在 startermod_basic 文件夹内，你应该看到几个文件夹和一些文件，其中一个名为 manifest.json。
  4. 启动游戏。在主菜单中，点击模组按钮。你应该在本地模组列表中看到 startermod_basic 模组并已启用。当我们在这里时，如果 debugtools 模组未启用，请启用它。

该模组向世界添加了两个新对象，并为木匠添加了一个新配方，以便你至少可以看到世界中的一个对象。

在深入探讨模组的文件之前，开始一个新游戏并制作一个木匠。你应该在他的配方列表中的"家具"类别下看到一个"Sample Recipe"（示例配方）。示例配方创建一个你可以放置在世界中的装饰对象。

startermod_basic 还向世界添加了一个"Sample Object"（示例对象）。这个对象非常基础，在游戏中几乎没有用途。为了甚至看到它（因为它不参与任何配方），你必须打开物品生成器调试工具，并输入：

  startermod_basic:sample_object

按 Enter 键，然后点击游戏中的地面某处：

现在你已经看到了 startermod_basic 向游戏添加的内容，你现在可以使用此模组向 Stonehearth 添加你自己的对象了。

最简单的方法是用你自己的模型替换示例对象的模型文件（startermod_basic/entities/sample_object/sample.qb）。（要创建 stonehearth 模型，你需要一个可以保存 .qb 文件格式的体素编辑程序。前往指南的工具或美术部分了解如何操作）。

一旦你将新模型命名为 sample.qb 并通过覆盖现有的 sample.qb 文件将其复制到文件夹中，重新启动 Stonehearth 并使用物品生成器在游戏中查看你的模型。

如果你用木匠制作的装饰对象进行此模型替换，他将制作你的装饰对象，而不是默认对象。

你可以通过复制 sample_object 和装饰对象的示例并修改文件来向模组添加其他对象。尝试这些练习：

  • 在 startermod_basic 模组中创建一个全新的对象，并使用 debugtools 物品生成器将它和 sample_object 放置到世界中。
  • 给你的木匠一个额外的配方，使他能够创建你的对象。

如果你已经阅读了基础指南，你现在应该能够弄清楚这些练习。这个模组很小，所以你不会因为太多文件而感到不知所措。以防万一，我们将在模组制作指南的其他部分介绍如何创建物品和配方。
