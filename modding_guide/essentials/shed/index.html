什么是 SHED？

SHED 是 Stonehearth Editor 的缩写。它使用 WinForms 以 C# 编写，尽管其中一些编辑器是使用 CEF 制作的，并且它还使用一些 Python 脚本。

设置 SHED

我们现在提供与游戏捆绑的 SHED 版本。你可以在 Stonehearth 安装文件夹中找到它：

  • 如果使用 Humble Bundle 安装程序安装游戏，通常位于 C:\Program Files (x86)\Stonehearth\shed\。
  • 如果通过 Steam 安装，则 mods 文件夹位于 C:\Program Files (x86)\Steam\steamApps\common\Stonehearth\shed\。

或者，如果你想从我们的公共仓库下载并编译最新的 SHED 版本，请按照以下步骤操作：

  1. 获取 Visual Studio 版本（最低 VS 2014）。如果你拥有流行的游戏引擎之一，你可能已经安装了免费的 Visual Studio Community 版本。否则你可以在这里找到它。 这个 IDE 下载和安装需要很长时间，所以请确保尽管有 bug 你也想使用 SHED（你总是可以使用任何文本编辑器制作模组）。
  2. 从这里下载 SHED 代码。 点击 "Clone or Download" 按钮并将其保存到你计算机上的某个文件夹中。
  3. 打开 Visual Studio 并转到 File->Open->Project/Solution。在你下载代码的位置找到 StonehearthEditor.sln 文件。
  4. 选择 Debug 或 Release 以及 x86 或 64 位（取决于你计算机的架构）。点击 Start 按钮或 Build->Build Solution。
  5. 编译完成后，你可以使用 Start 按钮从 VS 运行它，或在相应文件夹中找到 exe 文件。

额外设置

如果你想使用自动本地化功能（它会自动为你的模组准备翻译文件，以便可以翻译成其他语言），你必须在计算机上安装 Python 2.7。确保是该版本而不是 Python 3。 然后确保你的环境变量中有 Python 路径，以便你的计算机知道在哪里找到它。

首次运行

在 "shed" 文件夹中找到 StonehearthEditor.exe 并运行它。第一次运行 SHED 时，它会要求你找到模组目录。通常你会选择 Stonehearth 文件夹中的 "mods" 文件夹。

为了能够在 SHED 中查看和创建任何内容，模组必须是普通文件夹，而不是 .smod 或 .zip。如果你的模组文件夹包含 .smod 文件，SHED 将为你提供自动解压它们的选项。完成后，你可能需要点击 "File > Reload" 才能使 SHED 检测到模组。 如果你手动解压模组，请确保结构与我们在游戏中运行未压缩模组时相同，否则 SHED 将不知道如何从 "mods" 文件夹加载它们。

SHED 可能需要一点时间才能出现和/或加载/重新加载所有模组，请耐心等待。

如果你打算将模组上传到 Steam 创意工坊，你将已经从游戏中的模组菜单创建了它们。在这种情况下，在 SHED 完成加载基础模组后，转到 File->Change Mod Directory：

你可以浏览并选择第二个文本框的 steam_uploads 目录，然后点击 'Apply and reload'，这将重新加载模组列表。你的模组将出现在基础模组列表下方，它们不会一起排序。

SHED 期望基础模组始终被加载，并且你也需要它们以便从它们克隆物品到你的模组中，所以确保你选择了正确的 mods 目录。

概览

让我们看看 SHED 的界面。 主菜单中的选项是：

File：

  • New Mod（新建模组）
  • Save All（全部保存）
  • Reload（重新加载）
  • Change Mod Directory（更改模组目录）

View：

  • Net Worth Visualizer（净资产可视化器）

Help：

  • About（关于）

然后我们有不同的选项卡：

  • Manifest（清单）
  • Encounter Designer（遭遇战设计器）
  • Entity Stats Browser（实体统计浏览器）
  • Effects Editor（效果编辑器）
  • Recipe Editor（配方编辑器）

我们将在这里解释一些使用 SHED 的基础知识，具体的教程将放在模组制作指南的相应部分。

SHED 基础

"File > New Mod" 选项会自动为你创建一个模组文件夹和预填充的清单，但它将在基础模组目录中创建（本地模组）。这对不使用 Steam 的用户很有用。

如果你使用此选项，你必须在对话框的文本框中输入模组的命名空间，而不是模组的名称。

对于基本模组制作，我们通常会在 Manifest 选项卡中工作。 当在树视图中选择一个节点时，相应的文件将在中央区域的不同选项卡中打开。 这些是选项卡顶部的按钮：

  • Open File in Text Editor：这将在你默认分配用于打开 JSON 文件的文本编辑器中打开文件。你可以通过在 Windows 资源管理器中右键单击 JSON 文件，点击 "Properties"，点击 "Change..." 按钮并选择你选择的程序来更改默认编辑器。

  • Open Containing Folder：将打开当前文件所在的文件夹。

  • Save File：在 SHED 中对此文件进行更改时，请确保经常使用它。

  • Localize This File：如果不存在，这将创建所需的 locale/en.json 文件，用适当的字符串填充它，并调整你的 JSON 文件以指向它们。有关如何设置文件以使此选项正常工作的说明在这里。

    此选项可能不适用于所有文件，因此有时你必须手动设置文件的本地化。

  • Preview with Mixins：这将打开一个窗口，显示应用所有 mixins 后游戏将如何读取文件。它将显示注释以突出显示来自 mixins 的属性。这些注释是无效的 JSON，所以不要从此窗口复制任何内容。

并非所有这些选项都适用于所有文件，只有适用的选项才会显示。

在 JSON 文件的选项卡顶部，你可能还会看到更多按钮。如果你有一个默认用于打开 QB 文件的应用程序，你可以点击带有 Qubicle 图标的按钮以打开关联的 QB 模型。图标（.png 文件）也是如此，尽管你通常可以在右侧列中看到它们，在依赖关系上方。

对于本地化字符串，你可以 Ctrl+点击它们，将出现一个对话框允许你更改文本。你还可以通过将鼠标悬停在它们上面来知道它们有什么文本，这非常方便。

你可以右键单击左侧树视图中的节点以获取不同的选项。这些选项将根据所选项目的类型而不同。

请记住，克隆可能仍有 bug。只有具有别名的 JSON 和 Lua 文件才会出现在 Manifest 选项卡的树视图中。

当有深色背景的文本时，你可以 Ctrl+点击它，它将在你默认的文件扩展名编辑器中打开引用的文件。这也是 SHED 告诉你它识别文本为对现有文件的引用的一种方式。

如果我们的文件有问题，SHED 也会告诉我们。树视图项目和选项卡中将有一个红色圆圈带叉的图标。

当有错误时，SHED 的状态栏中会有红色文本。双击它，受影响的文件列表将在对话框中显示。点击它们一次将在上方的文本区域中显示更多信息，双击它们也将在 SHED 中打开它们，以便你可以修改它们并修复错误。

如果 SHED 抱怨你的某个文件，但在你修复并保存这些更改后没有刷新错误，请使用 "File > Reload" 选项。

一些变通方法和重要信息

在此页面最后更新时：

  • 不幸的是，配方编辑器只会显示你模组中制作的新工匠的配方。

  • 克隆制作职业不会正确克隆配方索引。你必须单独克隆它并手动编辑其配方。

  • 为了让遭遇战设计器检测你的战役，你必须在模组中创建自己的 "gm:index" 别名（SHED 不检测 mixintos 或 overrides）。

  • 如果你认为文件应该因为你保存了它而更新，但 SHED 似乎没有注意到，请从 File 菜单重新加载。

  • 为了让效果编辑器找到你的效果文件（以及所有派生文件，如 cubemitters 等），你必须像 stonehearth 模组中那样设置它们（"data" 和 "data/horde" 下的相同目录）。

  • 请记住，在 SHED 打开并列出文件时，在 SHED 外部大幅编辑文件可能会导致其崩溃，特别是如果你做的是删除文件或移动它们。

一些故障排除提示

  • 如果 SHED 变得无响应，不要点击它。让它运行一会儿。有许多操作会导致它重新加载大量文件，并且它没有为此优化。所以很多时候它需要一些时间来加载或刷新。

  • 如果除了 SHED 变得无响应之外还弹出错误，你可以尝试点击 "Continue"。但如果它仍然感觉不稳定，请尝试重新启动它。最坏的情况：如果你无法关闭 SHED，打开任务管理器，右键单击 "StonehearthEditor" 进程并点击 "End Task"。

  • 如果效果编辑器看起来奇怪，并且你无法使用右侧的 CEF 编辑器因为它出现故障，请确保你的显示器显示设置的缩放比例不高于或低于 100%。将其设置为 100% 并重新启动 SHED 应该可以消除这种错误并让你舒适地使用效果编辑器。
