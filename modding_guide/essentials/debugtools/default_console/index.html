默认控制台

Stonehearth 也带有一个内置控制台，但它只能执行少数预定义的命令。这与后来添加的 debugtools 中的 Lua 控制台不同。默认控制台出现在游戏内屏幕的顶部。

按 Ctrl+C 打开控制台。按 Ctrl+C 或 Esc 关闭它。你可以通过拖动栏底部右侧上方的控件来使控制台变高/变矮。

如果我们启用 debugtools 模组，它将有许多更高级的命令，这些命令非常有用（例如，promote_to 命令）。 输入 help 并按 Enter 获取可用命令列表。

其中一些高级命令可以通过按住 Shift 并点击实体来更快地执行。将出现一个小窗口，其中包含适用命令的列表。点击其中一个命令以执行它。

以下是启用 debugtools 后默认控制台的所有命令。请注意，其中一些命令可能不再有效，或者存在更快的命令来执行相同的操作。让我们将它们分组以便于阅读：

加速测试的命令：

许多这些命令要求我们在执行它们之前选择一个实体。其中一些命令有参数，如别名/URI 或数字。

  • add_citizen - 向你的城镇添加一个新的 Hearthling。可选地指定一个职业（必须为模组指定完整别名）。用法：add_citizen shepherd

  • promote_to - 立即将所选 Hearthling 提升到指定的职业（必须为模组指定完整别名，即 fancy_mod:jobs:fancy_chef）。用法：promote_to footman

  • add_exp - 向当前所选实体的职业添加经验值。如果未给出经验值数量，将升级到下一级。用法：add_exp 1000

  • add_gold - 向当前玩家的库存添加金币。它将在城镇旗帜处生成金币箱。负值将减去金币。用法：add_gold 1000

  • teleport - 传送所选实体，或传入的实体 ID。将在光标处显示一个旗帜以选择位置。用法：teleport 12345

  • destroy - 销毁一个实体。参数 0 是实体的 ID。如果未提供参数，则销毁当前所选实体。用法：destroy 12345

  • destroy_immediately - 立即销毁一个实体。可能不会运行通常在有人被杀时运行的其他代码，如掉落战利品等。参数 0 是实体的 ID。如果未提供参数，则销毁当前所选实体。用法：destroy_immediately 12345

  • ib - 立即建造所选建筑，或参数 0。用法：ib object://game/12345

  • im - 立即挖掘所选挖掘区域，或参数 0。用法：im object://game/12345

  • add_buff - 将指定的增益效果 URI 添加到当前所选实体。用法：add_buff stonehearth:buffs:starving

  • remove_buff - 从当前所选实体中移除指定的增益效果 URI。用法：remove_buff stonehearth:buffs:starving

  • add_trait - 向当前所选实体添加特性。用法：add_trait stonehearth:traits:night_owl

  • remove_trait - 从当前所选实体中移除特性。用法：remove_trait stonehearth:traits:night_owl

  • fill_storage - 用指定的 URI 填充所选储物空间。如果未指定 URI，则默认为橡木原木。用法：fill_storage stonehearth:resources:wood:oak_log

  • grow - 告诉所选实体生长（农场作物或动物）。例如：让小羊长成绵羊。用法：grow

  • decay - 使食物立即腐烂。用法：decay

  • renew - 告诉所选实体更新其资源。例如：让绵羊再次长出羊毛或让枯竭的丝草生长。用法：renew

  • reproduce - 选择一个牧场并强制该牧场繁殖动物。用法：reproduce

  • increase_city_tier - 将定居点的城市等级提高 1 级

  • set - 将所选实体的属性设置为指定值。用法：set spirit 6

  • set_attr - 将所选实体的属性设置为指定值。用法：set_attr health 10

  • set_blink - 让你的 Hearthlings 移动得更快一点...（将使它们跳过"走到这个位置"的行为，立即传送到他们认为应该去的地方）

  • set_health - 将所选实体的生命值设置为指定值。用法：set_health 10

  • set_resource - 将所选实体的消耗性资源设置为指定值。用法：set_resource health 10

  • set_happiness - 设置当前所选实体的当前幸福感。

  • set_time - 将游戏时间设置为传入的时间。用法：set_time 1:25PM

  • set_amenity - 更改玩家与所选实体派系的好感度/关系（友好、中立或敌对）。用法：set_amenity hostile

  • make_friendly - 使玩家与所选实体的派系友好

  • make_hostile - 使玩家与所选实体的派系敌对

  • make_neutral - 使玩家与所选实体的派系中立

  • make_hungry - 如果实体具有卡路里属性，则使所选实体饥饿。如果实体有卡路里观察者，实体将尝试进食。用法：make_hungry

  • make_full - 如果实体具有卡路里资源，则使所选实体饱腹。如果未选择实体，则所有人都饱腹。用法：make_full

  • make_sleepy - 如果实体具有困倦属性，则使所选实体极度困倦。如果实体有困倦观察者，实体将尝试睡觉。用法：make_sleepy

  • make_hurt - 使所选实体生命值低。用法：make_hurt

  • make_lonely - 使所选实体的社交满意度低。用法：make_lonely

  • add_thought - 将指定的想法键（格式为 mod_namespace:thoughts:category:..）添加到当前所选实体。用法：add_thought stonehearth:thoughts:hunger:hungry

  • remove_thought - 从当前所选实体中移除指定的想法键（格式为 mod_name:thought_type:category:name）。用法：remove_thought stonehearth:thoughts:hunger:hungry

  • unlock - 手动解锁给定职业的给定作物或配方。示例用法：unlock corn stonehearth:jobs:farmer

  • release - 释放诱捕陷阱中的宠物

  • get_score - 获取城镇指定分数类型的分数。用法：get_score military_strength

  • change_score - 将所选实体的指定分数更改指定数量。用法：change_score nutrition -10

  • reset_scores - 将所选实体的所有分数重置为其起始值。用法：reset_scores

  • show_animation_text - 启用以在 Hearthling 头顶显示文本，显示他们当前正在播放的动画。需要重新启动/重新加载游戏才能生效。用法：show_animation_text true/false

  • show_untranslated - 用于显示带有 *** 包围的未翻译字符串。用法：show_untranslated true/false

  • spawn_effect - 在所选实体上生成效果，可以选择无限循环效果。可以在生成循环之间添加延迟（以毫秒为单位）。用法：spawn_effect /stonehearth/data/effects/level_up true 1000

  • spawn_encounter - 生成指定的遭遇战。警告：仅用于测试目的，如果在使用命令后保存游戏，可能会导致持久问题。参数应为：战役、遭遇战、弧线。用法：spawn_encounter ambient_threats create_necromancer_crypt trigger

  • add_conversation_subject - 向实体的社交组件中添加对话主题，可选情感。用法：add_conversation_subject goblins --sentiment=1

  • add_journal - 强制为所选 Hearthling 添加日记条目。用法：add_journal dreams

解决错误的命令：

这些命令可以在遇到某些类型的错误时使用，以帮助修复游戏状态。作为模组制作者，我们不经常使用这些命令。

  • ai_reconsider - 调用所选实体的重新考虑

  • destroy_npc_stockpiles - 销毁 NPC 玩家（参数 0）的储藏室。如果未提供参数，则销毁所有 NPC 的储藏室。用法：destroy_npc_stockpiles goblins

  • destroy_scaffolding - 销毁所有脚手架

  • dump_backpack - 立即将所选 Hearthling 背包中的所有物品掉落到地面上。用法：dump_backpack

  • dump_inaccessible_items - 丢弃城镇库存中不在世界中或不在储物空间中的所有物品。这些物品将被丢弃在城镇旗帜周围。

  • fixup_components - 如果此实体缺少任何组件，我们将尝试添加它们

  • reset - 将实体的位置重置为地面上的适当位置。用法：reset

  • reset_location - 将实体的位置重置为地面上的适当位置。还可以传入新位置。用法：reset_location {可选 x y z}

  • select - 选择 ID = 参数 0 的实体。用法：select 12345

  • select_storage - 选择包含实体的储物空间（如果有的话）。

其他通用命令：

  • dump_building - 将所选建筑转储到 qubicle 文件。

  • get_game_mode - 显示当前游戏的游戏模式。通常是 peaceful 或 normal

  • set_game_speed - 将游戏速度设置为默认游戏速度乘以指定数量。用法：set_game_speed 5

  • world_seed - 返回当前世界的世界种子。用法：world_seed

  • help - 打印出所有注册的控制台命令。

  • now - 返回 radiant.gamestate.now()

  • get_config - 从 user_settings.config 获取配置值。用法：get_config foo.bar.baz

  • set_config - 设置指定的配置值。用法：set_config foo.bar.baz {value = 1}

  • get_entity_info - 返回有关实体的信息，即使实体已被销毁。传入实体的 ID。用法：get_entity_info 82215

  • get_global_vision - 返回调用者人口全局视野中的所有对象列表

  • get_cost - 获取所选建筑的成本，或参数 0。用法：get_cost object://game/12345

  • get_conversation_actives - 获取实体的所有活动对话主题。

  • get_conversation_subject - 获取实体的社交组件中的对话主题，可选情感。用法：get_conversation_subject goblins

  • get_current_interaction - 获取所选实体的当前交互

用于调试各种内容的开发者命令：

作为模组制作者，我们不太可能使用这些命令，但这里列出了以防我们需要它们。

  • call - 调用任何已注册路由的通用处理程序。

  • collect_cpu_profile - 收集指定持续时间（以毫秒为单位）的 LUA 代码分析。默认为 30 秒。用法：collect_cpu_profile 150000

  • destroy_all - 销毁与所选实体具有相同 URI 的所有实体。注意：必须先加载实体跟踪器。

  • dump_cache_stats - 显示空间缓存统计信息。

  • dump_lua_heaps - 转储 lua 堆（确保在 user_settings 中将 lua.enable_memory_profiler 设置为 true。）

  • dump_memory_stats - 转储所有跟踪的客户端和服务器内存表

  • dump_trace_stats - 转储所有跟踪的客户端和服务器内存表

  • load_entity_tracker - 告诉 debugtools 实体跟踪器加载所有实体。然后你可以在输入 'debugtools' 后在对象浏览器中检查它们

  • navgrid_viz - 启用不同的导航网格可视化。用法：navgrid_viz [navgrid | water_tight | topology | none]

  • print_ai_stack - 打印 AI 的当前协程堆栈跟踪

  • query_pf - 运行查询寻路器命令。无参数。

  • show_item_ids - 使物品调色板在其物品列表中显示所有物品 ID 的列表。用法：show_item_ids

  • step_plan - 逐步执行当前所选建筑的计划

  • toggle_profile_long_ticks - 启用/禁用每个游戏刻的 LUA 代码分析（每当 lua 评估花费超过一个游戏刻时记录。）

已弃用的命令：

  • hotload_manifest - 热加载清单。（不应再使用此命令。有关更多信息，请参阅热加载清单部分）。

  • hot_reload - 清除 json 缓存，以便客户端上更改的 json 文件将重新加载。对于测试粒子效果很有用。（将此添加到你的 user_settings.json 中与 "user_id" 相同的级别以永久应用此设置："enable_renderer_file_watcher" : true）。用法：hot_reload
