当今模组的编写方式

首先，简单解释一下在早期 Alpha 版本中模组制作者是如何修改游戏的。
Stonehearth 游戏内容存储在安装目录中的 "stonehearth.smod" 文件中。这只是一个重命名的 zip 压缩文件。你可以解压它，查看其中的所有图像、模型和数据文件。要模组化游戏，模组制作者会解压该文件，找到特定的游戏资源，并用他们自己的版本覆盖它。这种方法可行，但存在很多问题...

它需要修改游戏安装程序拥有的文件。如果你安装了更新版本的游戏，它会清除模组所做的更改。 管理多个试图覆盖 Stonehearth 模组不同部分的模组很困难。 多个模组无法对同一个游戏对象进行更改。它们都试图覆盖同一个文件，只能有一个成功。 因此，以下是我们解决当前状况的计划。

模组的结构

一个 Stonehearth 模组包含：

  1. 包含模组文件的目录
  2. 目录中描述模组的 manifest.json 文件

Stonehearth 模组只是位于 Stonehearth 安装目录中 "mods" 目录下的文件或目录（来自 Steam 创意工坊的模组位于不同的目录）。如果游戏看到一个目录或 .smod 文件（这只是重命名的目录 zip 压缩包！），它会在游戏启动时尝试将其作为模组加载。如果文件和目录具有相同的名称（如 /foo 与 foo.smod），游戏会选择加载目录而不是 smod 文件。

如何开始模组制作

要开始制作 Stonehearth 模组，首先需要了解 Stonehearth 模组本身的结构。 让我们前往 Stonehearth 的安装文件夹：

  • 如果使用 Humble Bundle 安装程序安装，通常位于 C:\Program Files (x86)\Stonehearth。
  • 如果通过 Steam 安装，则文件夹位于 C:\Program Files (x86)\Steam\SteamApps\common\Stonehearth。

在此文件夹内，你会看到一个名为 mods 的文件夹。打开它，你将看到构成游戏的主要模组。
这些模组是运行游戏所必需的，如果直接修改它们的文件，可能会出现故障和错误。

为了更方便地制作模组，请在 Windows 资源管理器中显示文件扩展名。
对于 Windows 7，打开控制面板并点击"外观和个性化"。现在，点击"文件资源管理器选项"（也可能显示为"文件夹选项"），然后转到"查看"选项卡。在此选项卡的"高级设置"下，你会看到"隐藏已知文件类型的扩展名"选项。取消选中此选项，然后点击"应用"和"确定"。
对于 Windows 8/8.1/10，打开任意文件夹，选择顶部的"查看"选项卡，然后勾选"文件扩展名"。

现在，选择 stonehearth.smod 文件，并将其重命名，将扩展名从 .smod 改为 .zip。

然后，使用 Windows 内置选项解压 stonehearth.zip（右键单击文件，选择"解压到此处"），或使用你喜欢的任何压缩软件（7-Zip、Winrar 等）。解压所有文件可能需要一些时间。

确保解压后，你有一个名为 stonehearth 的文件夹，其中包含游戏文件。某些解压方法可能会将 stonehearth 文件夹放在另一个名为 stonehearth 的文件夹内；在这种情况下，将其剪切并粘贴到上一级目录，以便合并内容。
如果文件夹不在该路径中，游戏将无法启动。

为了方便引用，大多数模组制作者使用与 stonehearth 模组相同的文件夹结构，但这不是必需的。正如你所见，文件夹名称非常具有描述性，尽管有时如果你不确定要找什么（实体？组件？服务？），可能会有点难找。

大多数常规模组制作是通过将文件从 Stonehearth 模组复制粘贴到你自己的模组中，然后修改它们来完成的。

因此，在模组制作期间保持此文件夹解压状态很有用，但请记住在下载游戏新更新时删除它，否则你会遇到冲突，并且在更新后开始新游戏时可能会出现错误。

或者，你可以将 stonehearth.smod 复制到 mods 文件夹外的另一个目录，重命名并解压以供参考。

开发者为新模组制作者发布了一些示例模组。你可以下载它们并开始尝试，而不会被 stonehearth 模组中海量的文件所压倒。本指南中有一个专门章节介绍它们。

最后说明

你可以使用任何游戏文件作为模组的参考。

模组制作指南包含了游戏大多数系统的说明，但仍有许多文件未在此讨论。请始终检查游戏文件以获取参考。
