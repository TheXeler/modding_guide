JSON 简介

在 Stonehearth 中，我们使用 JSON 作为数据文件（那些不是 3D 模型/2D 图像/脚本等的文件）。 JSON 文件是具有特定格式的文本文件，它们是将信息结构化为对象的一种方式。以下是为 Stonehearth 模组编写 JSON 数据的基本规则简要列表。

编写 JSON

JSON 对象通常由键值对或信息列表组成。在某些编程语言中，键值对的集合被称为字典，因为它类似于单词与其定义链接的结构。 由于 JSON 由计算机程序读取，它使用标点符号：如括号和逗号来 delineate 行的结尾。如果没有适当的标点符号，程序将不知道如何从 JSON 文件中读取数据。

对象

JSON 中的基本标点符号是花括号。 要表示一个对象是 JSON 对象，它必须在花括号之间。因此，我们所有的 JSON 文件都以花括号开始和结束： 

  {

  }

与任何编程语言一样，如果你打开了花括号/圆括号/方括号或类似符号，你必须始终确保关闭它。

字符串

我们要放入此对象的第一件事是键值对。JSON 对象中的键是程序识别稍后将使用的数据的方式。例如：

  {
     "game_name" : "Stonehearth"
  }

在这种情况下，键是 "game_name"。在 JSON 中，键始终是我们称为字符串的东西。字符串是由字母、数字、空格或你在书写时通常使用的任何字符组成的单词或句子。尽管字符串中可以有空格，但在将它们用作键时我们倾向于不喜欢这样，所以如果我们需要分隔几个单词以便于阅读，我们使用下划线代替。对于 Stonehearth，我们也倾向于对所有键使用小写，这样如果我们混合使用大写和小写，可以避免拼写错误。

字符串始终放在引号内（某些编程语言也允许单引号内的字符串，但对于 JSON 我们使用双引号）。 键（"game_name"）与值（"Stonehearth"，也是字符串）用冒号分隔。在冒号周围放置空格并不重要，但如果这样做，文件更容易阅读。

在示例中，我们有一个只有一个键值对的 JSON 对象。当我们的对象在其列表中有多个键值对时，我们需要用逗号分隔每个键值对。这样读取 JSON 的程序就知道在哪里停止读取一行。列表中的最后一项末尾不会有逗号。如下所示：

  {
     "game_name" : "Stonehearth",
     "genre" : "City Builder/Sim/Sandbox"
  }

我们不能在 JSON 对象的同一级别拥有重复的键。键是标识符，所以如果我们在示例中再次写入 "game_name" 而不是 "genre"，即使值不同，读取 JSON 的程序也会难以确定哪个是我们想要分配给该键的正确值。

数字

当我们需要编写不是字符串的值时，我们不需要使用引号。例如，对于数字，我们这样写：

  {
     "num_days" : 7,
     "scale" : 0.5,
     "temperature" : -10
  }

布尔值

在编程语言中，还有一种称为"布尔值"的数据类型，用于表示是/否值（例如，我们可以使用布尔值来表示复选框是否被勾选）。 在 JSON 中，我们也可以通过写入不带引号的 "true" 或 "false" 来表示这些是/否值： 

  {
     "is_enabled" : true,
     "finished" : false
  }

数组

如果我们需要在 JSON 对象中写入事物列表，我们可以使用数组。数组是值的集合，它们没有键，只有值。这些值也用逗号分隔。数组必须以方括号开始和结束。例如：

  {
     "days" : [ "Sunday", "Monday", "Thursday" ],
     "colors" : [
        "red",
        "blue",
        "yellow"
     ],
     "some_numbers" : [ 1, 4.5, 6 ]
  }

JSON 数组可以具有不同类型的值，但通常我们将它们与相同类型一起使用（例如：所有值都是字符串，或数字，或对象等）。正如你所见，只要它们用逗号分隔，将每个值写在不同的行上还是全部写在同一行上都无关紧要。注意数组中的最后一个元素在结束方括号之前没有逗号。

嵌套对象

最后，我们还可以使用其他 JSON 对象作为值。这意味着我们可以嵌套对象和数组，甚至可以制作对象数组。我们还可以将空对象作为值，这有时很有用。例如：

  {
     "raid": {
        "difficulty" : 3,
        "enemies" : [
           {
              "name" : "Evil Zombie",
              "model" : "zombie1.obj",
              "scale" : 1.5
           },
           {
              "name" : "Skeleton (Weak)",
              "model" : "skeleton2.obj",
              "scale" : 2
           }
        ],
        "boss" : {
           "name" : "Titan",
           "attributes" : [
              "speed", "strength", "health"
           ],
           "random_spawn_points" : [
              [0, 1],
              [21, 3],
              [34, 7],
              [50, 10]
           ]
        },
        "is_enabled" : false
     },
     "empty_object" : {}
  }

注意 "enemies" 数组中的每个项目都是一个对象。"random_spawn_points" 中的每个项目本身就是一个数组，我们可以有重复的键（"name"），因为它们位于不同的对象中并且在 JSON 对象中的不同级别。因此，在 Stonehearth 文件中，你将经常看到嵌套数组和嵌套对象。

还有一个称为 "null" 的特殊值，它不带引号写入（如 "true" 和 "false"）。它通常表示值的缺失，与空对象相反（因为你不能有没有值的键）。我们在 JSON 文件中不经常使用 null，所以我们没有将其添加到此列表中。

最后说明

在本模组制作指南中，我们可能会使用不同的词语来指代 JSON 键，例如字段、键、节点、别名、URI、条目或部分，具体取决于上下文。

如果你在模组的 JSON 文件中遇到问题，可以使用名为 JSONLint 的在线工具。你可以复制整个 JSON 文件的内容，粘贴到那里，然后验证它。很多时候 JSON 出错是因为在不应该有逗号的地方有逗号，或者你忘记关闭某些引号，或者在同一级别有重复的键，或者文件中的任何其他不一致之处使 JSON 无效，因此游戏无法读取它。

此外，由于 JSON 文件中键值对的顺序无关紧要，有时你可能会发现很难知道是否与游戏中的另一个文件具有相同的字段。要检查这一点，你可以使用 JSON Diff。这是一个在线工具，即使键值对的顺序不同，它也会显示两个 JSON 对象之间的差异。
