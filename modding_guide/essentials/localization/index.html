本地化

关于本地化

Stonehearth 的用户界面采用 UTF-8 编码，我们使用这个 Javascript 框架进行本地化。

要为你的模组启用本地化功能，你需要在清单文件中添加此键。通常我们将其放在 "info" 部分下方以避免忘记（记得在需要的地方添加逗号！）：

  "default_locale" : "en"

游戏中的本地化工作方式如下：对于我们想要翻译的每个文本，我们在 stonehearth/locales/en.json 文件中创建一个键。然后，在我们的 JSON 文件中，我们通过用 "i18n()" 包裹来引用该键。 例如，这是我们指定 brightbell 窗台花箱名称和描述的方式： 

  "entity_data": {
     "stonehearth:catalog": {
        "display_name": "i18n(stonehearth:entities.decoration.window_box_brightbell.window_box_brightbell_ghost.display_name)",
        "description": "i18n(stonehearth:entities.decoration.window_box_brightbell.window_box_brightbell_ghost.description)",
        "icon": "file(window_box_brightbell.png)",
        "category": "decoration",
        "material_tags": "decoration crafted stockpile_decoration"
     }
  }

注意用点号分隔的键。在 en.json 文件中，我们将有这样的结构：

  {
     "entities" : {
        "decoration": {
           "window_box_brightbell" : {
              "window_box_brightbell_ghost" : {
                 "display_name" : "Brightbell planter",
                 "description" : "A tidy home for the plant."
              }
           }
        }
     }
  }

因此，我们使用 "i18n()"，在其中写入模组的命名空间、一个冒号，然后是我们在语言文件中设置的键层次结构，用点号分隔。当游戏加载此文件时，它将用我们在语言文件中指定的实际翻译替换 i18n() 字符串。

为了快速测试，我们可以编写纯文本而不是 "i18n(...)"，游戏应该能够正常读取它。但稍后我们必须按照上述说明更改它以使其可翻译。

stonehearth 模组中有数千行文本需要翻译，因此我们模仿文件夹结构来组织键。在这种情况下，我们要翻译的文本位于 stonehearth/entities/decoration/window_box_brightbell/window_box_brightbell_ghost.json 中。

当为你自己的模组添加本地化时，你可以为 en.json 文件使用任何你喜欢的节点，你甚至可以将所有键放在同一级别，如下所示：

  {
     "name_1" : "Test",
     "description_1" : "My description",
     "info" : "Another text"
  }

但将它们组织起来很有用，这样你可以在不同节点内拥有重复的键。

在为现有的 en.json 文件制作翻译模组时，你必须使用完全相同的键，然后将文本/值部分翻译成你的语言。

在翻译 stonehearth 模组时，你可能会在本地化字符串中找到特殊文本，如 [str(i18n_data.town_name)]。这些应该保持不变。它们是将在游戏中被替换的变量。例如，[str(i18n_data.town_name)] 代表你的城镇名称。这些变量在触发对话框的 JSON 文件（如战役遭遇战）中声明。这样，我们就可以在可翻译文本中使用由游戏动态生成的专有名词，如 Boss 名称、你的城镇名称（你在开始游戏时自定义的）等。
