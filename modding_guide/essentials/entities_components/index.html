实体与组件

Stonehearth 中的所有事物都由实体和组件构成。

  • 实体
  • 组件
  • 实体数据

实体

实体是世界中存在的对象。你在 Stonehearth 中看到的所有东西（除了用户界面元素）都是实体。例如：树木、 Hearthlings、兔子、原木。

Stonehearth\mods\radiant\modules 中的 entities.lua 文件描述了许多你可以对实体执行的操作。然而，实体本身包含的关于自身的数据非常少。它们几乎将所有行为存储在组件中。

你可以在 mods\stonehearth\entities 目录中找到游戏中的所有实体。有些位于 mods\stonehearth\jobs 目录中（如工作台或不同的职业服装）。

实体由包含描述该实体的 JSON 文本文件表示。此文件描述应将哪些组件添加到实体中。如果实体具有混入（mixins），它可能会继承其他组件等。以下是 small_boulder.json 的摘录：

  {
     "type": "entity",
     "mixins": "stonehearth:mixins:boulder",
     "components": {
        "model_variants": {
           "default": {
              "models": [
                 "file(small_boulder_1.qb)"
              ]
           }
        },
        "stonehearth:resource_node": {
           "durability": 1
        },
        "mob": {
           "model_origin": { "x": -0.5, "y": 0, "z": 0.5 },
           "region_origin": { "x": -0.5, "y": 0, "z": 0.5 },
           "align_to_grid": [
              "x",
              "z"
           ]
        },
     },
     "entity_data": {
        "stonehearth:catalog": {
           "display_name": "i18n(stonehearth:entities.boulders.small_boulder.display_name)",
           "description": "i18n(stonehearth:entities.boulders.small_boulder.description)"
        },
        "stonehearth:appeal": {
           "appeal": -1
        }
     }
  }

这些是我们可以在这些 JSON 文件中找到的不同键或节点：

  • "type" : "entity"
    这指示此 JSON 文件描述的内容。在这种情况下，它是一个实体。其他文件可能包含 "index"、"mixin" 或 "job" 等 "type"。当我们的代码使用此文件时，代码将知道如何解释其余部分。

  • "mixins" : "stonehearth:mixins:boulder"
    此键允许你引用另一个 JSON 文件。来自该其他文件的所有键都被合并到此文件中，重复的键会被合并在一起。因此，如果两个文件都有 "components" 键，最终文件将包含来自两个文件的子组件。有关混入的更多信息，请参阅指南的下一部分。

组件

  • "components" : { ... }
    默认情况下与实体关联的所有组件都在 "components" 键内声明。

以下是来自 arch_backed_chair.json 的简单示例：

  {
     "mixins": "file(arch_backed_chair_ghost.json)",
     "components": {
        "stonehearth:entity_forms": {
           "iconic_form" : "file(arch_backed_chair_iconic.json)",
           "ghost_form" : "file(arch_backed_chair_ghost.json)",
           "placeable_on_ground" : true
        },
        "region_collision_shape": {
           "region": [
              {
                 "min" : { "x" : 0, "y" : 0, "z" : 0 },
                 "max" : { "x" : 1, "y" : 2, "z" : 1 }
              }
           ]
        }
     },
     "entity_data": {
        "stonehearth:net_worth": {
          "value_in_gold": 6,
          "rarity": "common",
          "shop_info": {
             "buyable": true,
             "sellable": true,
             "shopkeeper_level": 1,
             "shopkeeper_type": "caravan"
          }
        }
     }
  }

在这种情况下，"stonehearth:entity_forms" 和 "region_collision_shape" 都是在创建实体时将添加到实体的组件。其他组件将根据 mixin arch_backed_chair_ghost.json 中的内容添加。

当我们从实体中添加/删除功能时，可以从文件中添加/删除不同的组件。编写 Stonehearth 很大程度上是一个定义和实现描述游戏中许多交互的可重用组件的练习。

因此，在考虑你的模组时，想想你想要添加的实体以及属于你想要添加到游戏的每个项目的功能片段。它们可以分解为通用的、可重用的组件吗？每个组件需要什么数据？

组件的名称将根据其模组以及它是用 C++ 还是 Lua 实现而改变。对于模组制作，我们只能用 Lua 创建组件。 如何创建你自己的组件将在指南的高级部分中进行介绍。

实体数据

可以通过其实体 JSON 文件向实体添加实体数据。这是一种将只读、在所有实例中始终为真的数据附加到实体的方法。

  "entity_data" : { ... }

然后可以通过 get_entity_data() 在 Lua 中再次访问这些数据。这允许组件和其他 Lua 脚本使用它：

  foo = radiant.entities.get_entity_data(my_chair, "stonehearth:net_worth").value_in_gold
