组件

正如基础指南中所述，组件是定义实体行为的可重用功能。您可以在 stonehearth/components 下找到 Lua 组件。

设计组件

理想情况下，组件应该是可重用功能的集合。在决定如何构建与实体关联的功能时，请记住以下几点：

  • 这个组件可以变得更通用吗？例如，床和火坑座位都需要预订机制，因此一次只能有一个人使用每件家具。与其创建床预订组件和火坑座位预订组件，不如创建一个在所有可租赁物品之间共享的租赁组件。

  • 这个组件只包含只读数据吗？如果是这样，请考虑将其中一些数据放入"entity_data"而不是"components"中。您可以稍后从组件内部访问该数据，例如：

    local foo = radiant.entities.get_entity_data(chair, "stonehearth:appeal").appeal

添加新组件

所有 Lua 组件都是类，看起来像这样：

  local MyComponent = class()

  -- initialize 函数总是被调用
  function MyComponent:initialize()
    -- json 变量将包含来自实体 .json 文件的关联组件数据
    local json = radiant.entities.get_json(self)
    -- 组件上的保存变量必须在 initialize 函数中声明
    -- 即使是 nil 变量！
    self._sv.first_saved_variable = nil
    self._sv.second_saved_variable = 'test string'
    self._sv._unremoted_save_variable = json.my_json_data_variable
  end

  function MyComponent:getter_or_setter()
  end

  function MyComponent:other_functions()
  end

  return MyComponent

所有组件都有一个 self._entity 字段，它是创建组件的实体。我们可以在它们中使用保存变量和生命周期函数。

您可以通过在实体的 JSON 文件的 "components" 部分中提供组件的名称（以 mod 的命名空间加冒号为前缀）将 Lua 实现的组件添加到 JSON 文件中。例如，这是来自 carpenter_workbench.json 的工作坊组件（在 workshop_component.lua 中实现）：

  "components" : {
     "stonehearth:workshop": {
        "job_alias": "stonehearth:jobs:carpenter"
     }
  }

可以使用调用 radiant.entities.get_json(self) 检索在 JSON 文件中提供给组件的任何数据。

最后，要访问组件，必须在 mod 清单的 "components" 部分（与"aliases"、"mixintos"等同级）中引用它。例如：

  "components" :  {
     "workshop" : "file(components/workshop_component.lua)",
     "crafter"  : "file(components/crafter_component.lua)"
  }

我们还可以创建客户端组件，它们被视为客户端控制器。我们将在清单中的 "client_components" 下声明它们。

使用组件

一旦您有了对实体的引用（通常由实体的创建生成），您就可以通过以下方式访问其组件：

  local comp = entity:get_component('stonehearth:workshop')

对于 Lua 组件，我们传递的名称将是 mod 的命名空间、一个冒号，以及在清单中定义的组件名称（与实体 JSON 文件中的结构相同）。请注意，C++ 组件的使用不带命名空间（例如 mob 和 render_info 组件）。

然后，您可以对该对象上实现的 comp 变量执行任何操作。例如：

  local workbench = comp:get_entity()
  comp:start_crafting_progress(order)

何时使用"get_component"与"add_component"

当您需要测试组件是否存在时使用 get_component()。这通常用于查看实体是否扮演某个角色。例如，如果此返回非 nil 值，则实体用作采矿区：

  entity:get_component('stonehearth:mining_zone')

当组件必须存在并在必要时创建时使用 add_component()：

  entity:add_component('stonehearth:mining_zone'):get_region()

组件渲染器

组件渲染器是您用来创建与组件相关的某种视觉表示的东西。

Stonehearth 实际上并不渲染实体本身，因为实体本身并不是任何东西——它们是组件（数据和功能）的集合。因此，如果您想要显示某些视觉内容，您需要将该对象附加一个（或两个！）渲染器，这意味着需要一个组件渲染器。

（注意：炉民、哥布林、鹿等似乎没有与之关联的渲染器，但实际上它们有——它们在 C++ 中，而不是 Lua 中。它们是通过将这些实体的 render_info 组件添加到它们身上而添加的）。

组件渲染器将在客户端使用拥有实体和组件进行初始化；然后由组件来决定它想要渲染什么以及如何渲染。一个简单的组件渲染器的良好示例可以在 stonehearth/renderers/stockpile/stockpile_renderer.lua 中找到。在那里，我们从库存组件获取有关库存的信息（其大小以及其中的物品），并使用这些信息构建在库存放置模式下可见的 2D 库存区域。

更复杂的示例是 structure_renderer.lua，它处理建筑物的各种 RPG/剖面视图模式，为建筑物的视觉效果及其阴影体积构建多个可渲染事物（因此即使墙壁被剖开的建筑物也能投射适当的阴影）。

组件渲染器在清单的 "component_renderers" 下声明（与"aliases"、"components"等同级）。它们具有与它们渲染的组件相同的名称。例如：

  "components" : {
     "stockpile" : "stockpile": "file(components/stockpile/stockpile_component.lua)"
  },
  "component_renderers" : {
     "stockpile": "file(renderers/stockpile/stockpile_renderer.lua)"
  }

您可能还会在 stonehearth 清单中看到一个名为 "invariant_renderers" 的部分，但 modder 无需担心它。
