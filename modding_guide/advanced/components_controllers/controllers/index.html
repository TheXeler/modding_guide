关于控制器

控制器是一个实用类，用于存储数据/执行功能（并且不是组件/服务/调用处理器等）。组件本身也是控制器，只是它们在游戏启动时的特定时间加载，并且专门绑定到实体。控制器具有一些使初始化/加载/销毁更容易的行为。

控制器示例：

  • 所有场景
  • 库存跟踪器（以及区分每个库存跟踪器功能的实用类）
  • 特性/增益效果脚本
  • 特定职业的功能/逻辑

控制器的结构

控制器是具有用于保存变量的 self._sv 和initialize/restore/etc 函数的类，这些函数保证在保存和加载之间的一些保存状态流控制。

  local MyController = class()

  -- Initialize 在控制器首次创建时调用
  function MyController:initialize()
     -- 注意：所有控制器都自动配备一个
     -- self._sv 变量，您可以将内容保存到其中
     self._sv._entity = nil
     self._sv._uri = nil
  end

  function MyController:create(entity, uri)
     self._sv._entity = entity
     self._sv._uri = uri
  end

  -- 在加载时调用，在 self._sv 中保存的所有数据存储恢复之后
  -- 注意，如果您想确保某些内容已被恢复，请在
  -- 我们初始化自己时将其保存在 self._sv 中
  function MyController:restore()
  end

  function MyController:activate()
     local json = radiant.resources.load_json(self._sv._uri)
     local data = json.data

     local some_component = self._sv._entity:get_component('my_mod:custom_comp')
     some_component:set_data(data)
  end

  function MyController:destroy()
     local some_component = self._sv._entity:get_component('my_mod:custom_comp')
     some_component:reset_data()
  end

  -- 返回控制器以便它可以用作类。
  return MyController

请记住，您可能不需要向控制器添加所有特殊函数，只需要添加其正确行为所需的函数。

使用控制器

假设您有一个如上所述的 controller.lua 文件，您可以在 Lua 代码中的任何位置创建它的实例。您甚至可以将控制器相互传递！

首先，在 mod 清单的"controllers"部分声明控制器（与"aliases"、"mixintos"等同级）。这将 stonehearth 可见的名称映射到控制器文件：

  "controllers" : {
     "my_controller" : "file(services/server/my_controller/my_controller.lua)"
  }

您保存控制器 Lua 文件的文件夹会有所不同。通常在与将使用它的主脚本或 JSON 文件相同的文件夹中，以便所有相关文件放在一起。

然后在其他脚本中，您可以通过调用以下命令创建控制器的实例：

  local controller = radiant.create_controller('my_mod_namespace:my_controller')

您在控制器名称之后传递的任何内容都将作为参数传递给控制器的 create 方法，例如：

  local tracker = radiant.create_controller('stonehearth:inventory_tracker', controller)

最后，您可以在脚本中调用该控制器中的函数：

  local tr_data = tracker:get_tracking_data()

请注意，有时 stonehearth mod 中已经存在将创建和使用您的控制器的代码。一些 JSON 文件引用将为您创建的控制器别名/文件，因此您只需在清单中声明控制器，编写其 Lua 文件并在 JSON 中引用它（例如：组件、特性/增益效果脚本等）。

客户端控制器

清单的 "controllers" 部分用于服务器控制器。大多数控制器都属于此类别。您可以从它们访问服务器端服务，如人口服务。控制器状态会被保存。

客户端控制器通常用于相机、渲染、声音和其他客户端行为等事物。与常规控制器不同，客户端控制器可以访问客户端服务，如 seasons_render 服务或相机服务。它们还可以访问客户端 C++ 命令，如 _radiant.client.get_render_entity。

客户端控制器没有保存状态。它们的 _sv 变量用于远程传输到 UI。

您应该将通过 radiant.create_controller 计划创建的任何控制器添加到清单中。不添加它们可能会导致控制器生命周期方法出现问题。对于客户端控制器，您应该将它们添加到 "client_controllers" 部分下。
