UI

Stonehearth 使用 HTML、Javascript（JQuery）和 CSS（Less）来构建 UI。对于创建 UI 视图，我们使用 Ember 和 Handlebars 模板。游戏使用的 Ember 版本是 1.8，虽然有点过时，但也随着时间的推移针对游戏的特定问题进行了修补，因此升级它并不简单。

您可以在 stonehearth/ui 中找到现有的 UI 视图和所有相关文件。

添加简单的 UI 视图

我们将添加一个小型 UI 视图来解释 mod 中文件的结构。

  1. 在您的 manifest.json 中，添加 "ui" 部分（与"aliases"或"mixintos"同级）。在其中，将 HTML、Javascript 和 Less 文件添加到相应的数组中：

    "ui" : {
   "html" : [
      "file(ui/custom_view/custom_view.html)"
   ],
   "js" : [
      "file(ui/custom_view/custom_ui_view.js)"
   ],
   "less" : [
      "file(ui/custom_view/custom_ui_view.less)"
   ]
}

    仅凭这一点，游戏就已经加载了我们的 UI 文件。但我们需要定义一些 HTML 和 Javascript，以便它们实际上在屏幕上显示内容。

  2. 编辑 HTML 文件。如上所述，Stonehearth 使用带有 Handlebars 模板的 Ember。因此，通常您的 HTML 文件看起来像这样：

    <script type="text/x-handlebars" data-template-name="customView">
</script>

    在 script 标签内，我们将添加我们的 HTML，并且可以使用 Handlebars 表达式：

    <script type="text/x-handlebars" data-template-name="customView">
   <div id="customWindow" class="customWindow">
      <div class="customTitle">
         The title is "{{view.title}}"
      </div>
   </div>
</script>

  3. 编辑 Javascript 文件。在 custom_view.js 中，我们需要定义一个视图。通常，我们扩展 App.View，但您也可以扩展游戏中的任何其他视图（例如，要创建自定义对话框，您可以扩展现有的任何对话框，这样您将继承一些功能）。

    $(document).ready(function() {
   $(top).on('stonehearthReady', function (_, e) {
      var view = App.gameView.addView(App.UIModCustomView);
   });
 });

 App.UIModCustomView = App.View.extend({
    templateName: 'customView',
    modal: false,
    closeOnEsc: true,

    didInsertElement: function() {
       var self = this;
       self.set('title', 'A new beginning');
       // 如果您希望 UI 视图可拖动，请添加此行
       self.$().draggable();
    }
 });

    首先，我们等待游戏加载完成，然后调用 App.gameView.addView 将我们的视图添加到 UI 中。请注意，此时我们的视图是永久可见的，但主菜单、登船屏幕和加载屏幕的图像覆盖了它。

    我们通过扩展 App.View 定义我们的视图，添加 templateName 属性并将其值设置为我们的 Handlebars 模板的名称。

    然后在 didInsertElement 中设置 title 属性，以便在我们示例模板中显示时它具有值。

条件显示视图

我们可以让视图保持隐藏，直到玩家在 UI 中点击某处（例如：我们从单位框架命令触发事件并对其做出反应）。

例如，可以通过点击单位框架命令（触发 radiant_show_workshop 事件）或直接调用 App.workshopManager.toggleWorkshop() 来显示工作坊 UI（参见 stonehearth/ui/game/start_menu/start_menu.js）。

来自 stonehearth/ui/game/show_workshop/show_team_workshop.js：

  $(document).ready(function () {
     App.workshopManager.init();

     // 从工作坊和工匠显示制作 UI。
     $(top).on("radiant_show_workshop", function (_, e) {
        App.workshopManager.toggleWorkshop(e.event_data.crafter_type);
     });
     $(top).on("radiant_show_workshop_from_crafter", function (_, e) {
        App.workshopManager.toggleWorkshop(e.event_data.crafter_type);
     });
  });

添加到现有视图

开始菜单中的所有操作都是从 start_menu.js 执行的。那么，如果我们 mixinto 一个新菜单或子菜单，如何向 JS 文件添加新操作呢？

嗯，Ember 允许重新打开实例以从其他文件修改它。 从我们的一个 .js 文件中，我们会这样做： 

  App.StonehearthStartMenuView.reopen({
     init: function() {
        var self = this;

        self.menuActions.create_custom_zone = function(){
           self.createCustomZone();
        }

        self._super();
     },

     createCustomZone : function() {
        var self = this;
        // 做一些事情
     }
  });

覆盖 UI 文件

有时我们确实需要覆盖基础游戏中的 Javascript 文件。在这种情况下，我们将在清单的 "overrides" 部分中使用覆盖。

请小心，因为它可能会破坏与其他 mod 的兼容性。

更多关于 UI

使用 Ember，我们可以观察属性的变化并根据它们自动更新 UI。

在 stonehearth/ui/game/character_sheets/pet_character_sheet/pet_character_sheet.js 中可以看到一个简短的示例。此视图在 stonehearth_client.js 中创建，我们向其传递宠物的引用：

  _petCharacterSheet: null,
  showPetCharacterSheet: function(entity) {
     if (this._characterSheet != null && !this._characterSheet.isDestroyed) {
        this._characterSheet.destroy();
        this._characterSheet = null;
     }

     if (this._petCharacterSheet != null && !this._petCharacterSheet.isDestroyed) {
        this._petCharacterSheet.set('uri', entity);
     } else {
        this._petCharacterSheet = App.gameView.addView(App.StonehearthPetCharacterSheetView, { uri: entity });
     }
  },

上面的函数从 stonehearth/ui/unit_frame/unit_frame.js 调用：

  this.$('#nametag').click(function() {
     if ($(this).hasClass('clickable')) {
        var isPet = self.get('model.stonehearth:pet');
        if (isPet) {
           App.stonehearthClient.showPetCharacterSheet(self.get('uri'));
        }
     }
  });

单位框架监听 radiant_selection_changed 事件（它从 C++ 端触发并返回选定的实体）：

  var updateUnitFrame = function(data) {
     if (!App.gameView) {
        return;
     }
     let unitFrame = App.gameView.getView(App.StonehearthUnitFrameView);
     if (unitFrame) {
        unitFrame.set('uri', data.selected_entity);
     }
  };

  $(document).ready(function(){
     $(top).on("radiant_selection_changed.unit_frame", function (_, data) {
        updateUnitFrame(data);
     });
     $(top).on("radiant_toggle_lock", function(_, data) {
        if (!App.gameView) {
           return;
        }
        radiant.call('stonehearth:toggle_lock', data.entity);
     });
  });

然后将 'uri' 属性设置为选定的实体，以便我们在其他视图中使用。因此，我们可以将 'uri' 属性传递给 pet_character_sheet.js 并根据它更新 Handlebars 模板中的变量：

  _updateNameAndDescription: function() {
     var alias = this.get('model.uri');

     var description = this.get('model.stonehearth:unit_info.description');
     var display_name = this.get('model.stonehearth:unit_info.display_name');
     if (alias) {
        var catalogData = App.catalog.getCatalogData(alias);
        if (!catalogData) {
           console.log("no catalog data found for " + alias);
        } else {
           if (!display_name) {
              display_name = catalogData.display_name;
           }

           if (!description) {
              description = catalogData.description;
           }
        }
     }
     if (display_name) {
        var unit_name = i18n.t(display_name, {self: this.get('model')});
        this.set('unit_name', unit_name);
     }
     this.set('description', description);
  }.observes('model.uri'),

我们还可以定义一个 "components" 变量，列出我们将在宠物中找到的组件：

  components: {
     'stonehearth:unit_info': {},
     'stonehearth:attributes' : {},
     'stonehearth:expendable_resources' : {},
     'stonehearth:pet' : {

  }

并观察它们的变化以更新我们的 UI 视图的值：

关于本地化字符串中的变量

如之前所述，如果 UI 字符串需要变量，请使用 [str(my_var_path.my_variable_name)]：

  "bulletin_title": "Hello [str(town_name)]!"

然后，当您显示公告时，您需要提供一个包含 town_name 的变量。Javascript 代码示例：

  i18n.t("stonehearth:bulletin_title", {town_name: App.stonehearthClient.settlementName()}).

在 Lua 中，我们也可以在更新 UI 文本时传递变量：

  ai:set_status_text_key('stonehearth:ai.actions.status_text.chase_entity', { target = args.target })

所有字符串都在 UI（Javascript）中显示和本地化，因此您需要知道显示时字符串需要哪些变量，并在调用 i18n 时将那些变量传递到您的字符串中。

HTML 文件示例——这是 unit_frame.html 显示实体名称的方式：

  {{i18n_key view.display_name self=view.model}}

i18n_key 是一个新的 Ember 助手，它将第一个参数解析为字符串键（因此它知道第一个参数不是平铺字符串，而是变量），所有通过 param_name=param_value 传递的可选参数都是字符串将使用的变量。

在这种情况下，要显示单位的名称，我们传入 self=this，这表示参数"self"是我们正在追踪的实体。这样，我们可以使单位的 display_name 看起来像："Chieftan [str(self.stonehearth:unit_info.custom_name)]"，这将显示实体的随机生成的自定义名称，周围有 Chieftan 头衔。如果实体可能死亡（例如：用于 boss 死亡对话框中的 boss 名称），请使用缓存的显示名称来显示实体的名称。

要在 Javascript 中传入 self 参数，请执行：

  i18n.t(this.get('unit_info.display_name'), {self: this});

如果您的字符串需要需要本地化的子字符串，请在 en.json 中使用 i18n(stonehearth:my.loc.var.path)：

  "bulletin_message": "I would like to give you a i18n(stonehearth:entities.food.corn.corn_basket.display_name)!"
