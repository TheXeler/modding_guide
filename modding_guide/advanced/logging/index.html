向 user_settings.json 添加自定义标志

我们可以在 Lua 脚本中向 user_settings.json 文件添加自定义配置变量，无论我们是否在游戏中添加 UI 选项来更改它们。

它们将出现在 user_settings 文件的 "mods" 部分的一个键内（该键将是我们的 mod 的命名空间）。

如果需要保存值，我们可以以编程方式添加它们，但如果是为了调试目的，我们可以简单地在用户设置中手动写入值以启用/禁用标志。

  -- 在用户设置中保存 'max_quantity' 属性，值为 5
  radiant.util.set_config('max_quantity', 5)

要从我们的代码中检索值，我们会这样做：

  local max_qty = radiant.util.get_config('max_quantity', 1)

该值将从脚本所属的 mod 中读取。如果游戏找不到配置字符串，将返回默认值（我们传递的第二个参数，通常是数字、字符串或布尔值）。

如果我们需要从我们的 mod 中设置或获取不同 mod 的配置值，我们可以使用这些：

  radiant.util.set_global_config('mods.other_mod.config_variable', 'custom_value')
  local auto_loot_enabled = radiant.util.get_global_config('mods.stonehearth.auto_loot', false)

要从 Javascript 文件中执行此操作，请查看 stonehearth/ui/shell/settings/settings.js：

  _updateGameplayTabPage: function() {
     var self = this;
     radiant.call('radiant:get_config', 'mods.stonehearth')
        .done(function (response) {
           var stonehearthOptions = response['mods.stonehearth'] || {};
           if (stonehearthOptions) {
              self.set('showHearthlingPaths', stonehearthOptions.show_hearthling_paths);
              // 更多代码...
           }
        })
  }

如何从代码中记录日志

有几种方法可以从我们 mod 的 Lua 文件中在 stonehearth.log 文件中打印文本。

首先，在 Lua 文件的顶部，创建一个记录器并为其赋予一个唯一标签：

  local log = radiant.log.create_logger('tag_name')

要在 Lua 函数中写入日志，请在记录器上调用方法：

  log:info('hello world')

我们有多个日志级别。以下是它们：

  ALWAYS = 0,
  ERROR = 1,
  WARNING = 3,
  INFO = 5,
  DEBUG = 7,
  DETAIL = 8,
  SPAM = 9

所有这些在记录器上都有相应的方法：

  log:detail('really fine grained logging information')
  log:error('OMG CRAZY IMPORTANT ERROR!')

您可能已经自己弄清楚了大部分内容。这里是秘诀。

要在日志中实际显示内容，请在您的 user_settings.json 中添加一个日志记录块（与您的 user_id 同级）：

  "logging" : {
     "show_console" : true,
     "mods" : {
        "your_mod_namespace" : {
           "log_level" : 1,
           "tag_name" : 5
        }
     }
  },

show_console 标志将在您启动游戏时在运行时弹出一个显示 stonehearth.log 的窗口。

然后我们有一个 "mods" 部分，按命名空间列出不同的 mod。在其中我们有不同的日志标签，以及可选的 log_level 标志，这是应用于整个 mod 的默认日志级别。对于每个标签，我们可以定义我们要使用的日志级别。

这个想法是，如果您的 mod 有几个不同的逻辑功能，您为每个功能初始化一个单独的记录器，带有不同的标签。然后，当您调试功能 A 与功能 B 时，您可以调整用户设置中的日志级别，这样您的日志就不会被您不关心的信息淹没。

示例：

这将仅记录使用标签"feature_a"创建的记录器中的 log:error('...') 条目：

  "mods" : {
     "your_mod_namespace" : {
        "feature_a" : 1
     }
  }

这将记录使用"feature_b"标签创建的记录器中的所有日志条目：

  "mods" : {
     "your_mod_namespace" : {
        "feature_b" : 9
     }
  }

当然，这也有效：

  "mods" : {
     "your_mod_namespace" : {
        "feature_a" : 1
        "feature_b" : 9
     }
  }

对于快速打印到您知道在发布 mod 之前会删除的 stonehearth.log 的文本，您还可以使用 print() 函数：

  print('debugging stuff')
  printf('debugging variables %d and %s', num_times, my_string)

来自 print() 函数的这些调试文本将在 stonehearth.log 中以前缀'DEBUG'标签显示，因此您应该能够快速找到它们。
