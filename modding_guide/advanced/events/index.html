什么是事件？

事件本质上是广播给游戏中所有 Lua 类的消息。当游戏中发生关键事情时，它们会被触发。mod 可以监听游戏事件，以便在适当的时刻执行行为。

事件有一个目标对象和一个目标标识符键，因为某个类的所有项目可能都希望监听与该类相关的事件，但只实际响应涉及它们自己或其子项的事件。

您可以监听来自原版 mod 触发的事件，或者从您的代码中触发自定义事件并监听它们。

触发事件

要触发事件，请调用 radiant.events.trigger_async() 并传入一个标识对象（通常是事件所关注的实体；如果是通用事件，有时是与该事件相关的服务）、事件的名称（命名空间到相关的 mod）和一组可选参数：

  radiant.events.trigger_async(entity, 'stonehearth:admiring_fire', {})

请注意，上述函数异步触发事件。因此，当您捕获事件时，您应该确认世界状态。例如，您不能假设在触发事件时存在的对象仍然存在于世界中。还有 radiant.events.trigger_sync()，但异步触发事件对性能更好。

监听事件

如果一个类希望接收事件，它会注册以监听标识对象/字符串键对，并指定在触发事件时要调用的函数。

要监听事件，请调用 radiant.events.listen() 并传入一个标识对象（应与传递给触发器的对象匹配）、事件的名称，以及一个应接收参数并在触发事件时被调用的函数。例如：

  radiant.events.listen(entity, 'stonehearth:buff_added', function(e)
        --e 包含传递给触发器的参数
        if e.uri == uri then
           -- 做一些事情
        end
     end)

或者您可以传入类中其他地方存在的函数：

  self._attribute_listener = radiant.events.listen(entity, 'stonehearth:foo_event', self, self._on_attribute_changed)

  function ClassName:_on_attribute_changed(e)
     --做一些事情
  end

取消监听事件

当您知道事件监听器不再相关时，请销毁它以节省事件管道上的流量并防止错误。通过您用于保存它的名称来引用它。

如果您不再想监听事件，您可以在回调函数中返回 radiant.events.UNLISTEN，或者您可以保存事件并在以后销毁它：

  • 如果您想取消监听刚刚触发的事件，您的函数应该只返回 radiant.events.UNLISTEN 而不是直接取消监听，以便事件服务可以正确计数事件。从回调中取消监听的示例：

    function ClassName:_on_attribute_changed()
   return radiant.events.UNLISTEN
end

  • 如果具有监听器的对象在即将被销毁时仍在监听事件，请取消监听该事件。此处的示例假定事件已保存到 self._attribute_listener，如上节示例所示：

    if self._attribute_listener then
   self._attribute_listener:destroy()
   self._attribute_listener = nil
end

    这通常会在类的 destroy() 函数中调用。
