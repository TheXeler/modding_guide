如何添加建筑颜色

如何添加建筑图案在这里解释。

向建筑调色板添加更多颜色很容易：

  1. 在您的清单中对"stonehearth:build:brushes"添加 mixinto。

    "mixintos" : {
   "stonehearth:build:brushes" : "file(data/build/building_brushes.json)"
}

  2. 您的 mixinto 文件应仅包含我们要添加的颜色，位于每种材质的"always_available"键内。例如，如果我们要为每个 stonehearth 建筑资源添加 2 种自定义颜色，我们的 mixinto 将如下所示：

    {
   "always_available": {
       "wood resource": {
          "colors": {
             "#A3990D": {
                "display_name": "i18n(more_colors:data.build.building_brushes.colors.#A3990D)",
                "ordinal": 100
             },
             "#D89211": {
                "display_name": "i18n(more_colors:data.build.building_brushes.colors.#D89211)",
                "ordinal": 101
             }
          }
       },
       "clay_brick resource": {
         "colors": {
            "#9CB716": {
               "display_name": "i18n(more_colors:data.build.building_brushes.colors.#9CB716)",
               "ordinal": 100
            },
            "#0B912F": {
               "display_name": "i18n(more_colors:data.build.building_brushes.colors.#0B912F)",
               "ordinal": 101
            }
         }
       },
       "stone resource": {
          "colors": {
             "#AF00E0": {
                "display_name": "i18n(more_colors:data.build.building_brushes.colors.#AF00E0)",
                "ordinal": 100
             },
             "#3400C4": {
                "display_name": "i18n(more_colors:data.build.building_brushes.colors.#3400C4)",
                "ordinal": 101
             }
          }
       }
    }
}

    "always_available" 内的键与可用资源的材质标签匹配。如果您想拥有自己的自定义建筑资源，这里是如何添加自定义建筑资源的方法。

    我们在此处添加的颜色将适用于任何建筑部件，屋顶除外。

    在每个材质标签的键内，我们有一个包含颜色列表的 "colors" 条目。每种颜色都表示为以井号 (#) 开头的十六进制 RGB 颜色。按照惯例，这些颜色为大写。

    每个颜色条目都有一个 "display_name"，这是您希望在工具提示中显示的名称的本地化键，以及一个用于在调色板中排列颜色的 ordinal。游戏中现有颜色的名称是使用此网站获得的，但您可以使用任何您想要的名称。

    列表中是否有重复的 ordinals 并不重要，但我们不能有重复的颜色。这包括在不同材质条目中重复的颜色。游戏需要将颜色与特定的资源材质关联起来以计算建筑成本。因此，请确保您添加的颜色在您的 mod 或 stonehearth mod 中都没有重复。

  3. 在游戏中测试您的新颜色（您可以使用 debugtools 中的Item Stamper生成资源）：

如何添加自定义外观

炉民的自定义外观由 customization 服务和 customization 组件控制。

对于炉民，它们在 base_human.json mixin 中声明：

    "stonehearth:customization": {},

然后，在人类实体的相应 JSON 文件（male_1.json、male_2.json、female_1.json 等）中，它们在 entity_data 部分内有 "stonehearth:customization_variants"：

  ...
  "entity_data": {
     "stonehearth:customization_variants": {
        "customization_options": "stonehearth:humans:male:customizations",
        "root": {
           "hair_and_skin_color": {
              "weights": {
                 "hair_and_skin_color": 1
              }
           },
           "age": {
              "weights": {
                 "young": 3,
                 "old": 1
              }
           }
        },
        "packages": {
           "hair_and_skin_color": {
  ...

这里有 "customization_options"，它指向一个包含我们可以应用于此实体的所有可能样式的 customization JSON 文件，"root" 它为每个包定义权重，以及 "packages"，这是一个仅包含我们希望可用于此实体的自定义（如在我们链接的 customization JSON 文件中定义）的列表，也带有权重。

"root"下的键是标识符（我们可以在此处使用任何标识符），"weights"内的键是下面声明的包的名称。

我们可以在与其基础 JSON 文件相同的文件夹中找到炉民的 customization 文件，即 stonehearth/entities/humans/male 和 stonehearth/entities/humans/female。

这些 customization 文件是索引，包含我们希望炉民能够拥有的所有可能样式，然后在 male_1.json（等）文件中，我们可以定义它们的子集，以便在创建该实体时仅从这些样式中选择。

它们包含 "categories" 和 "styles"。类别定义如下：

  "categories": {
     "head_hair": ["head_hair"],
     "face_hair": ["eyebrows", "facial_hair"],
     "skin_color": ["skin_color"],
     "eye_color": ["eye_color"],
     "hair_color": ["hair_color"]
  },

左侧是我们编造的标识符，右侧是与下面定义的样式名称匹配的字符串数组（或 "[none]"，如 female_customizations.json 中的"face_hair"类别）。 这些类别在名册屏幕中使用，用于检索样式的索引并能够通过 UI 循环浏览它们。

"styles" 包含我们用作类别值和 male_N.json / female_N.json 文件中的值的键。

这样，我们可以通过 mixinto 向现有实体添加更多类别和样式，并可以为我们的 modded 实体设置自定义变体。

添加自定义发型

发型是具有默认调色板的 QB 模型，映射在 stonehearth/data/materials/hair_color_map.json 中。当我们在名册屏幕中选择炉民的外观时，游戏会将不同的调色板应用于同一模型。

在设计您自己的发型时，您需要重用此颜色图中的颜色，但也可以添加更多颜色并添加 mixintos 以在颜色和材质图中覆盖它们。

复制现有发型的 QB 模型时，请注意它分为 2 层：

对应于头发上部的层命名为 hat，下部的层命名为 head。这种分离是实现帽子所必需的。根据您为头饰使用的透明度设置，当炉民装备该物品时，头发模型的"hat"层将被隐藏或不隐藏。

因此，在导出模型之前，我们必须确保正确命名图层。还要注意，男性炉民比女性炉民高 1 个体素，因此最好导入现有发型，以便我们可以使用滴管工具选择颜色并知道在哪里切割帽子部分。

要添加我们的自定义发型模型，我们如上所述对 male/female_customizations.json 文件进行 mixinto。

例如，我们的 mixinto 将如下所示：

  {
     "styles": {
        "head_hair": {
           "type": "model",
           "values": {
              "female_hair_9": {
                 "file": "file(hair_9.qb)"
              }
           }
        }
     }
  }

"type" 是 "model"，所以这里我们可以引用 QB 文件。 确保"values"列表中没有重复的键，否则这些模型将在游戏中被覆盖。

为皮肤、头发和眼睛添加自定义颜色

与我们添加新发型的方式相同，我们可以对 customization 文件进行 mixinto 并添加我们自己的材质图。

炉民的皮肤、头发和眼睛的颜色定义在 color_map 中。它是 stonehearth/data/materials/color_maps/human_color_map.json，它 mixin 了 3 个颜色图（一个用于皮肤，一个用于眼睛，一个用于头发）。

在这些颜色图中，我们可以看到默认模型使用了哪些颜色以及它们为材质图使用了哪些键。 因此，我们的自定义材质图必须与这些键匹配。

为了添加自定义颜色（例如，用于头发），我们必须为 male_customizations.json 创建 mixinto（如果我们想将颜色添加到两者中，则为 female_customizations.json 创建另一个）：

  "mixintos" : {
     "stonehearth:humans:male:customizations" : "file(entities/humans/male/male_customizations.json)",
     "stonehearth:humans:female:customizations" : "file(entities/humans/female/female_customizations.json)"
  }

在这个例子中，我们将为男性添加 1 种自定义头发颜色。这是 male_customizations.json mixinto：

  {
     "styles": {
        "hair_color": {
           "type": "material_map",
           "values": {
              "blue": {
                 "file": "/more_colors/data/materials/material_maps/hair_blue_material_map.json"
              }
           }
        }
     }
  }

在"hair_color"（这是包/类别值的名称）内，我们有 "type": "material_map"。这意味着我们将应用一个 material_map，它将模型的颜色更改为此 JSON 文件中定义的颜色。 在 "values" 内，我们有一个键来标识此自定义样式。

您可以在不同类别内定义的键具有相同的键名，但请确保不要与任何现有样式重复键，否则您将使用此 mixinto 覆盖它们。

在这种情况下，female_customizations.json 的 mixinto 看起来完全一样。

现在我们可以复制现有的头发材质图之一，以便获得正确的头发键（例如，stonehearth/data/material_maps/hair_red_material_map.json），重命名并编辑它。

这是编辑后的 hair_blue_material_map.json 的样子：

  {
     "stonehearth:hair:hilight" : {
        "color" : "#287ACB"
     },
     "stonehearth:hair:midtone" : {
        "color" : "#236CA4"
     },
     "stonehearth:hair:shadows" : {
        "color" : "#1C498B"
     }
  }

使用 Lua 更改自定义

我们可以在 customization 组件（_add_style 和 _remove_style 函数）中找到如何使用 Lua 更改自定义的示例：

  self._entity:get_component('render_info'):add_material_map(file_path)
  self._entity:add_component('model_variants')
                :get_variant('default')
                :add_model(file_path)
  self:_set_added_styles(subcategory, style_info)

  self._entity:get_component('render_info'):remove_material_map(file_path)
  self._entity:add_component('model_variants')
                :get_variant('default')
                :remove_model(file_path)
  self:_set_added_styles(subcategory, nil)

我们可以使用 render_info 组件添加和移除材质图，并使用 model_variants 组件更改模型。

一些来自社区成员的 mod 已经使用了自定义，例如 Hearthlings of Many Faces 或 Locks of Many Hair。
