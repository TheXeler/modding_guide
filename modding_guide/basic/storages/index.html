存储物品

在 Stonehearth 中有几种类型的存储。

堆放在 stonehearth/entities/construction/stockpile/stockpile.json 中定义。当它们在游戏中创建时，堆放组件会被添加到它们上面，因为它们需要计算一些无法在其 JSON 文件中默认分配的参数。它们在创建时也会获得存储组件。

其余的存储实体定义在 stonehearth/entities/containers 内，它们的 JSON 文件包括存储组件（资源堆除外，它们实际上是资源节点，不是存储）。

存储类型

存储类型定义在 stonehearth/components/storage/storage_component.lua 内：

  local STORAGE_TYPES = {
     CRATE = 'crate',                        -- 常规存储；无特殊行为。
     OUTPUT_CRATE = 'output_crate',          -- 不补货，但制作者可以将物品放置在那里。
     INPUT_CRATE = 'input_crate',            -- 由炉民从其他存储中取出物品来保持填充。
     URN = 'urn',                            -- 常规存储；无特殊行为。
     CRAFTER_BACKPACK = 'crafter_backpack',  -- 制作者收集材料时使用的无限容量背包。
     BACKPACK = 'backpack',                  -- 角色拥有的私人背包。
     ESCROW = 'escrow',                      -- 用于在商店交易期间临时存储物品。
  }

创建存储物品

存储物品的创建方式与其他物品相同。我们可以简单地从 stonehearth mod 中复制粘贴现有的存储物品，并调整其 JSON 文件、模型等。 这是 large_crate.json 中的存储组件： 

  ...
  "components" : {
     "stonehearth:storage": {
        "type": "crate",
        "capacity": 32
     },
     ...
  },
  ...

这里我们定义存储 "type"，它是存储组件中定义的键之一（小写的那些，见上文），以及 "capacity"，即此存储可以包含的最大物品数量。非常简单。

我们还可以将存储添加到不是箱子、垃圾桶等的实体上。例如，制作者有一个特殊的材料存储，当炉民晋升为制作职业时会注入它。它位于 stonehearth/jobs/common/crafter_storage 内。

  ...
  "stonehearth:storage": {
     "type": "crafter_backpack",
     "capacity": 128,
     "public": false
  },
  ...

炉民也有一个默认存储，在 base_human.json mixin 中分配：

  ...
  "stonehearth:storage": {
     "type": "backpack",
     "public": false,
     "capacity": 4
  },
  ...

游戏中的游戏大师遭遇战的商店也有自己的存储（定义在 stonehearth/services/server/game_master/controllers/script_encounters/trader_entity.json 内）：

  ...
  "stonehearth:storage": {
     "type": "escrow",
     "capacity": 1000000,
     "public": false
  },
  ...

如您所见，我们可以通过添加 "public" : false 来定义它们不应该可见。这样我们将防止默认的存储 UI 显示出来，并且我们仍然可以在后台访问和控制它们。默认情况下所有存储都是公开的。

输入和输出箱

如上文对存储类型的注释中所解释的，输入箱是一种特殊类型的存储。炉民将优先向它们补货，即使是已经存储在其他类型存储中的物品。它们的过滤器列表一次只允许启用一个过滤器。

这对制作者很有用，因为其他炉民可以为他们带来材料，他们不必走那么远去获取，对于建筑也很有用，因为您可以在建筑工地附近拥有一个优先存储，这样建造的炉民就不会耗尽资源。

另一方面，输出箱只能由制作者填充。他们会将刚刚制作的物品放置在输出箱中，其他炉民将从那里拿起这些物品并将它们带到具有适当过滤器的存储中（输出箱没有太多容量，尽管它们有标准的过滤器集，因为制作者可以制作任何类型的物品）。

这两种存储类型都已实现显示其内容的功能。堆放也可以显示存储的物品，但它们以线性方式执行，而对于这些存储类型，您可以自定义显示。如果您复制输入或输出箱的文件，您将已经定义了这一点，但让我们还是看一下参数：

  "stonehearth:storage": {
     "type": "input_crate",
     "capacity": 12,
     "render_contents": true,
     "filter_list": "stonehearth:ui:input_box:filters",
     "is_single_filter": true
  },
  "render_info": {
     "animation_table": "file(/data/rigs/entities/containers/input_bin/input_bin.json)"
  },
  "effect_list": {
     "default": "file(/data/rigs/entities/containers/input_bin/effects/idle.json)"
  }

它们非常不言自明。"render_contents" : true 将使存储的内容被渲染（使用此选项，您可以创建类型为'urn'或'crate'的存储，也可以显示其内容）。

过滤器列表指向此存储物品允许的过滤器列表，"is_single_filter" : true 将使我们在同一时间只能启用一个过滤器。它还将移除过滤器组的复选框，并在未选择过滤器时显示注意提示：

请记住，如果输入箱的"is_single_filter"设置为 false，它们将无法按预期工作（它们会抛出错误）。但除此之外，我们可以让我们的箱子和瓮具有单个过滤器和/或不同的过滤器列表，使用 "filter_list" 选项。

为了显示物品，我们还需要向存储添加一个骨架。我们通过定义"render_info"组件内的 "animation_table"，然后在"effect_list"组件内分配一个默认效果来实现。这是一个播放短动画（2 帧）以将物品定位在模型顶部的效果。

骨架将由一组骨骼组成，这些骨骼将确定在哪里显示每个物品。骨骼的数量对应于存储的容量，加上一些必需的帮助骨骼。

控制内容渲染的文件是 stonehearth/renderers/storage/storage_renderer.lua。此文件根据骨架初始化骨骼，将图标附加到它们上面，并在物品放下或拿起时更新显示。更多关于骨架和动画的信息在指南的这一部分。对于存储渲染器，骨骼需要以特定方式命名（以"ATTITEM"为前缀，检查 storage_renderer.lua）。如果您复制现有的输入/输出箱并重用或扩展其骨架，您就不必担心这一点。

过滤器

默认情况下，所有存储都有定义在 stonehearth/ui/data/stockpile_filters.json 中的过滤器。输入箱的过滤器定义在 stonehearth/ui/data/input_box_filters.json 中。我们可以通过 mixinto 向这些列表添加我们自己的新过滤器，或者我们可以创建自己的过滤器列表并将其用于现有或修改过的物品。

让我们创建一个新的过滤器列表并在我们的清单中为其添加别名：

  ...
  "aliases" : {
     "ui:barrel:filters" : "file(ui/data/barrel_filters.json)"
  },
  ...

barrel_filters.json 文件看起来像这样：

  {
     "type": "index",
     "stockpile": {
        "resources": {
           "display_name": "i18n(stonehearth:ui.game.zones_mode.stockpile.filters.resources)",
           "filter": "resources",
           "ordinal": 1,
           "categories": {
              "wood": {
                 "ordinal": 1,
                 "display_name": "i18n(stonehearth:ui.game.zones_mode.stockpile.filters.tooltip_wood)",
                 "icon": "/stonehearth/ui/game/modes/zones_mode/stockpile/images/taxonomy/wood.png",
                 "filter": "stockpile_wood"
              },
              "stone": {
                 "ordinal": 2,
                 "display_name": "i18n(stonehearth:ui.game.zones_mode.stockpile.filters.tooltip_stone)",
                 "icon": "/stonehearth/ui/game/modes/zones_mode/stockpile/images/taxonomy/stone.png",
                 "filter": "stockpile_stone"
              },
              "clay": {
                 "ordinal": 3,
                 "display_name": "i18n(stonehearth:ui.game.zones_mode.stockpile.filters.tooltip_clay)",
                 "icon": "/stonehearth/ui/game/modes/zones_mode/stockpile/images/taxonomy/clay.png",
                 "filter": "stockpile_clay"
              }
           }
        },
        "magic_ingredients": {
           "display_name": "i18n(my_mod:ui.game.zones_mode.stockpile.filters.magic_ingredients)",
           "filter": "magic_ingredients",
           "ordinal": 2,
           "categories": {
              "herb": {
                 "ordinal": 1,
                 "display_name": "i18n(my_mod:ui.game.zones_mode.stockpile.filters.tooltip_herb)",
                 "icon": "/my_mod/ui/game/modes/zones_mode/stockpile/images/taxonomy/herb.png",
                 "filter": "magic_ingredient herb"
              },
              "gem": {
                 "ordinal": 2,
                 "display_name": "i18n(my_mod:ui.game.zones_mode.stockpile.filters.tooltip_gem)",
                 "icon": "/my_mod/ui/game/modes/zones_mode/stockpile/images/taxonomy/gem.png",
                 "filter": "magic_ingredient gem"
              }
           }
        }
     }
  }

让我们回顾一下文件的结构：

  1. 确保添加 "type" : "index"。

  2. 在此之后创建一个 "stockpile" 键。

  3. 在"stockpile"键内添加过滤器组。这些可以命名为任何名称，并将用于在 UI 中对过滤器进行分组。它们有一个 "display_name" 用于在 UI 中显示标签，一个 "filter"，用于标识该组的复选框并同时启用/禁用该组的所有过滤器，一个 "ordinal" 用于从上到下在 UI 中对组进行排序，以及一个 "categories" 条目，我们将在其下添加我们的过滤器。

  4. 对于"resources"组，我们重用了一些现有的 stonehearth 过滤器。每个过滤器都有一个 "ordinal"（用于在组内水平排列过滤器），一个用于工具提示的 "display_name"，一个在 UI 中显示的 "icon"，以及一个关联的 "filter"。此过滤器对应于我们在"stonehearth:catalog"内的 entity_data 内的"material_tags"中编写的一个或多个标签。这样，我们告诉物品应该补货到启用了此过滤器的存储中。

    如果存储选中了"All"过滤器，材质标签将被忽略。如果改为选中"None"，则没有任何物品会补货到该存储中，炉民会将其中剩余的任何物品移动到其他存储中。

  5. 对于下一组，我们添加了一个以前不存在的新组，并将其称为"magic_ingredients"。它内部有两个过滤器，每个过滤器的"filter"有 2 个标签。只有在其"material_tags"中具有两个标签的物品才会归入这些过滤器。这样，如果我们有一种草药也不是 magic_ingredient，炉民将无法将其补货到使用此过滤器列表的容器中。

现在让我们将其添加到我们的存储物品中：

  ...
  "components" : {
     "stonehearth:storage": {
        "type": "crate",
        "capacity": 10,
        "filter_list": "silent_woods:ui:barrel:filters"
     },
     ...
  },
  ...

关于可巡逻物品

您可能已经注意到游戏中的一些存储在其 entity_data 内有一个 "stonehearth:town_patrol" 键。此键用于指定应由战斗单位巡逻的物品。其他实体，如农民田地，也有此 entity_data。

  "stonehearth:town_patrol": {
     "auto_patrol": true,
     "patrol_pattern": "perimeter"
  }

我们可以将此实体数据添加到我们希望步兵巡逻的任何实体。它必须将 "auto_patrol" 设置为 true。"patrol_pattern" 目前并未真正考虑。对于现有实体，它设置为"perimeter"，但如果可巡逻实体很小（例如：small_crate），战斗单位只会"访问"该实体，而不是围绕它转圈。

这适用于游戏中有许多此类对象的情况，以防止小的巡逻循环。
