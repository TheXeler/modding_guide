创建武器

武器示例可以在 stonehearth/entities/weapons 中找到。

与添加装备和护甲的步骤相同，但有一些细微差别。

  1. 我们将从复制 stonehearth/entities/weapons/short_sword 到我们的 mod 开始。

  2. 我们重命名文件和文件夹，例如称之为 magic_sword。请注意，有 2 个模型，一个是用于图标/默认版本，另一个将在装备武器时显示。

    注意其方向和位置。炉民抓取它的点应该在您的建模程序中的 (0,0,0)，无论模型大小如何。方向将使其在游戏中朝前，以便炉民以正确的方向挥动它：

    图标的 JSON 文件没有什么特别的。另一个 JSON 文件将在"entity_data"中具有净值和目录条目的常用属性。

    武器将只有一个默认模型变体，即"equipped"QB 模型。

    它在 entity_forms 组件中没有 ghost JSON 文件，像护甲部件那样，因为它不打算被放置。

    然后，在 "stonehearth:equipment_piece" 组件中，我们具有与护甲相同的属性。注意以下属性的值：

    • "slot": "mainhand"
    • "render_type": "attach_to_bone"

    这意味着剑将装备在"mainhand"槽位中。"mainhand"槽位在 stonehearth/data/constants.json 中映射到炉民骨架的"mainHand"骨骼。而"attach_to_bone"意味着剑将附加到该骨骼上，而不是与之合并。

    • "postures" 数组：

      "postures": [
   "stonehearth:combat",
   "stonehearth:patrol"
]

      此数组包含在姿势文件中定义的姿势，请查看 stonehearth/data/rigs/entities/humans/postures.json 获取示例（姿势将在动画指南中解释 - 待定）。

      如果我们包含此数组，这意味着武器仅在炉民播放为这些姿势定义的动画时可见，这样他们在播放进食、睡眠等动画时不会刺伤自己。

    现在，在"entity_data"中，我们有以下额外条目：

    • "stonehearth:combat:weapon_data"（与"stonehearth:combat:armor_data"相对）：

      "stonehearth:combat:weapon_data": {
   "base_damage": 16,
   "reach": 1.5
}

      "base_damage" 将显示在角色表的 Equipment 选项卡中（如果物品具有渲染类型），就像护甲物品的防御力一样。

      "reach" 以世界单位表示，因此这把剑可以触及距离我们炉民 1.5 格远的敌人（一旦炉民进入该攻击范围内，他就会开始攻击）。

    • "stonehearth:combat:melee_attacks"：

      "stonehearth:combat:melee_attacks": [
   {
      "name": "combat_1h_backhand",
      "effect": "combat_1h_backhand",
      "active_frame": 17,
      "cooldown": 0,
      "priority": 0
   },
   {
      "name": "combat_1h_forehand",
      "effect": "combat_1h_forehand",
      "active_frame": 16,
      "cooldown": 0,
      "priority": 0
   }
]

      这是一个对象数组。每个条目对应一个近战攻击。我们将在此简要介绍基本属性：

      • "name" 是一个标识符。
      • "effect" 是包含要播放动画的效果 JSON 文件的名称。请注意，它只是名称，不以".json"结尾，也不包括效果文件的完整路径。

      炉民的效果在 stonehearth/data/rigs/entities/humans/effects 内。

      • "active_frame" 是我们认为剑将击中敌人的动画帧（游戏在那一刻播放一些火花 VFX）。
      • "cooldown" 是攻击前等待的毫秒数。
      • "priority" 将使某些攻击比其他攻击更频繁。

      这些攻击将在装备此武器时在战斗中使用。它们将造成在"stonehearth:combat:weapon_data"的 "base_damage" 字段中定义的伤害，并由任何增益效果等修改。但是我们可以向它们添加一些额外属性，这将在创建自定义战斗职业的指南中深入解释 - 待定。

      您可以浏览不同的武器以同时了解它们。

  3. 我们将别名添加到我们的清单中：

    "aliases": {
   "weapons:magic_sword" : "file(entities/weapons/magic_sword)"
}

  4. 在游戏中测试您的新武器：

您可以通过在战役浏览器调试工具中触发环境威胁遭遇战来生成敌人。

双手武器

要实现双手武器，我们在 "stonehearth:equipment_piece" 组件中有这个额外字段：

  "additional_equipment": {
     "stonehearth:armor:offhand_placeholder": true
  }

这件装备可以在 stonehearth/entities/armor 中找到。它具有非常少的属性：

  {
     "type": "entity",
     "components": {
        "stonehearth:equipment_piece": {
           "slot": "leftArm",
           "ilevel": -1,
           "destroy_on_drop": true
        }
     },
     "entity_data": {}
  }

它只是一个可以装备的实体（没有"render_type"，因为我们希望它是隐形的）。

注意负的 "ilevel"。只有具有正"ilevel"的物品才能升级。所以这意味着炉民在穿着我们的双手剑时不会尝试装备盾牌，因为"leftArm"槽位将被这个隐形的额外装备占据。

他/她仍然能够升级主武器。

双手武器通常在"stonehearth:combat:weapon_data"中具有更高的攻击范围。

确保您添加的任何"additional_equipment"都具有负的 ilevel 和 "destroy_on_drop" : true，以便在炉民被降职或救援时将其销毁（否则，他们只会丢弃它，并且当他们恢复并重新装备武器时会创建一个新的，这意味着隐形物品将留在地图上的某个地方，使您的游戏变得杂乱）。

盾牌

盾牌的实现方式与任何其他护甲或武器相同。通常它们仅在某些姿势下佩戴，就像武器一样，但它们具有 armor_data，因为它们是防御物品。

因此在它们的"entity_data"内部，我们可以找到 base_damage_reduction，这在战斗时将始终应用：

  "stonehearth:combat:armor_data": {
     "base_damage_reduction": 1
  }

以及一个 "stonehearth:combat:melee_defenses" 数组（与"stonehearth:combat:melee_attacks"相对）：

  "stonehearth:combat:melee_defenses": [
     {
        "name": "simple_shield_block",
        "effect": "combat_1h_shield_block",
        "active_frame": 4,
        "cooldown": 10000,
        "priority": 0,
        "chance_of_success": 0.1
     }
  ]

请注意这里的 "chance_of_success" 属性。这是一个百分比，将用于计算运行此防御的几率，这将在该回合阻挡敌人的攻击。

"chance_of_success"属性是近战防御独有的。

请记住在您的建模程序中将盾牌模型定位在 (0,0,0) 坐标上：

远程武器

要实现远程武器，请将 stonehearth/weapons/bow 复制到您的 mod（忽略 bow_talisman.json 文件，它用于弓箭手，因为这把弓是他们的默认武器）。

在大多数方面，它与其他武器类似。重命名文件并编辑它们。注意装备版本的 QB 模型的位置和方向：

在装备部件组件中，您可能会注意到此字段：

    "no_drop": true

这用于战斗职业的默认武器或护甲，它们也没有"roles"字段。

当战斗单位丧失能力时，他们将丢弃当前的装备，因此为了保留他们的默认服装和武器（无法在游戏中获得，也不能自动装备），我们使用此 "no_drop" 属性。

某些敌人武器也将此属性设置为 true，因为我们从 Lua 手动装备他们的武器。

现在让我们检查 bow.json 的"entity_data"：

  • "stonehearth:combat:weapon_data" 包含比近战武器更多的属性：

    "stonehearth:combat:weapon_data": {
   "base_damage": 25,
   "base_ranged_damage": 25,
   "range": 30,
   "projectile_speed": 30,
   "projectile_uri": "stonehearth:weapons:arrow",
   "projectile_start_offset": { "x": -0.2, "y": 1.5, "z": -1 },
   "projectile_end_offset": { "x": 0, "y": 1.5, "z": 0 }
}

    我们既有 "base_damage" 又有 "base_ranged_damage"。远程攻击将具有优先权，因此将使用"base_ranged_damage"。

    在 "range" 中，我们有一个高值，弓箭手将在他们与目标之间达到该格数时开始攻击。

    "projectile_speed" 是以真实时间秒为单位到达目标所需的格数。

    "projectile_uri" 默认为"stonehearth:weapons:arrow"，但我们可以创建自己的投射物。

    "projectile_start_offset" 和 "projectile_end_offset" 是可选偏移量，如果我们需要在特定位置生成投射物以使其与动画匹配。

    我们可以在 stonehearth/entities/weapons 中找到投射物。例如，默认箭头看起来像这样：

    {
   "type": "entity",
   "components": {
      "model_variants": {
         "default": {
            "models": [
               "file(arrow.qb)"
            ]
         }
      },
      "mob": {
         "model_origin": { "x": -0.05, "y": 0, "z": 0.05 }
      },
      "stonehearth:projectile": {}
   },
   "entity_data": {
      "stonehearth:catalog": {
         "display_name": "i18n(stonehearth:entities.weapons.arrow.display_name)",
         "description": "i18n(stonehearth:entities.weapons.arrow.description)"
      }
   }
}

    它是一个具有非常少属性的实体。注意 "stonehearth:projectile" : {} 组件。我们可以用这种方式创建自己的投射物。

    arrow.qb 模型定位在 (0,0,0) 上，箭头的尖端溢出，这将是箭头消失之前沉入敌人的部分：

  • "stonehearth:combat:ranged_attacks"（与"stonehearth:combat:melee_attacks"相对）：

    "stonehearth:combat:ranged_attacks": [
   {
      "name": "archer_1h_shoot",
      "effect": "archer_1h_shoot",
      "active_frame": 41,
      "cooldown": 0,
      "priority": 0
   }
]

    与近战攻击相同的属性，但将由战斗服务用作远程攻击。

  • "stonehearth:combat:idle:ready"：

    "stonehearth:combat:idle:ready": {
   "name": "archer_1h_idle"
}

    通过此功能，我们可以更改他们在战斗但不攻击时播放的空闲动画。此字段只能在主武器中使用。
