关于装备部件

我们可以创建许多类型的可穿戴物品，例如护甲、头盔、服装、剑、盾牌、弓等。为了控制它们，我们在穿戴它们的实体上拥有 equipment 组件。而这些可穿戴物品中的每一个都有 equipment_piece 组件。某些装备部件实际上是隐形的，例如我们用于指定动物为宠物的项圈、一些职业能力，或狼使用的武器。

通过 equipment_piece 组件，我们不仅可以让炉民（或任何其他生物）装备物品，还可以注入一旦装备该物品就会激活的增益效果和 AI 动作。

为了使实体能够穿戴物品，它们需要在它们的组件中具有

  "stonehearth:equipment": {}

（对于炉民，这是在 base_human.json mixin 中添加的）。它们还需要具有 upgrade_equipment AI 包才能找到并装备可穿戴物品（参见 base_human.json 的 entity_data）。

添加服装和装备的一般步骤

  1. 考虑哪些职业将能够装备该物品，以及可选地，他们必须达到的等级才能装备它。

  2. 从 stonehearth mod 复制现有的装备部件，执行通常的文件重命名操作，并为您的物品创建 QB 模型。

  3. 编辑物品的 JSON 文件。以下是 "stonehearth:equipment_piece" 组件中最常见的属性：

    • "slot" -- 这是我们想要附加/合并装备的位置。它们的工作方式类似于其他游戏中的装备槽。其值是一个字符串。

      以下是 stonehearth mod 中装备部件使用的槽位：

      "helmet" / "torso" / "mainhand" / "offhand" / "leftArm" / "accessory"

      使用它们将保证它们可以被同样用于这些槽位的更新/更好的物品替换（注意"leftArm"中的大写，必须完全那样书写）。

      某些装备部件没有此属性，例如如果它们仅用于注入增益效果、命令或 AI 动作；在这种情况下，它们在内部会自动命名为"no_slot_N"。

      只要它们具有 render_type，装备的物品就会显示在 UI 中角色表的 Equipment 选项卡中。

    • "render_type" -- 有两个可能的值：

      • 如果"render_type"是 "merge_with_model"，无论我们使用哪个槽位，物品都将与炉民的部分合并，只要其 QB 模型的矩阵/层命名相同于它们试图替换的炉民骨骼。这通常用于护甲，结合模型变体中的透明度选项。

      • 如果"render_type"是 "attach_to_bone"，它将附加到"slot"属性中声明的骨骼上，无论矩阵/层名称如何（因此通常武器在其 QB 模型中只有一层）。

        此骨骼必须在 stonehearth/data/constants.json 中映射到槽位。目前，我们仅将此渲染类型用于"mainHand"、"offHand"和"leftArm"，但如果您想将实体附加到任何其他骨骼，您必须为 constants.json 制作 mixinto。例如：

        {
   "constants": {
      "equipment": {
         "SLOT_TO_BONE_MAP": {
            "ring" : "rightFinger12"
         }
      }
   }
}

        默认骨骼在骨架文件中定义（在 stonehearth/data/rigs 子文件夹中）。对于男性炉民，此文件是 stonehearth/data/rigs/entities/humans/skeletons/male.json。

        • 名为"ATTOVERCOG"、"bodyPosition"和"root"的骨骼有其自己的用途，因此避免将它们用于装备部件。

        • 名为"carry"、"mainHand"和"offHand"的骨骼是不匹配炉民模型中任何矩阵名称的帮助骨骼，用于在某些动画中将物品附加到它们上面。

        我们还可以在"SLOT_TO_BONE_MAP"中为已映射的骨骼创建新槽位，以便我们可以在炉民模型的那部分装备多个物品。那里添加的任何槽位也将出现在角色表的 Equipment 选项卡中。

        这些新槽位旨在与"attach_to_bone"渲染类型一起使用，因为"merge_with_model"只关心模型层名称。

      • 如果装备部件没有模型/不打算渲染，您也可以对此属性使用 "none" 作为值，或者干脆不在 JSON 中包含它。

    • "ilevel" -- 这是一个与槽位结合使用的任意数字。炉民将自动装备具有比他们已经装备的相应槽位物品更高 ilevel 的物品。

      此属性也是可选的，例如职业服装是自动装备的，因此它们不需要 ilevel，或者任何意在从 Lua 脚本而不是由 AI 自动装备的物品。

    • "roles" -- 以空格分隔的角色字符串或字符串数组（后者更有利于 mod 兼容性）。

      只有其职业描述 JSON 文件中具有这些角色之一或多个的炉民才能装备此物品（当我们选择物品时，他们的职业图标将出现在单位框架中，以及如果我们制作制作配方时也会出现）。因此，我们必须确保正确设置此属性，否则没有人会装备该物品。

      与"ilevel"属性类似，如果我们打算手动装备此物品，则角色是可选的。

    • "required_job_level" -- 如果我们设置了"roles"属性，我们可以选择要求满足这些角色的职业达到一定等级才能装备此物品。工人在原版游戏中无法升级，因此例如我们不会在工人的服装中使用此属性。

      不要将此所需等级与"ilevel"属性或制作此物品所需的等级混淆，它们是不同的东西。

    • "equip_effect" -- 我们想要在装备物品时播放的效果的 URI。确保效果不是循环的，否则它将永远不会完成。此字段也是可选的。

  4. 创建您需要的任何其他可选文件（例如，用于注入自定义增益效果、自定义命令、自定义AI）。

  5. 使物品在游戏中可获得（例如：通过配方、战利品等）。

  6. 测试物品是否可以获得、补货、装备，是否应用了您添加到其中的任何能力/增益效果，以及在卸下物品时它们是否被移除。

请注意，对于装备，如果您向其添加颜色映射以使其发光或类似效果，如果设置为"merge_with_model"，它将不会被正确应用。为了解决这个问题，请通过 mixinto 将颜色映射应用到 base_human.json mixin。

注入增益效果

注入增益效果、命令和 AI 是通过"stonehearth:equipment_piece"组件内的数组完成的：

  "stonehearth:equipment_piece": {
     "render_type": "merge_with_model",
     "slot": "torso",
     "ilevel": 6,
     "roles": [ "cleric_job" ],
     "required_job_level": 3,
     "equip_effect": "stonehearth:effects:weapon_level_up",
     "injected_buffs": [
        "stonehearth:buffs:devoted"
     ]
  }

在这里，我们添加了一个包含增益效果别名的 "injected_buffs" 数组。

"stonehearth:buffs:devoted"是一个永久增益效果，用于 stonehearth/jobs/worker/mixins/worker_outfit_2.json 和游戏中的其他服装中。

您可以在 stonehearth/data/buffs 中找到大多数现有增益效果。

注入命令

让我们检查 stonehearth/entities/critters/collars/pet_collar.json。这是我们在将小动物转换为宠物时装备给它的装备部件。

在"stonehearth:equipment_piece"组件内部，我们有 "injected_commands" 数组：

  "injected_commands": [
     "stonehearth:commands:release_pet"
  ]

当此物品被装备时，它将向小动物注入 release_pet 命令。现有命令可以在 stonehearth/data/commands 中找到。

注入 AI

同样，我们可以在 pet_collar.json 的"stonehearth:equipment_piece"组件中看到一个示例：

  "injected_ai": {
     "actions": [
        "stonehearth:actions:sit_on_ground",
        "stonehearth:actions:goto_sleep",
        "stonehearth:actions:sleep_when_exhausted",
        "stonehearth:actions:pet:pet_sleep_on_ground",
        "stonehearth:actions:pet:pet_sleep_near_owners_bed",
        "stonehearth:actions:pet:pet_sleep_on_ground_adjacent",
        "stonehearth:actions:run_sleep_effect",
        "stonehearth:actions:wait_for_expendable_resource_above",
        "stonehearth:actions:eat_item",
        "stonehearth:actions:pet:pet_eat",
        "stonehearth:actions:pet:pet_eat_from_ground",
        "stonehearth:actions:pet:eat_from_container",
        "stonehearth:actions:pet:eat_from_container_adjacent",
        "stonehearth:actions:pet:check_out_random_friend",
        "stonehearth:actions:pet:get_off_ladder",
        "stonehearth:actions:pet:run_in_circles",
        "stonehearth:actions:pet:follow_owner",
        "stonehearth:actions:pet:pet_run_to_safety",
        "stonehearth:actions:pet:pet_wait_at_safety_point"
     ],
     "ai_packs": [
        "stonehearth:ai_pack:conversation"
     ],
     "observers": [
        "stonehearth:observers:sleepiness"
     ],
     "components": [
        "stonehearth:consumption",
        "stonehearth:conversation",
        "stonehearth:animal_starvation",
        "stonehearth:animal_social"
     ]
  }

当小动物装备项圈时，这些 AI 包、AI 动作、观察者和组件将被注入，并在我们释放宠物时被移除。

通过这种方式，我们可以使相同的敌人具有不同的能力。它也用于在将某人晋升为特定职业时添加不同的职业能力。

如果职业应该增加军事分数，您可以在没有槽位的装备部件的装备组件内添加此项（以便它仅在降职时被移除）。来自 cleric_abilities.json 的示例：

  "military_score": 10

在 Lua 中装备物品

我们也可以在 Lua 中装备/卸下物品（来自 pet_component.lua 的示例）：

  local pet_collar = radiant.entities.create_entity('stonehearth:pet_collar')
  local equipment = self._entity:add_component('stonehearth:equipment')
  equipment:equip_item(pet_collar)

  local equipment = self._entity:get_component('stonehearth:equipment')
  if equipment then
     local outfit = equipment:unequip_item('stonehearth:pet_collar')
  end

在为袭击遭遇战等创建 JSON 文件时，可以使用调整文件（也是 JSON）声明我们希望敌人拥有的装备，因此我们不需要为此创建自定义脚本。

使用 SHED

在 SHED 中创建可穿戴物品的步骤与创建其他类型的物品相同。

在 Entity Stats Browser 中，有 Weapons 和 Defensive Items 选项卡，您可以更容易地比较它们的统计数据和 iLevel。
