创建护甲和服装

让我们为步兵创建一件新护甲。与往常一样，让我们从 stonehearth mod 中复制一个类似的物品到我们的 mod 中，并修改文件。例如，复制 stonehearth/entities/armor/iron_mail。

  1. 重命名文件。我们将在这里使用 startermod_items 魔法护甲作为示例，因此将任何"iron_mail"替换为"magic_armor"（或您想要的任何护甲标识符）。

  2. 将模型导入到您选择的体素编辑器中并进行修改。确保在导出之前矩阵/层已正确命名（否则护甲将无法粘附到炉民的身体上并且不会动画），并记得截取图标截图。如果您不愿意，护甲不需要包含所有身体部位，因此您可以删除任何不需要的矩阵/层。

  3. 修改 JSON 文件。让我们从修改 magic_armor_iconic.json 开始。这里我们只需要更新对 QB 模型、图标和 display_name/description 的引用。

    现在来看 magic_armor.json。以下是与装备部件相关的不同组件：

    • "model_variants": {...}

      "model_variants": {
   "default": {
      "layer": "armor",
      "transparent": "false",
      "models": [
         "file(magic_armor.qb)"
      ]
   },
   "female": {
      "layer": "armor",
      "transparent": "false",
      "models": [
         "file(magic_armor_female.qb)"
      ]
   },
   ...
},

      这里有几个变体。"default"用于男性炉民，"female"用于女性。这样当女性装备此护甲时，我们可以为她们使用不同的 QB 模型。我们还可以为兽人等拥有模型变体。

      "female" 键来自 stonehearth/entities/humans/female 内的 female_N.json 文件：

      "render_info": {
   "animation_table": "stonehearth:skeletons:humanoid:female",
   "model_variant": "female"
}

      男性没有定义此键，因此他们将获得 "default" 变体。
      兽人和兔子也有模型变体，定义在 stonehearth/entities/monsters/orcs/orc 和 stonehearth/entities/rabbit_biped 内的一些文件中。哥布林没有它们，它们有自己的装备部件，我们不用于任何其他实体，因此它们使用默认变体。

      您可以使用相应实体的 render info 中的该属性为装备部件定义自己的变体。

      • "layer"：这里我们声明服装的层。以下是可能的值：

        "skeleton" / "skin" / "hat" / "clothing" / "armor" / "cloak"

        如果未指定层，将使用"skin"。

        这些层允许我们将不同的模型堆叠在一起，基本上是身体上的衣服。不要将其与 QB 文件中的矩阵/层混淆，这个"layer"只是像一个组/类别一样起作用。

        通常我们不会使用"skeleton"或"skin"。其余的层值，与透明度设置一起使用，将隐藏（或不隐藏）与 QB 模型中的矩阵/层名称匹配的炉民身体部分。

      • "transparent"：如果未定义此属性，默认情况下它将是 "true"（注意引号）。当它为 "false" 时，它将完全用此模型替换炉民的身体部位（当我们使用"merge_with_model"的渲染类型时）。对于帽子很有用，可以保持显示炉民的头发（例如对于头冠），以及对于手镯等物品。

        QB 模型的所有矩阵/层都将使用此处声明的相同透明度进行处理。如果您需要不同的透明度，则需要制作单独的物品。

    • "stonehearth:entity_forms" : {}

      "stonehearth:entity_forms": {
   "iconic_form": "file(magic_armor_iconic.json)"
},

      这里我们只声明图标形态，因为此物品应该是可装备和可补货的，但不可放置。

    • "stonehearth:equipment_piece" : {}

      "stonehearth:equipment_piece": {
   "render_type": "merge_with_model",
   "slot": "torso",
   "ilevel": 7,
   "roles": [ "mail_wearer" ],
   "required_job_level": 4,
   "equip_effect": "stonehearth:effects:weapon_level_up"
}

      这里我们指定此装备部件的属性：

      • "render_type"：我们使用"merge_with_model"，因为我们希望护甲与炉民的身体合并，我们导出了具有正确层名称的 QB 模型。由于我们在 model_variants 组件中将透明度设置为"false"，任何可能从我们的护甲中弹出的皮肤部分都将被隐藏（就像护甲部分替换了该身体部位一样）。

      • "slot"：我们使用"torso"，这是游戏使用的预定义槽位之一。这样炉民可以用更好的护甲替换此护甲。

      • "ilevel"：我们使用值 7，略高于我们复制的 iron_mail 的 ilevel。

      在游戏的当前实现中，无法手动将装备分配给特定的炉民。因此，如果您希望您的物品被装备，请考虑其他物品的 ilevel，并考虑到平衡性。

      • "roles"：我们保留"mail_wearer"角色，因为我们希望此护甲仅由步兵装备。

      • "required_job_level"：我们将使用 4，因此只有 4 级或以上的步兵才会装备此物品。

      • "equip_effect"：我们重用了游戏中其他可装备物品使用的相同效果。

    在实体数据内部，我们有护甲数据：

    "entity_data": {
   "stonehearth:combat:armor_data": {
      "base_damage_reduction": 8
   },
   ...

    对于 "stonehearth:combat:armor_data"，我们只有 "base_damage_reduction" 属性，它代表此护甲的防御力。

    炉民在战斗时的最终防御力将根据其他装备的物品、增益/减益效果等而变化。

    我们还可以更改"entity_data"部分中的其余属性。

  4. 向您的清单添加别名：

    "aliases": {
   "armor:magic_armor" : "file(entities/armor/magic_armor)"
}

  5. 在游戏中测试您的新护甲：

创建头饰

大多数现有的头饰可以在 stonehearth/entities/armor 中找到。

您可以拥有一件既是衣服又是帽子的服装，但随后它们都必须共享 model_variants 组件中的"transparent"设置。

如果您单独制作帽子，那么您可以为其声明不同的透明度。

  1. 对于我们的示例，我们将创建一个王冠。让我们复制 stonehearth/entities/armor/bronze_helm 到我们的 mod，或任何其他头冠/头盔。

  2. 让我们编辑模型。层/矩阵必须命名为 "hat"：

    我们还将编辑图标的模型并截取图标截图。

  3. 现在我们编辑 JSON 文件。对于图标，我们调整模型名称。对于 magic_crown.json，我们将编辑这些属性：

    • "model_variants": {...}

      "model_variants": {
   "default": {
      "layer": "hat",
      "transparent": "true",
      "models": [
         "file(magic_crown.qb)"
      ]
   },
   "female": {
      "layer": "hat",
      "transparent": "true",
      "models": [
         "file(magic_crown_female.qb)"
      ]
   }
}

      "layer" 将是"hat"，以便我们的模型合并到身体顶部。

      我们将 "transparent" 属性设置为"true"（请注意它是一个字符串，而不是布尔值）。

      我们在装备组件中使用"merge_with_model"的渲染类型，我们的 QB 模型的层命名为'hat'，包含炉民头发上部的层也命名为'hat'，因此此透明度将使头发在合并模型后可见。

      如果我们的头饰覆盖整个头部（例如头盔或弓箭手兜帽），我们可以将其设置为"false"，这将完全隐藏炉民头发的上部。

      最后，我们更改文件名以指向我们的模型。

    • "stonehearth:entity_forms" : {}

      "stonehearth:entity_forms": {
   "iconic_form": "file(magic_crown_iconic.json)"
}

      这里我们只声明图标形态，因为此物品应该是可装备和可补货的，但不可放置。

    • "stonehearth:equipment_piece" : {}

      "stonehearth:equipment_piece": {
   "render_type": "merge_with_model",
   "slot": "helmet",
   "ilevel": 2,
   "required_job_level": 1,
   "roles": [ "cleric_job" ],
   "equip_effect": "stonehearth:effects:weapon_level_up"
}

      这里我们具有与上面护甲示例相同的属性。

      对于"render_type"，我们使用"merge_with_model"，我们的王冠将放入"helmet"槽位。我们使其仅可由牧师装备（使用"cleric_job"角色），更改"ilevel"和"required_job_level"，并将默认的"equip_effect"保持不变。

    对于实体数据，我们将编辑"stonehearth:net_worth"和"stonehearth:catalog"部分中的属性。

    我们的王冠还将增加一些防御力，因此我们在"entity_data"内包含 "stonehearth:combat:armor_data" 条目：

        "stonehearth:combat:armor_data": {
       "base_damage_reduction": 1
    }

    此条目仅包含"base_damage_reduction"属性。战斗中使用的最终防御值还将取决于炉民装备的其他物品、来自增益效果的乘数属性等。

  4. 向您的清单添加别名：

    "aliases": {
   "armor:magic_crown" : "file(entities/armor/magic_crown)"
}

  5. 在游戏中测试您的新头饰。如果它与某些发型 clipping 太多，或者对于头部来说太高/太低，您可能需要多次调整模型：
