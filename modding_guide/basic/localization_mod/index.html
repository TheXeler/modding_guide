startermod_locale mod

要制作 stonehearth mod 或任何其他 mod 的粉丝翻译，我们可以使用 startermod_locale mod 作为模板。

如果您通过 Mods 菜单中的“创建新 Mod”按钮创建翻译 mod，请确保勾选“这是一个翻译 mod吗？”复选框，以便将 mod 正确标记为 "client_only"。否则，请确保在清单的"info"部分将此属性设置为 true。

从 Github 下载 startermod_locale。将其解压缩到"mods"文件夹中，并确保文件夹名为 startermod_locale，并且具有正确的结构，以便可以加载而不会出现冲突。

在 startermod_locale 文件夹内，您应该看到一个 manifest.json 文件、一个 added_languages.json 文件和一个 translations 文件夹，其中包含 2 个文件夹，代表创建此 mod 时存在的两个 mods（stonehearth 和 rayyas_children）。在每个文件夹内，将有 2 个名为 en-XA.json 和 en-DEV.json 的文件。

startermod_locale mod 添加了 stonehearth 和 rayya children 的 en.json 文件的伪本地化。en-DEV.json 文件是用于生成 en-XA.json 的纯英文。伪本地化将 stonehearth 的 en.json 文件中的所有字符替换为带有重音符号的字符。这模拟了外语在游戏中显示时的样子。

当您安装此 mod 启动 Stonehearth 时，打开设置菜单并转到系统选项卡。您将看到名为"[!!Ēņģľĭšħ!!]"的语言和名为"English(Dev)"的语言已添加到语言列表中。选择"[!!Ēņģľĭšħ!!]"语言并点击“确定”以查看它如何更改游戏的语言。

任何未翻译的字符串将以原始语言（英语）显示。

工作原理

startermod_locale 的 manifest.json 有一个针对 supported_languages.json 的 mixinto：

    "mixintos" : {
       "stonehearth/locales/supported_languages.json" : "file(added_languages.json)"
    }

这会导致 added_languages.json 中的所有数据添加到 stonehearth mod 识别的支持语言列表中。

added_languages.json 指定了两种新语言，en-XA 和 en-DEV，如下所示：

    {
       "languages": {
          "en-XA": {
             "display_name": "[!!Ēņģľĭšħ!!]"
          },
          "en-DEV": {
             "display_name": "English(Dev)"
          }
       }
    }

"display_name" 的值是设置语言下拉菜单将显示的语言名称。您可以使用它以 native 形式显示语言名称。

名称（"en-XA"）是翻译的语言代码（参见：http://www.science.co.il/Language/Locale-codes.asp）。大多数语言和语言变体都有语言代码。在这种情况下，"en"表示英语，"XA"是为伪区域设置制作的名称。德语翻译将是 de-DE.json，中文普通话翻译将是 zh-CN.json。

如果您有特定区域的翻译，翻译文件名的最后 2 个字母必须大写！

然后，回到 startermod_locale 的 manifest.json，我们使用 overrides 添加此 mod 提供的翻译：

  "overrides": {
     "stonehearth/locales/en-XA.json": "file(translations/stonehearth/en-XA.json)",
     "stonehearth/locales/en-DEV.json": "file(translations/stonehearth/en-DEV.json)",
     "rayyas_children/locales/en-XA.json": "file(translations/rayyas_children/en-XA.json)",
     "rayyas_children/locales/en-DEV.json": "file(translations/rayyas_children/en-DEV.json)"
  }

这些 overrides 的作用就像您在 stonehearth/locales/ 和 rayyas_children/locales/ 目录下添加了新文件一样。

指向的文件是 translations 文件夹下的 en-XA.json / en-DEV.json 文件。这些是包含所有翻译字符串的文件。

对于您的翻译 mod，请从 stonehearth/locales 复制"en.json"，此示例 mod 提供的示例不包含最新版本游戏的所有字符串。还要从 rayyas_children 和 northern_alliance mods 以及您希望翻译的任何其他 mod 中复制 en.json。

如何翻译 en.json

仅翻译 JSON 文件中冒号（':'）后面的字符串。您可以使用像 Stonehearth-Translator 这样的工具更轻松地编辑 JSON。

不要修改 ':' 左侧的字符串，这些是游戏使用的字符串地址。另外，不要修改括号（'[' 或 ']'）包围的字符串。

示例：

    ...
    "ai": {
       "actions": {
          "status_text": {
             "chase_entity": "chasing [name(data.target)]",
    ...

这里，需要翻译的字符串是"chasing [name(data.target)]"。
不要翻译"[name(data.target)]"部分。这是一个特殊的字符串替换。

目标的名称将在游戏中替换"[name(data.target)]"（如果炉民正在追逐名为"Jib"的哥布林，游戏将用"Jib"替换"[name(data.target)]"，最终字符串将显示为："chasing Jib"）。

替换是区分大小写的，因此不要更改字符串是大写还是小写，也不要用不同的键替换任何字符串。最好将特殊部分复制并粘贴到您的翻译文本中。

但是，您可以将"[name(data.target)]"移动到字符串内的不同位置。
例如，日语翻译文件可能显示如下内容： 

    ...
    "ai": {
       "actions": {
          "status_text": {
             "chase_entity": "[name(data.target)]を追う",
    ...

中文文件可能看起来像：

    ...
    "ai": {
       "actions": {
          "status_text": {
             "chase_entity": "追逐[name(data.target)]",
    ...

翻译专有名词

炉民、怪物和城镇的名称不存储在 en.json 文件中。如果您也希望翻译这些名称，您可以对 stonehearth/services/server/population/data 下的文件添加 mixintos。完整示例请参见这里。

如何更改游戏字体

某些语言具有游戏默认字体中的一种或多种不支持的字符。如果您想更改它们，请在您的翻译 mod 中执行以下操作：

  1. 找到您要更改的字体。为此：

    • 在游戏打开并显示使用该字体的 UI 片段时，在浏览器中输入 localhost:1338，然后点击"Stonehearth UI"。
    • 然后激活“检查元素”按钮并返回游戏。
    • 点击具有相关字体的文本并返回浏览器。在“Style”选项卡中，您应该在其中一个 CSS 样式（font-family 属性）中找到字体名称。

  2. 游戏的字体可以在 stonehearth/ui/root/css/fonts 中找到。一旦找到对应于您要更改的字体的文件夹，检查它使用哪些格式（woff、ttf、eot、svg...）。如果您缺少特定格式的文件，可以使用像这个这样的在线工具来转换字体。

  3. 覆盖字体：将新字体复制到您的 mod 中，并为要替换的每个字体文件（.woff、.ttf、.eot、.svg）在清单中创建 overrides。

    或者，如果您在样式表中定义了自己的 @font-face，您可以仅覆盖 variables.less 文件，该文件位于 stonehearth/ui/root/css 内。为您的字体添加导入（确保字体路径是绝对的）：

    @import "/my_translation/ui/root/css/fonts/my_font/stylesheet.css";

    并更改任何排版变量以指向您导入的字体。

  4. 可选地覆盖任何其他 .less 文件，如果您看到翻译后的文本溢出 UI，或者您想更改字体大小等。

    请记住，覆盖 .less 文件将与也覆盖相同文件的其他 mod 不兼容。

    只要您的翻译 mod 已启用，即使玩家未在系统设置中选择您的语言，也会应用您的新字体。

在您的 mod 中启用本地化

如果您正在创建自己的具有可翻译字符串的内容 mod（不是纯粹的翻译 mod），您需要在 mod 的 manifest.json 中指定 "default_locale"，以告诉游戏您的 mod 包含区域设置数据：

    {
       "info" : {
          "name" : "rayyas_children",
          "namespace" : "rayyas_children",
          "version" : 3
       },
       "default_locale": "en"
    }

这告诉游戏，如果主 stonehearth 游戏运行在您的 mod 不支持的语言下，它将知道默认加载英语翻译文件。

如果您的 en.json 文件中不存在本地化路径，因为您在某处拼写错误，它将在游戏中按原样显示（"i18n(my_namespace:my_broken_item.my_display_name)"）。更多关于本地化的信息请参见这里。
