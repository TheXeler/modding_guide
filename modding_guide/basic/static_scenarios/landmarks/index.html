地标

地标是一种特殊类型的场景，允许我们通过 QB 文件向地图添加地形特征。它们是使用 stonehearth/lib/landmark_lib.lua 文件中的函数创建的。

您可以在这里和这里找到关于地标的几个直播视频。

让我们复制一个并看看它们是如何工作的。

  1. 复制 stonehearth/scenarios/static/landmarks/128x_sacred_stone.json 到您的 mod 并重命名它。

  2. 打开地标的 JSON 文件并修改它：

    {
   "mixins": "stonehearth/scenarios/static/landmarks/qb_to_terrain.json",
   "biomes": ["stonehearth:biome:temperate","stonehearth:biome:arctic","stonehearth:biome:desert"],
   "name": "custom_landmark",
   "category": "terrain_feature_common",
   "unique" : false,
   "size": {
      "width": 128,
      "length": 128
   },
   "habitat_types": ["plains", "forest"],
   "overwrite_occupied_size": 5,
   "weight" : 0.33,
   "data": {
      "brush" : "/my_mod/data/models/landmark/128x_custom_landmark.qb",
      "landmark_block_types": "stonehearth:landmark_blocks"
   }
}

    我们编辑的字段是：

    • "mixins" -- 确保它指向 stonehearth mod 中的场景文件。

    • "biomes" -- 在这里我们可以限制它仅出现在某些生物群落中。如果省略此字段，它将在任何生物群落中生成，甚至是修改过的生物群落。

    • "name" -- 为我们的地标指定一个唯一的名称。

    • "category" -- 这里我们使用 scenario_index.json 中列出的地标类别之一。

    • "unique" -- 我们允许此地标在每张地图上生成多次。

    • "size" -- 此场景在计算其生成位置时将占用的方块大小。

    • "habitat_types" -- 此场景可以生成的生境。可能的值是"plains"、"forest"、"foothills"、"mountains"和"water"。

    • "overwrite_occupied_size" -- 我们的地标将在其"size"内搜索最多为此值的占用大小的方块组以进行覆盖。因此，一棵具有 2x2 体素基础（4 个体素）的树将需要"overwrite_occupied_size"为 5 或更大才能在其上放置地标。不过不要将此值设置得太高（大约在 6 左右之后，它基本上会覆盖在所有东西之上）。

    • "weight" -- 用于计算与其他现有地标一起生成几率的权重。

    • "data" -- 在这里我们指定 "brush"（即 QB 模型）和 "landmark_block_types"，默认情况下将是"stonehearth:landmark_blocks"。可选地，我们还可以像这样指定一个掩码：

      "data": {
   "brush" : "/stonehearth/data/models/landmark/128x_skeleton_caribou_arctic.qb",
   "brush_mask" : "/stonehearth/data/models/landmark/128x_skeleton_caribou_arctic_mask.qb",
   "landmark_block_types": "stonehearth:landmark_blocks"
}

    掩码是一个类似的模型，但使用了来自可能的地标块的其他颜色。这些颜色代表当我们挖掘这些块时将从地标掉落什么，无论它们在原始模型中具有什么颜色。在上面的示例中，挖掘驯鹿的骨架将掉落骨头，无论我们挖掘的是它的身体还是角（它们各自具有不同的颜色）。

  3. 编辑 QB 模型并导出它。如果它太大，您的体素编辑器可能会卡顿或挂起，请注意这一点。

    我们可以为地标模型使用任何颜色。但我们需要考虑在 stonehearth/data/terrain/landmark_blocks.json 中定义的地标块：

    "drop_chance_default": 0.04,
"color_water": "#0000FF",
"color_water_lowered": "#0001FF",
"color_mimic_tag": "#FFFF00",
"#FF00FF": {
   "terrain_tag": "null"
},
"#25221A": {
   "terrain_tag": "unknown"
},
"#514E45": {
   "terrain_tag": "bedrock"
},
"#7575FF": {
   "loot_bag": "stonehearth:manipulation:wet_stone",
   "facing": "random",
   "force_iconic": false,
   "fill_voxel_with_water": true
},
...

    我们在这里定义了不同的键。对于代表十六进制颜色的键，我们可以：

    • 定义匹配的地形标签（来自 terrain_blocks.json）。当地标在游戏中加载时，QB 模型中的该体素将被替换为此处定义的地形的 1 个方块。
    • 定义一个物品来替换游戏中的该方块。请注意，某些物品在游戏中占据超过 1 个方块。我们可以在此处使用不同的属性：
      • "loot_bag" -- 这是物品的 URI。
      • "facing" -- 这要么是 "random"，要么是表示直角的数字（0、90、180、270）。
      • "force_iconic" -- 通常它将是 false，因为我们希望生成实体的根形态，但有时我们可能希望它为 true。
      • "fill_voxel_with_water" -- 这仅用于湿石头。
      • "fill_voxel_with_low_water" -- 这仅用于干石头。
      • "drop_chance" -- 如果我们不包含它，默认情况下它将如上所述为 0.04。这是当方块被挖掘时 URI 或资源掉落的几率。
      • "placement_chance" -- 此物品的放置几率，因此有时不会在此方块中放置任何物品。它是可选的。
      • "owner" -- 我们可以选择定义物品的所有者。例如，带有旧农场的地标有小麦堆，其 "owner" : "" 允许玩家掠夺它们。墓地有它们的实体，其 "owner" : "undead"，因此它们将是敌对的，我们的战斗单位将自动攻击它们。

    如果我们为我们的体素使用 "color_mimic_tag" 颜色，它们将被着色为它们生成的地形颜色（颜色将从地标生成的中心位置的单个方块获取）。

    不同地形标签的颜色也将适应它们所在的生物群落。例如，映射到 rock_layer_3 的颜色 #82807B 将在温带生物群落中显示为灰色，但在沙漠生物群落中显示为粉色，因为那是该生物群落中第三岩石层的颜色。

    "color_water" 用于将水填充到体素顶部 - 适用于您希望水在地标生成后立即流出的瀑布。"color_water_lowered" 用于有水盆的地方，其中有干石头（或者您只是不希望水将其所在的区域填充到边缘）。
    一般来说，如果水要从盆地中流出，您希望使用"color_water"。如果水要结束在盆地中，请使用"color_water_lowered"。

    所有颜色的十六进制值必须为大写，否则它们将无法适应生物群落。

    使用未在 landmark_blocks.json 中定义的颜色将使它们无论地标在哪个生物群落中生成都显示相同的颜色。如果我们向 terrain_blocks.json 和/或 landmark_blocks.json 添加一些新的块类型，我们也可以在地标中使用它们。

    有许多地标在地下生成，除非玩家挖掘，否则他们看不到它们。我们可以通过使用 "null" 标签的颜色来创建地下洞穴，它将从与其相交的地形中移除方块。但它们应该尽可能短，否则如果玩家将他们的旗帜放置在洞穴上方，起始炉民可能会在其中生成，这会令人困惑。

    这是现有洞穴地标之一的模型，从下方查看： 当地标生成时，所有粉色都将从地形中移除（此场景的类别为 "terrain_feature_underground"）。如您所见，它只有 3 个方块高。

  4. 在您的清单中向场景索引添加 mixinto：

    "mixintos" : {
   "stonehearth:scenarios:scenario_index" : "file(scenarios/scenario_index.json)"
}

    我们的 mixinto 看起来像这样：

    {
   "static" : {
      "scenarios" : [
         "file(custom_landmark)"
      ]
   }
}

  5. 测试地标是否在我们定义的生物群落中生成。您可能需要调整值以获得更高的生成几率，甚至更改生境，因为有许多地标可供选择，可能需要多次尝试才能使其出现。
