小动物巢穴和废墟

小动物巢穴

向游戏中添加您自己的小动物巢穴非常容易。 这种类型的场景在 stonehearth/scenarios/static/critter_nest 内实现，通过一个 JSON 文件和一个脚本。

然后，我们在其他场景（beetle_nest、deer_nest 等）中将 critter_nest.json 作为 mixin 重用。

让我们复制 stonehearth/scenarios/static/ostrich_nest 到我们的 mod 并重命名它。

然后我们必须修改 JSON 文件：

  {
     "mixins" : "stonehearth/scenarios/static/critter_nest/critter_nest.json",
     "name" : "jelly_nest",
     "habitat_types" : ["forest", "plains", "foothills"],
     "biomes": [ "stonehearth:biome:temperate", "candyland:biome:sugarfield_plains" ],
     "weight" : 3,
     "data" : {
        "num_critters" : {
           "min" : 3,
           "max" : 5
        },
        "critter_type" : [ "candyland:critters:jelly", "candyland:critters:jelly:tiny" ]
     }
  }

  1. 调整到小动物 mixin 的路径，使其为绝对路径，否则游戏将尝试在您的 mod 中查找它。

  2. 为此场景指定一个 "name"。

  3. 在 "habitat_types" 数组中，我们可以有一个或多个此场景将生成的生境。可能的值是：

    "plains" / "foothills" / "forest" / "mountains" / "water"

  4. 可选地，我们可以指定我们希望此场景生成的生物群落。如果没有 "biomes" 数组，小动物巢穴将在所有生物群落中生成，甚至是修改过的生物群落。

  5. 还有更多可选属性，如果我们不在 JSON 文件中包含它们，它们在 critter_nest.json 中有默认值：

    • "weight" : 10 -- 以便它可以比其他小动物巢穴更频繁地生成。

    • "size" : { "width" : 16, "length" : 16 } -- 小动物巢穴将占据的区域。

    • "unique" : false -- 同一地图中可以生成多个小动物巢穴。

    • 在 "data" 内部，唯一必需的属性是 "critter_type"（一个包含我们小动物别名的数组）。

      "num_critters" 的默认值为 "min" : 1 / "max" : 3。还有 "critter_length" : 1，这是小动物生成时的默认边距，我们也可以为我们的巢穴调整该值。

  6. 创建对 scenario_index.json 的 mixinto 以添加您的小动物巢穴：

    "mixintos" : {
   "stonehearth:scenarios:scenario_index" : "file(scenarios/scenario_index.json)"
}

    在我们的示例中，mixinto 将看起来像这样：

    {
   "static" : {
      "scenarios" : [
         "file(static/jelly_nest/jelly_nest.json)"
      ]
   }
}

  7. 测试您的小动物巢穴。您可能需要提升一个战斗单位以更快地清除战争迷雾。由于还有其他小动物巢穴，可能需要一段时间才能找到您自定义小动物的巢穴（您可以增加场景的权重以进行测试）。

您可以使用 mixintypes 等方法删除其他默认场景，但请注意，这可能会影响玩家正在使用的其他生物群落。

废墟

在游戏中，兔子雕像作为静态场景生成。我们也可以将它们作为生成实体的一部分添加到每个生物群落文件中，但这种方式它们可以自动在任何生物群落中生成。

我们可以复制 stonehearth/scenarios/static/terrain/rabbit_clan_statue 到我们的 mod 并重命名它。

  {
     "mixins" : "stonehearth/scenarios/static/entity_cluster/entity_cluster.json",
     "name" : "bunny_altar",
     "category" : "ruins",
     "size" : {
        "width" : 10,
        "length" : 10
     },
     "weight" : 1,
     "habitat_types" : ["forest"],
     "data" : {
        "entity_type" : [
           "candyland:decoration:bunny_altar"
        ],
        "entity_footprint_length" : 5,
        "quantity" : {
           "min" : 1,
           "max" : 1
        }
     }
  }

这些是我们需要更改的属性：

  1. 确保编辑 "mixins"，将路径更改为绝对路径，否则游戏将尝试在我们的 mod 中查找 entity_cluster.json。

  2. 为场景指定一个 "name"。这些属性与小动物巢穴场景的属性非常相似。

  3. 更改 "size"（如果我们不包含它，默认情况下它将是 16x16）。

  4. 可选地为其指定一个 "weight"（默认情况下它将是 10），以便它在其他"ruins"场景上具有更高的优先级。

  5. 编辑 "habitat_types"，可以是 "forest"、"plains"、"foothills"、"mountains" 和/或 "water"。

  6. 在 "data" 内部（将由 entity_cluster.lua 脚本使用），我们有一个包含 "entity_type" 的数组，包含我们废墟的别名，一个用于在我们实体周围提供边距的 "entity_footprint_length"，以及要生成的实体的 "quantity"。

  7. 然后，添加对 scenario_index.json 的 mixinto 以添加您的场景：

    "mixintos" : {
   "stonehearth:scenarios:scenario_index" : "file(scenarios/scenario_index.json)"
}

    在我们的示例中，mixinto 将看起来像这样：

    {
   "static" : {
      "scenarios" : [
         "file(static/terrain/bunny_altar/bunny_altar.json)"
      ]
   }
}

  8. 最后，让我们测试我们的废墟是否可以出现在任何生物群落中（我们可以通过使用 "biomes" 数组来限制我们希望它出现的生物群落）。
