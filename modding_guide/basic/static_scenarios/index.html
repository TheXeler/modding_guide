静态场景

静态场景是出现在地图上并在世界生成时创建的静态元素。它们可以默认显示，或者隐藏直到炉民清除战争迷雾。它们与生物群落文件是分开的。

它们在 static_scenario_service.lua 文件中管理。

以下是我们可以仅使用数据创建的不同类型的静态场景（您也可以使用自定义脚本创建您自己的静态场景类型）：

  • 矿脉
  • 废墟
  • 地标
  • 小动物巢穴

动态场景很久以前就被弃用，转而使用游戏大师的遭遇战系统。

为 Ascendancy 王国制作的起始教程也是使用场景实现的，其大部分数据位于 stonehearth/data/tutorial 内。

在 scenario_index.json 内，我们可以找到应用于游戏的场景。我们还会找到可以用于每个场景的类别，因此我们可以通过 mixinto 向此部分添加我们自己的类别：

  "categories" : {
     "terrain" : {
        "density" : 12,
        "priority" : 0,
        "location_type" : "surface",
        "activation_type" : "immediate"
     }
  }

每个类别都有这些字段：

  • "density" -- 场景按其体积加权，因此密度为 1 意味着每个单元格都被占用。一个 10x10 的场景占据的体积与 100 个 1x1 的场景相同。为了使它们的密度相等，10x10 的场景被选择的频率应该低 100 倍。
  • "priority" -- 为了使某些场景在其他场景之前生成，它们首先按优先级排序，然后按面积排序，最后按权重排序。
  • "location_type" -- 要么是 "surface"（地表），要么是 "underground"（地下）。
  • "activation_type" -- "immediate" 将使场景在世界生成期间出现，"revealed" 将使其在炉民清除其所在位置的战争迷雾时出现。

对于地标场景，我们有其他类别：

  "terrain_feature_common" : {
     "density" : 100,
     "max_count": [2,2,3,3,3,4,4],
     "prevents_categories_in_same_tile": ["terrain_feature_common", "terrain_feature_rare"],
     "priority" : "random",
     "location_type" : "surface",
     "activation_type" : "immediate"
  }

它们还有几个额外的属性：

  • "density"、"location_type" 和 "activation_type" 与上面的类别相同。

  • "priority" 是 "random"，它们与具有数值优先级的场景分开处理，并将随机选择。

  • "max_count" 是一个数字数组。将从中选择一个随机数来表示要在地图上生成多少个此类别的地标。

    在此示例中，有三个 3，以及两个 2 和 4，这意味着 3 更有可能被选择，因此在地图上生成 3 个此类别地标的次数会更多，有时只有 2 个或 4 个。

  • "prevents_categories_in_same_tile" 是我们不希望与此类别出现在同一图块中的其他场景类别的数组。

自定义静态场景类型

场景可以使用 JSON 文件来保存数据，并使用 Lua 脚本来实现。然后我们可以重用该 JSON 文件作为 mixin 来实现该类型的不同场景（就像我们对矿脉、废墟、小动物巢穴和地标所做的那样）。

具体的 JSON 文件将被添加到场景索引中，游戏将知道如何生成它们。

如果我们想创建我们自己的自定义静态场景类型，我们的 JSON 文件将需要具有以下字段：

  "name" : "my_custom_scenario",
  "size" : {
     "width" : 16,
     "length" : 16
  },
  "weight" : 10,
  "unique" : false,
  "script" : "file(my_custom_scenario.lua)",
  "data" : {
     "entity_footprint_length" : 1,
     "quantity" : {
        "min" : 2,
        "max" : 5
     }
  }

然后，我们的自定义脚本应该看起来像这样：

  local MyCustomScenario = class()

  function MyCustomScenario:__init()
  end

  function MyCustomScenario:initialize(properties, context, services)
     -- 在这里我们可以像这样从 JSON 的"data"块中检索数据：
     local data = properties.data

     -- 这里是我们的自定义代码。我们可以使用名为 services 的参数来调用
     -- stonehearth/services/server/static_scenario/scenario_modder_services.lua 中的函数
  end

  return MyCustomScenario

我们可以要求任何其他类或 Lua 文件，但如果它们来自 stonehearth mod，我们需要在其前面加上 stonehearth. 前缀。例如，要要求与 qb_to_terrain.lua 脚本中相同的文件，我们将这样写：

  local landmark_lib = require 'stonehearth.lib.landmark.landmark_lib'
  local Point3 = _radiant.csg.Point3

之后我们将为我们的场景声明类。
