矿脉

矿石和其他资源可以从采矿中掉落。我们可以创建自己的资源矿脉，它们将在地形中生成，并使它们掉落更多的特定资源。

  1. 首先，创建一个采矿资源。这可以是任何东西，我们将用它作为采矿的掉落物。您可以从 stonehearth/entities/resources 复制现有的资源。

    在我们的示例中，我们复制了 lump_of_coal 并将其重命名为 emerald：

    {
   "mixins": "stonehearth:mixins:item_properties",
   "type": "entity",
   "components": {
      "model_variants": {
         "default": {
            "models": [
               "file(emerald.qb)"
            ]
         }
      }
   },
   "entity_data": {
      "stonehearth:net_worth": {
         "value_in_gold": 1,
         "rarity": "common",
         "shop_info": {
            "buyable": true,
            "sellable": true,
            "shopkeeper_level": 1,
            "shopkeeper_type": "caravan"
         }
      },
      "stonehearth:catalog": {
         "display_name": "i18n(mineral_mod:entities.resources.emerald.display_name)",
         "description": "i18n(mineral_mod:entities.resources.emerald.description)",
         "icon": "file(emerald.png)",
         "is_item": true,
         "category": "resources",
         "material_tags": ["stockpile_ore", "emerald" "resource", "gem"]
      },
      "stonehearth:appeal": {
         "appeal": -2
      }
   }
}

    该模型将用于图标/可补货的资源。如果此资源将用于建筑，您可以在组件内添加堆叠：

    "stonehearth:stacks": {
   "max_stacks": 60
}

    这样，在使用它进行建筑时，资源将持续 N 个方块。您还需要将您的资源添加到建筑资源列表中才能使其工作。

    确保将 material_tags 中的 coal（或任何资源名称）重命名为唯一的内容，并且它具有有效的存储过滤器。

  2. 在清单中为您的资源添加别名：

    "aliases" : {
   "resources:emerald": "file(entities/resources/emerald)"
}

  3. 使您的资源从采矿中掉落：

    为此，我们需要通过 mixinto 将我们的资源添加到 stonehearth/services/server/mining/mining_loot_tables.json 文件中：

    "mixintos" : {
   "stonehearth:mining:base_loot_table" : "file(mixintos/mining_loot_tables_mixinto.json)"
}

    我们的 mixinto 文件看起来像这样：

    {
   "mineable_blocks": {
      "rock": {
         "entries": {
            "default": {
               "items": {
                  "emerald": {
                     "uri": "minerals_mod:resources:emerald",
                     "weight": 50
                  }
               }
            }
         }
      },
      "emerald": {
         "entries": {
            "default": {
               "num_rolls": 1,
               "items": {
                  "none": {
                     "uri": "",
                     "weight": 100
                  },
                  "gem": {
                     "uri": "minerals_mod:resources:emerald",
                     "weight": 50
                  }
               }
            }
         }
      }
   }
}

    这是一个战利品表，列出了地形类型以及我们在挖掘它们时可以掉落的物品。

    这段数据告诉我们将我们的资源添加到"rock"战利品表中，并赋予它 50 的权重。在调整此权重之前，请检查 stonehearth/services/server/mining/mining_loot_tables.json 中的数字。如果您的数字太低，您的资源将不会掉落太多（如果有的话）。如果您的数字太高，它将超过所有其他资源，而其他任何东西都不会掉落！

    如果您愿意，mining_loot_tables 中还有其他战利品表可以添加您的资源。Rock 只是最合乎逻辑的开始。

    生物群落文件夹中存在名为 temperate_mining_loot_table.json、desert_mining_loot_table.json 和 arctic_mining_loot_table.json 的文件。这些文件用于您要添加的任何仅特定于该生物群落类型的额外采矿战利品表。您可以以相同的方式 mixinto 到它们中，使您的资源依赖于生物群落。我们使用的通用战利品表将应用于所有生物群落。

    "mineable_blocks" 内的键对应于 stonehearth/data/terrain/terrain_blocks.json 中 "kind" 键的值。这样，无论我们挖掘哪种颜色的岩石、草地等，资源都会掉落。

  4. 将您的资源添加到地形块中：

    由于我们在"mineable_blocks"中添加了"emerald"作为条目，我们需要确保存在名为"emerald"的地形类型。我们对 terrain_blocks.json 的 mixinto 看起来像这样：

    {
   "block_types" : {
       "emerald" :               { "tag" :   550, "kind" : "emerald" }
   },
   "selectable_kinds" : {
      "emerald" :       "mineral_mod:terrain:ui:emerald"
   }
}

    我们添加一个名为"emerald"的新键，为其分配一个 tag（这是一个唯一的数字，检查 terrain_blocks.json 中的其他数字以避免重复），以及一个 "kind"，它是我们在"mineable_blocks"中的条目名称。"kind"可以是与主键字符串不同的字符串。

    我们还在 "selectable_kinds" 部分中添加它。这样，当我们点击矿脉时，其名称将显示在单位框架中。它指向的别名对应于此文件：

    {
   "mixins": "stonehearth/entities/terrain/ui/ore_block/ore_block.json",
   "components": {},
   "entity_data": {
      "stonehearth:catalog": {
         "display_name": "i18n(mineral_mod:entities.terrain.ui.emerald_block.display_name)",
         "description": "i18n(mineral_mod:entities.terrain.ui.emerald_block.description)"
      }
   }
}

    确保"mixins"指向 stonehearth mod，并调整我们地形类型的名称和描述。

  5. 指示新地形块类型的颜色。

    目前，这必须通过对生物群落 generation_data 文件的 mixinto 来完成，这意味着我们需要为我们将生成矿脉的所有生物群落注入此 mixinto，否则其颜色将是默认颜色（几乎是黑色的红色）：

    "mixintos" : {
   "stonehearth:mining:base_loot_table" : "file(mixintos/mining_loot_tables_mixinto.json)",
   "stonehearth/data/biome/temperate_generation_data.json" : "file(data/biome/ore_color_mixinto.json)",
   "stonehearth/data/biome/desert_generation_data.json" : "file(data/biome/ore_color_mixinto.json)",
   "stonehearth/data/biome/arctic_generation_data.json" : "file(data/biome/ore_color_mixinto.json)"
}

    mixinto 将看起来像这样：

    {
   "palettes": {
       "spring": {
          "emerald": "#4EA85A"
       },
       "summer": {
          "emerald": "#4EA85A"
       },
       "autumn": {
          "emerald": "#4EA85A"
       },
       "winter": {
          "emerald": "#4EA85A"
       }
   }
}

    我们为每个季节添加一个包含我们地形类型（terrain_blocks.json 中 "block_types" 内的主键）的条目，并将其链接到我们想要用于矿脉的十六进制格式的颜色。地形块不接受自定义渲染材质，它们都将使用相同的着色器（在 C++ 端硬编码）。水有一个单独的渲染器。

    即使玩家没有安装其他 mod，我们也可以向它们的文件添加 mixinto。那些文件无法找到的警告将写入 stonehearth.log，但不会在游戏中弹出。如果来自多个 mod 的警告太多，游戏可能需要更长的时间才能启动。

  6. 最后，我们可以创建我们的矿脉并将其添加为静态场景，以便它在地形中生成。

    复制 stonehearth/scenarios/static/terrain/coal_vein（或任何其他矿脉），重命名并编辑它：

    {
   "mixins" : "stonehearth/scenarios/static/terrain/ore_vein/ore_vein.json",
   "name" : "emerald_vein",
   "kind" : "emerald",
   "weight" : 75
}

    确保"mixins"指向 stonehearth mod。"weight" 将与其他矿脉一起使用，因此请检查其他文件，以确保您的矿脉有足够的生成机会。

    我们可以包含在此 JSON 文件中的其他字段在 ore_vein.json 中有默认值：

    • "habitat_types" : ["mountains"] -- 将使其仅在声明的生境中生成。通常我们希望矿脉仅在"mountains"生境中生成。

    • "size" : { "width" : 88, "length" : 88 } -- 这是场景在计算其生成位置时将占用的默认大小。

    • "unique" : false -- 同一地图中可以生成多个此资源类型的矿脉。

    • "data" : {} -- 我们可以在此处自定义几个属性，并将用于在 stonehearth/services/server/mining/ore_generator.lua 中创建矿脉。以下是它们的默认值：

      • "num_veins_min" : 2 -- 将在地图中生成的此类型矿脉的最小数量。
      • "num_veins_max" : 4 -- 将在地图中生成的此类型矿脉的最大数量。总数是在最小值和最大值之间的随机数。
      • "vein_length_min" : 44 -- 矿脉的最小长度。
      • "vein_length_max" : 72 -- 矿脉的最大长度。最终长度将是最小值和最大值之间的随机方块数。
      • "vein_radius" : 2 -- 矿脉的半径（因此矿脉的总宽度默认为 4 个方块）。
      • "mother_lode_radius_min" : 3 -- 母矿脉的最小半径。
      • "mother_lode_radius_max" : 5 -- 母矿脉的最大半径。最终半径将是最小值和最大值之间的随机数，因此最终宽度将是该值的两倍。

    将 mixinto 添加到场景索引中：

    "mixintos" : {
   "stonehearth:mining:base_loot_table" : "file(mixintos/mining_loot_tables_mixinto.json)",
   "stonehearth/data/biome/temperate_generation_data.json" : "file(data/biome/ore_color_mixinto.json)",
   "stonehearth/data/biome/desert_generation_data.json" : "file(data/biome/ore_color_mixinto.json)",
   "stonehearth/data/biome/arctic_generation_data.json" : "file(data/biome/ore_color_mixinto.json)",
   "stonehearth:scenarios:scenario_index" : "file(scenarios/scenario_index.json)"
}

    mixinto 将看起来像这样：

    {
   "static" : {
      "scenarios" : [
         "file(static/terrain/emerald_vein)"
      ]
   }
}

  7. 测试您的矿脉是否出现在地图中。

    这可能需要一段时间，因为放置是随机的。使用debugtools提升一个战斗单位，并让他们沿着山脉的边缘奔跑，直到您发现您的资源矿脉。然后点击它以检查单位框架，并命令您的炉民挖掘它以查看您的资源是否正在掉落。还可以挖掘岩石或在采矿战利品表中使用的任何其他地形类型，并检查掉落是否正常工作。
