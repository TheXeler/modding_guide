添加配方

我们在之前的部分中已经介绍了如何创建我们自己的物品。当然，您可以为这些物品添加任何您想要的组件，如果您从头创建模型，请确保查看建模指南和关于碰撞区域的指南，以便您的模型正确导出并在世界中对齐。如果需要，这里还有如何更改物品比例的方法。

在这篇指南中，我们介绍了如何向游戏中添加您自己的制作职业。

  • 向 stonehearth mod 中的现有制作者添加配方的步骤
  • 配方的结构
  • 将配方添加到配方列表
  • 使用 SHED 添加配方

向 stonehearth mod 中的现有制作者添加配方的步骤

  1. 决定哪个现有制作者将制作该物品。

  2. 为该物品创建配方。

  3. 添加 mixinto 以将新配方集成到该制作者的配方列表中。

  4. 测试制作者是否可以制作该物品。

让我们开始吧。我们将为在 startermod_basic mod 的先前示例中创建的自定义桌子添加配方。 我们将让石匠制作它。

配方的结构

让我们复制木匠的"table_for_one"配方并调整其值以引用我们的自定义桌子和我们 mod 的本地化文件。它位于 stonehearth\jobs\carpenter\recipes 内。我们将我们的文件命名为 custom_table_recipe.json。让我们还添加一些可能在配方中遇到的额外字段：

  {
     "type": "recipe",
     "effort": 30,
     "work_units": 3,
     "recipe_name": "i18n(startermod_basic:jobs.mason.recipes.custom_table_recipe.recipe_name)",
     "description": "i18n(startermod_basic:jobs.mason.recipes.custom_table_recipe.description)",
     "flavor": "i18n(startermod_basic:jobs.mason.recipes.custom_table_recipe.flavor)",
     "portrait": "/startermod_basic/entities/furniture/custom_table/custom_table.png",
     "level_requirement" : 2,
     "workshop": "stonehearth:mason:workbench",
     "ingredients": [
        {
           "material": "stone resource",
           "count": 1
        }
     ],
     "produces": [
        {
           "item": "startermod_basic:furniture:custom_table"
        }
     ]
  }

字段解释：

  • "type" : "recipe"。通过此字段，我们指示此 JSON 文件代表一个制作配方。

  • "work_units" : 3。此值是制作动画将播放的次数。数值越高，制作物品所需的时间越长。

  • "effort" : 30。这是"基于努力的制作"的数值。它代表制作物品所需的游戏内分钟数（如 constants.json 中所定义）。

    在本指南发布时，它可能已启用/可能尚未启用（如果您不在配方中包含它，将根据拥有该配方的 stonehearth 制作者使用默认值）。在此字段在游戏中启用之前，请确保至少指定"work_units"。

    基于努力的制作替换了制作的"work_units"，使制作速度不依赖于制作动画的长度。某些制作动画比其他动画短，因此对于相同的"工作单位"，某些制作者制作物品所需的时间更少。

    "基于努力的制作"不仅消除了动画依赖，它还根据制作者是否疲倦（具有"Groggy"减益效果）或高兴/悲伤而使制作更快或更慢。

    您可以在 stonehearth/components/workshop/crafting_progress.lua 和 stonehearth/components/crafter/crafter_component.lua 中找到有关此的更多信息。

  • "recipe_name" : "i18n(...)"。将在工作坊 UI 中显示的配方名称。通常与生产的物品名称相同。

  • "description" : "i18n()"。将在工作坊 UI 中显示的配方描述。

  • "flavor" : "i18n()"。将在工作坊 UI 中显示的配方的小号斜体文本。它是可选的。

  • "portrait" : "..."。配方图像的路径。此图像通常在物品的文件夹中，而不是在配方的文件夹中，因为它也用作单位框架的图标。

  • "level_requirement"。这是一个可选字段。制作者解锁此配方所需的等级。在他/她达到此等级之前，配方将在工作坊 UI 中显示为灰色。所有职业都从 1 级开始（但工人不会升级，他们是基本职业）。如果我们不添加等级要求，如果他们有所需的原料并放置了相应的工作台，他们将能够立即制作物品。

  • "workshop" : "uri_of_the_workbench"。这是制作者将用于制作物品的工作台的限定别名。我们只能指定一个别名。您可以在 stonehearth 清单中找到现有工作台的别名。记得在这里用定义它的 mod 的命名空间作为前缀（在这种情况下是 stonehearth）。

如果我们不在配方文件中添加"workshop"字段，制作者将用手制作物品。这通常用于制作初始工作台。

  • "ingredients" : [{...}]。这是配方的原料列表。它是一个对象数组。您可以像在我们的自定义桌子配方中那样使用通用资源（使用资源的 entity_data 的材质标签），或使用特定的 URI 作为原料，如下所示：

    "ingredients": [
   {
      "uri": "stonehearth:food:egg",
      "count": 2
   },
   {
      "uri": "stonehearth:food:carrot:carrot_basket",
      "count": 1
   }
],

使用通用资源意味着制作者可以使用任何算作该资源的物品（例如，来自任何类型树木的木材）。记得用 mod 的命名空间加冒号作为物品别名的前缀。"count" 字段的原料是不言自明的。

  • "produces" : [{...}]。这是另一个对象数组。这里我们放置配方的结果。不过与前一个对象数组有一个区别。如果我们希望配方生产多个物品，我们不能在这里使用"count"字段。我们需要在不同的条目中重复相同的"item"；如下所示：

    "produces": [
   {
      "item": "startermod_basic:furniture:custom_table"
   },
   {
      "item": "startermod_basic:furniture:custom_table"
   },
   {
      "item": "startermod_basic:furniture:custom_table"
   }
]

我们也可以通过这种方法使配方生产几种不同的物品，只需编写不同的 URI。

将配方添加到配方列表

我们需要为配方列表创建一个 mixinto。我们将从 stonehearth/jobs/mason/recipes/recipes.json 文件中获取键：

  {
     "craftable_recipes" : {
        "furniture" : {
           "recipes" : {
              "custom_table_recipe" : {
                 "recipe" : "file(custom_table_recipe.json)"
              }
           }
        }
     }
  }

在这种情况下，我们重用了石匠配方中的现有类别（家具）。如果我们想在我们自己的新类别中添加我们的配方（这将在工作坊 UI 的配方列表中单独显示），我们需要在我们的 mixinto 中添加另外两个字段。示例：

  {
     "craftable_recipes" : {
        "custom_category" : {
           "ordinal": 80,
           "name": "i18n(startermod_basic:jobs.mason.recipes.custom_category_name)",
           "recipes" : {
              "custom_table_recipe" : {
                 "recipe" : "file(custom_table_recipe.json)"
              }
           }
        }
     }
  }

那里，"custom_category" 是配方的组的标识符。"ordinal" 是一个用于在 UI 中对组进行排序的数字，哪些组将在其他组之前（如果两个类别具有相同的 ordinal，游戏将选择其中一个先出现，哪个获胜是不确定的）。"name" 是组的本地化名称，将在该类别的工作坊 UI 中显示。

然后，不要忘记将我们的 mixinto 添加到 mod 的清单中（记得在需要的地方添加逗号）：

  ...
  "mixintos" : {
     "stonehearth/jobs/mason/recipes/recipes.json" : "file(recipes/recipes.json)"
  }
  ...

如果您添加自定义制作职业，其配方不需要通过 mixinto 添加，而是通过组件的属性添加。

最后，让我们测试石匠是否可以使用 debugtools 制作我们的配方：

  • 开始新游戏。
  • 选择一个炉民。
  • 打开默认控制台并输入 promote_to mason，然后按 Enter。
  • 使用物品印章生成石匠工作台和一些石块，或者只是让炉民采集一些石头。
  • 打开石匠工作台并检查配方是否存在并正确显示图像和文本：
  • 我们的石匠没有足够的等级来制作它，所以我们 shift+点击他，然后点击"Level Up"。
  • 现在我们可以命令制作我们的新物品并检查 iconic 是否正确生成。

使用 SHED 添加配方

正如在创建物品部分中解释的那样，我们可以一步创建物品及其配方。 否则，我们可以取消选中克隆弹出窗口中我们不想要创建的任何不需要的文件，只保留配方文件被选中。

在编写本指南时，克隆基础 mod 中现有职业的配方列表或单个配方不会使它们出现在我们 mod 的树视图中，我们需要在 SHED 之外编辑它们。当然，mixintos 也必须在 SHED 之外设置。如果我们用其生产的物品克隆配方，物品将显示在树视图中，因为将创建别名，因此我们将能够在 SHED 内编辑它。
