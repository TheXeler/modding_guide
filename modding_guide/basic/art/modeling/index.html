体素建模工具

团队使用 Qubicle Constructor 1.0 为游戏创建 3D 模型。这是一个付费程序。当前版本的 Qubicle（在编写本指南时）是 Qubicle 3.0。

至于免费工具，Magicavoxel、StoneVox、VoxelShop 和 Goxel 应该可以帮助您，如果您无法负担付费程序的话。

模组作者的常用流程

通常，模组作者会在游戏文件中找到他们想要的资产，将其导入到他们选择的程序中，编辑它，然后再次导出以在他们的模组中使用。 这是最容易做的事情，因为它有助于弄清楚比例和位置。比较不同的物品也会有所帮助。

基本比例是：

  • 地形、树木和建筑块的比例为 1。
  • 您使用 3D 程序创建的其余物品将自动获得 0.1 的比例（1/10 个方块）。

您可以通过检查炉民来轻松看到这一点。炉民的脸宽 13 个体素，游戏中的炉民占据略多于 1 个地形方块。 一根木柴在 Qubicle 内是 10x10x10 体素，在游戏中正好是 1 个方块。

您可以使用较少的体素制作模型，然后在游戏中使用 JSON 放大或缩小它。体素的数量将取决于您想要实现的细节级别。 然而，过多的体素/细节对性能不利。

其他关于高性能模型的提示是：

  • 用体素完全填充模型内部（如果从外部看不到模型内部）。

  • 在模型完成后减小矩阵/层的大小。这将节省模组大小，不幸的是游戏不支持 QB 模型的压缩，因此这是目前减小模型大小的唯一方法。

关于矩阵/层

矩阵指的是 3D 模型中的不同层。这是来自 Qubicle 的术语，对于其他程序，您可能会看到它被称为“层”。

如果您的模型有不同的可移动部分，每个部分都必须在不同的矩阵中。

矩阵的大小最终并不重要，但为了更容易建模，您可以在模型周围留出一些空间，以便在需要时可以添加更多体素（您不能在矩阵/层之外添加体素，它就像一个边界框）。不过，这可能因您选择的体素编辑器而异。

在处理模型时，最好使每个矩阵的大小完全相同，以便各部分正确对齐。然后，在导出最终模型之前，您可以优化每个矩阵的大小，以便文件不会太大。

通常，您的模型会接触矩阵的底部，因为这会使其与游戏中的地板对齐，但也有一些例外。

如果您的模型将是一个只有 1 个部分的静态对象（例如，椅子），您可以使用 Magicavoxel。然而，Magicavoxel 不适合用于打算制作动画的模型，如盔甲、生物、门或某些类型的工作台。希望在未来的 Magicavoxel 版本中，我们可能会获得在同一 .qb 文件中导出多个层的选项。与此同时，您可以使用其他支持层的免费编辑器，或将所有文件作为 JSON 文件中模型的一部分使用。

模型对齐

您可能会看到某些对象（如武器或壁挂物品）在体素编辑器中打开其模型时对得很奇怪。这是因为附加点对应于程序中的中心（在 0,0,0）。在 Qubicle 中，大多数 Stonehearth 模型没有居中，而是扩展到一个象限中。

然而，有些模型要求您在选择的程序中围绕 (0,0,0) 居中对齐它们。对齐不良的模型在放入游戏后会出现在错误的位置，因此请务必仔细检查这一点。

这就是为什么大多数模组作者导入类似的对象并在其上编辑的原因之一，以便居中正确，并且他们不必在 JSON 文件中过多地调整值。

由于模型可以具有奇数个数的体素，它们可能看起来稍微偏离中心，但这不是问题，因为我们将调整 JSON 文件中的中心。有关在 JSON 中调整模型原点，请参见此页面。

命名矩阵/层

这对于盔甲或重用现有 rigs 的生物等物品非常重要（例如，如果您想通过重新换皮现有种群来创建自己的种群/派系）。

对于其他物品，名称不是那么重要，但对于动画物品，请确保它们的矩阵都已正确命名。如果您编辑了现有模型，它们可能已经具有正确的名称（除非您导入了其他对象作为参考，在这种情况下，它们可能会更改为我们不想要的内容）。

将模型导入 Qubicle

将现有 .qb 模型快速导入 Qubicle 的一种方法是将它们拖放到主建模区域。您可以将多个模型导入到同一场景中，但根据您使用的 Qubicle 版本，矩阵的名称可能会更改为我们不想要的内容。

使用 Qubicle 导出

导出 .qb 模型所需的设置：

  • 必须取消选中压缩。游戏无法读取压缩的 QB 模型。
  • Z 轴必须是右手坐标系。这将使模型与游戏内轴正确对齐。
  • 必须取消选中单独导出，否则每个矩阵将被导出到单独的文件中。

如果您碰巧使用较旧版本的 Qubicle，导出设置大致相同。在 Qubicle 1 中，颜色格式选项应为“RGBA”，并且只有在您希望将所有矩阵合并为一个时才应选中“Single Object”选项。取消选中它以在同一导出文件中保持矩阵分离（例如，用于动画）：

使用 Magicavoxel 建模物品

免责声明：在编写本指南时，最新版本的 Magicavoxel 是 0.99.1（alpha）。使用此版本，一些旧功能不可用，但我们可以在模型中使用层，这以前是不可用的。不幸的是，当导出为 .qb 时，每个层将被导出到不同的 .qb 文件中，因此您可能希望在同一层中创建物品，然后在 Voxelshop 或 Goxel 中将其拆分为不同的层。

将模型导入 Magicavoxel

这个程序可能有点挑剔，因此这里有一些将模型导入最新版 MagicaVoxel 的说明：

  1. 将 .qb 模型拖放到调色板区域。这将导入调色板。
  2. 再次将 .qb 模型拖放到模型区域。这将导入模型并应用我们刚刚导入的正确调色板。

导入时也不会保留层，它们将合并为一个对象。

使用 Magicavoxel 导出

使用 Magicavoxel 0.99.1，您可以点击导出（在右下角）并选择“qb”。 这将导出世界中存在的所有对象，每个对象在一个不同的文件中。 目前没有选项可以将所有对象（层）导出到单个文件中。

使用 Voxelshop 建模物品

许多模组作者在 Magicavoxel 中建模，然后导出模型并将其导入 Voxelshop 以将其拆分为不同的层（如果模型需要有多层）。 为了提高效率，请注意在 Magicavoxel 中工作时层的大小，以便您知道在 Voxelshop 中使用多大的边界框。

Voxelshop 将在导出时优化层的大小，这在为 Stonehearth 建模具有多个部分的物品时可能不是我们想要的，因此我们需要正确使用程序提供的边界框。

将模型导入 Voxelshop

您可以通过'File->Import'选项导入模型，但只会显示一个（与 Qubicle 不同，在 Qubicle 中我们可以拖放多个模型，并且所有模型都将显示在同一场景中）。Voxelshop 将正确显示导入模型的颜色和层，就像 Qubicle 一样。层列表中可能有一个额外的空层，但您可以删除它，或者在重新导出模型时，只要它是空的，Voxelshop 就会忽略它。

将现有模型导入 Voxelshop 时，它可能出现在边界框之外。这是因为边界框中心在 0,0,0，而模型对象角在 0,0,0（因为模型是如何使用 Qubicle 导出的）。

因此，首先，选择所有模型并将其移动到边界框内（或逐层进行），确保它在 3 个视图中居中。然后在导出时，确保使用边界框作为矩阵。这将保持所有部分的边界框大小，以便它们彼此正确对齐。

使用 Voxelshop 导出

在 Voxelshop 中选择导出为 .qb 格式时，不同导出设置的描述将告诉您是否需要选中或取消选中 Stonehearth 的选项。但以防万一，以下是选项及其必须具有的值的列表：

  • Use compression（使用压缩）: unchecked（未选中）
  • Use bounding box as matrix（使用边界框作为矩阵）: checked（选中）
  • Use origin as zero（使用原点为零）: unchecked（未选中）
  • Use visibility mask encoding（使用可见性掩码编码）: unchecked（未选中）
  • Use right handed z-axis orientation（使用右手 Z 轴方向）: checked（选中）

使用 Goxel 建模物品

要在 Goxel 中查看每个层的大小，请选择层，然后点击“Bounded”。然后您可以调整层的大小和原点。

将模型导入 Goxel

转到'File -> Import... -> qubicle'并选择您要导入的 QB 文件。

使用 Goxel 导出

转到'File -> Export As... -> qubicle'。

像 Magicavoxel 一样，Goxel 没有用于 Qubicle 文件的导出选项。
