重用现有动画

如果我们创建一个与 Stonehearth 中已存在实体具有相同 rig 的 mob（例如重新换皮），我们可以重用现有动画。

例如，许多小型生物（兔子、浣熊、松鼠……）重用相同的骨骼和动画。这是 squirrel.json 文件中的 render_info 组件：

  "render_info": {
     "animation_table": "file(/data/rigs/entities/critters/rabbit)",
     "scale": 0.07
  }

如您所见，"animation_table" 指向 rabbit.json 骨骼。松鼠模型的层名称与兔子模型相同（部分位于相同位置 - 它们可以使用相同的枢轴移动和旋转），因此我们无需额外工作即可重用所有兔子动画。

我们还可以缩放模型，动画仍然可以正常工作。

为现有 rigs 创建自定义动画

为了为现有 rigs 添加新动画（例如，为炉民添加新的空闲动画），我们需要在清单中使用覆盖，因为没有索引可以 mixinto 来添加动画。

这里提供了一个示例。我们覆盖 stonehearth mod 中不存在的文件，这将表现为好像我们将文件注入到适当的文件夹中。

使用现有 rigs 制作动画

这是在 Blender 中为现有骨骼创建动画的过程。假设您已经设置了 Blender 插件 并准备了场景。

将模型导入 Blender

这些说明假设您已经将模型导出为 .obj 格式。

  1. 在 Blender 中准备好场景后，转到'File->Import...'点击"Wavefront (.obj)"

  2. 选择 .obj 文件。确保 .mtl 和 .png 文件也与 .obj 在同一文件夹中（它们是导出为 .obj 时创建的附加文件）。

  3. 点击'Import File'。

  4. 如果您看不到导入的模型，请使用鼠标滚轮缩小视图（如果您已经准备好了场景，则不应该出现这种情况）。

  5. 如果模型显示为全白色，请点击此处的小圆形图标并将其更改为"Texture"：

  6. 检查对象是否每个部分都分开，以防导出为 .obj 时出现问题。当右键单击模型的不同部分时，只有该部分应显示橙色轮廓，其名称应出现在场景的左下角：

加载 rig 文件

导入 .obj 后，让我们导入并准备骨骼和 meta 文件。如果您要为现有的 stonehearth 实体制作新动画，请从 stonehearth/data/rigs 文件夹获取骨骼。您可以将 rig 文件复制到其他位置以更方便，以便将它们与您的 meta 文件放在一起：

  1. 转到'File > Import > Stonehearth (.json)'

  2. 找到骨骼 JSON 文件。在我们的例子中，我们将使用男性炉民模型，因此我们搜索男性骨骼。选择它，然后点击'Import File'：

  3. 现在您应该在场景菜单的"Stonehearth UI"部分看到骨骼文件的名称，这确认我们加载了骨骼文件。

  4. 现在我们再次点击'File > Import > Stonehearth (.json)'并选择适当的 meta 文件。它也应在 Scene 菜单中显示为已加载。

  5. 点击"Prepare Skeleton"按钮。如果没有错误，辅助骨骼应自动创建，并且模型上的骨骼/网格应自动相互父级连接。

    如果出现问题，"Skeleton"旁边的圆圈将变为红色。在这种情况下，您可以点击"Reset:"部分中的 Skeleton 或 Meta 按钮以卸载这些文件，手动检查它们并尝试重新加载它们。

    如果骨骼正确加载，圆圈应变为绿色，但如果不是（也不是红色），请选中"Debugging info (on/off)"复选框以展开调试信息。那里将提到是否有缺失或不应该存在的对象，并将说明骨骼和 meta 文件是否正确加载。

    例如，这里插件检测到一些名称不匹配。查看层次结构，结果发现肩膀的名称中有 _0：

    因此我们可以点击它们，转到对象菜单（一个橙色立方体图标），然后在那里重命名它们（按 Enter 以保存名称）。然后我们还必须在对象数据菜单中重命名它们（一个倒三角形图标）。

    如有疑问，您可以重置骨骼和 meta 文件并重新导入它们，但通常最好从干净的场景重新开始，因为错误可能携带了一些不良信息。

    通常，当出现问题时，是由于 meta 文件或骨骼的名称/数量与预期值不匹配。在继续之前，请确保圆圈是绿色的。如果您遇到问题，"Debugging Info"部分可能会提供一些有关问题的提示。

  6. 现在点击"Prepare Animation"。如果没有错误，模型应移动到场景中网格的中心，脚部与地面对齐。

    与之前一样，如果"Animation"的圆圈变为红色，请检查调试信息以查看发生了什么。如果无法将整个模型移动到场景中心，则骨骼/网格之间的关系可能有问题。检查是否加载了 meta 文件，或者关系是否正确创建。

    在开始动画工作之前，请确保已将模型居中。否则，动画在 Stonehearth 中运行时可能不会按预期执行。因此在继续之前，请确保圆圈是绿色的。

关于 meta 文件

meta 文件的目的是添加到骨骼文件提供的信息中。它们允许 Blender 了解实体各部分之间的关系（不仅仅是它们的枢轴）以及辅助骨骼的大小。插件使用它们使动画制作更容易。

兔子的示例 meta 文件：

  {
     "meta": {
        "helperBone": ["ATTOVERCOG", 3.500000, 5.000000, 3.000000, -3.000000],
        "helperBone": ["root", 3.500000, 5.000000, 3.000000, -3.000000],
        "hierarchy": ["tailEnd", "tail"],
        "hierarchy": ["leftBackPaw", "leftLeg"],
        "hierarchy": ["rightBackPaw", "rightLeg"],
        "hierarchy": ["rightEar", "leftEar", "ATTOVERCOG", "head"],
        "hierarchy": ["rightLeg", "leftLeg", "tail", "head", "rightPaw", "leftPaw", "torso"],
        "hierarchy": ["torso", "root"],
     },
  }

meta 文件是一个 JSON 文件（有重复的键，但我们不在乎，因为这只是为了让插件检索更多信息）。它包含这些字段：

  • "meta" -- 信息的容器键。也代表这是一个 meta 文件。包含许多这些类型的条目：

    • "helperBone" -- 代表一个辅助骨骼。数组的第一个值是辅助骨骼的名称（如在骨骼文件中定义），然后接下来的四个值提供有关辅助骨骼大小的信息（x, z, head, tail）。

      X 和 Z 将代表它在 Blender 中的宽度和高度，head 和 tail 用于其从原点/枢轴的深度（因此我们可以为"mainHand"辅助骨骼设置长棍，例如）。它有助于在 Blender 中动画化它们，无论我们将在游戏中附加什么到它们上面。

      定义它们时要小心，因为错误的值可能会导致在 Blender 中准备骨骼时出现问题。

    • "hierarchy" -- 代表骨骼/网格之间的层次结构。每个数组中的最后一个骨骼将是数组中提到的其余骨骼的父级，因此当我们移动或旋转它时，子骨骼将跟随父级。确保所有骨骼都是 "root" 的后代（参见上面的示例）。

我们可以从 Blender 创建 meta 文件，或重用现有的（在编写本指南时，大多数 meta 文件已丢失或已过时）。如果您只需要定义一些层次结构或添加一些已在骨骼文件中定义的辅助骨骼，也可以手动设置它们。

可以在没有 meta 文件的情况下创建动画，但如果加载/导入一个会更容易和更方便。只需确保它是适合您要编辑的模型的适当文件。虽然 meta 文件可以在性别之间交换（如果它们共享相同的骨骼层次结构），但它们不能在不同实体之间交换。

制作动画

一旦您准备好了动画，就该开始制作动画了。在我们的示例中，我们将提供简单的步骤来快速为 Stonehearth 制作动画，但如果您熟练掌握 Blender，您可以随意使用其动画工具。只需记住，对于 Stonehearth，我们只能定义模型整个部分（网格）的位置和旋转（插件尚不支持缩放），而不是单个顶点/平面。

  1. 首先，您可能希望将视图切换为正交视图。要做到这一点，请按 NumPad 5 或转到时间线上方的 View 菜单（主场景下方的栏）并点击'View Persp/Ortho'。您应该在场景视图的左上角看到'User Ortho'。

    • 要旋转相机，请按住中键（鼠标滚轮）并移动鼠标。
    • 要在场景中平移，请执行相同操作，但首先按住 Shift。

  2. 然后，决定您希望动画有多少帧，并在时间线的'End'文本框中输入它们（窗口底部的灰色宽部分），然后按 Enter 确认。如果需要，您可以稍后更改它们。您应该看到灰色条根据您的值增加或减少：

    Stonehearth 中的动画以每秒 30 帧的速度播放。通常，我们希望动画简短，但有时如果动作旨在缓慢（例如呼吸、行走……），它们需要较长。您可以通过将鼠标悬停在时间线上并使用鼠标滚轮滚动来放大/缩小时间线，然后底部有一个滚动条可以左右移动它。这将使选择帧更容易。

  3. 现在，通过点击时间线的开头来选择第一帧。要为模型摆姿势，这里有一些快捷键：

    • 右键单击身体部位以选择它。

    • 通过拖动每个轴的箭头或按一次 G，然后移动鼠标来移动它。要沿特定轴移动，请按 G，然后按 X/Y/Z，然后输入数字（可以是负数并包含小数）。

      请注意，在 Blender 中 Z 是向上，Y 是深度。制作动画时不用担心，插件会在必要时负责调整它。

    • 通过按一次 R，然后移动鼠标来旋转它。要在特定轴上旋转，请按 R，然后按 X/Y/Z，并可选地输入代表度数的数字（可以是负数并包含小数）。

    • 要缩放它，与移动或旋转相同，但使用 S。在编写本指南时，插件尚不支持缩放。

    • 您可以通过选择一个对象，然后按住 Shift，并右键单击选择另一个对象来选择多个对象。

    • 如果您需要更高的精度，请按 N 打开隐藏菜单，您将看到数字形式的变换/旋转/缩放，可以手动输入：

    • 要取消正在进行的变换，请在场景的任何地方右键单击一次。如果您希望撤消更改，也可以使用 Ctrl+Z。

    • 要确认您的更改，请在任何地方左键单击。

    • 您可以使用小键盘键从不同侧面查看场景（前、后、上等）。还有一些方法可以打开多个场景以同时打开不同的视图。

  4. 为您的动画设置初始姿势后，从左侧菜单中选择'Animation'选项卡：

    • 按两次 A 以选择所有骨骼（所有内容应以橙色轮廓选中）。
    • 在 Animation 菜单中，按'Insert'并点击'LocRotScale'。这将把当前姿势保存为时间线中的关键帧：
    • 现在选择时间线中的最后一帧并执行相同操作。您应该在插入关键帧的地方看到黄色标记。
    • 尝试使用时间线下方的播放按钮播放您的动画。这是一个静态姿势，但这样您的动画以相同的姿势开始和结束。

  5. 选择不同的帧（通常在时间线的中间某处）并再次为模型摆姿势。如上所述保存关键帧并按播放。Blender 将插值姿势，创建动画。

    您可以选择现有关键帧并使用'Remove'按钮删除它（确保首先选择所有身体部位！），或者编辑模型的某些部分并再次保存它（将被覆盖）。

  6. 继续插入（或删除）关键帧，直到您对结果满意为止。您可能需要延长或缩短时间线。

导出带有骨骼偏移的动画

想象您已经完成了男性炉民模型的动画，并希望为女性创建相同的动画（假设两者具有相同的骨骼和骨骼名称，但骨骼的枢轴不同 - 例如，女性炉民稍矮，因此她的骨骼位置较低）。

使用 Blender 插件，我们可以轻松地将偏移应用于动画，这样我们就不必从头开始为女性创建动画。

在编写本指南时，女性炉民骨骼与男性不同，但您可以简单地为其添加眉毛骨骼以便导出带有偏移的动画 - 游戏稍后将忽略动画中的那些骨骼（不过请确保在她的骨骼副本上工作）。

假设您刚刚导出了男性的动画，并且仍在 Blender 中加载了男性模型、rig 和动画的步骤：

  1. 导入女性骨骼，确保选中左下角的"Import skeleton for offset"选项：

    确保 Scene 菜单中"Prepare Skeleton"按钮上方的"Offset calculated"字段已更改。如果没有，则未正确导入（骨骼和骨骼名称应匹配）。

  2. 再次导出动画，选中"Export taking offset into account"：

  3. 要检查动画是否正确导出，您可以重新启动 Blender，导入女性的 .obj 文件、女性骨骼/meta 文件以及您刚刚导出的动画，并播放它（或者如果您已经准备好所有效果，只需在游戏中尝试它）。

导入动画

您可以以与骨骼或 meta 文件相同的方式从游戏中导入动画文件。插件中还有一个按钮可以删除导入的动画。

由于它们描述了所有帧，因此很难使用它们，但您可能能够删除不需要的帧并保留几个姿势以在动画中使用。

在导入动画之前，请确保点击"Prepare Skeleton"和"Prepare Animation"，并且两者都有绿色圆圈。否则，导入动画后内容看起来会损坏。
