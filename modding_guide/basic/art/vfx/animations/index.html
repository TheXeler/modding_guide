效果文件

效果 JSON 文件中动画轨道的示例：

  "animation": {
     "type": "animation_effect",
     "start_time": 0,
     "animation": "chop.json",
     "loop": true
  }

这些是我们可以在动画轨道中找到的字段：

  • "type" : "animation_effect" -- 轨道的类型。动画效果。
  • "animation" -- 要播放的动画文件的名称。这里我们只需要说文件的名称（例如，work.json）。我们必须正确设置实体的 rig（骨骼/装备），因为它将有一个属性来定义存储此实体动画的文件夹，以便游戏可以仅通过名称找到它们。
  • "loop" -- 布尔值（默认为 false）。通常对于播放恒定动画或姿势的实体（如某些工作台和容器），它将为 true，而对于只打算播放一次的动画，它将为 false。AI 播放的某些动画仍将此设置为 true，例如吃、砍伐或呼吸动画，但通常要小心，因为如果配置不正确，循环轨道可能会导致 AI 动作卡住。
  • "start_time" -- 可选字段（默认为 0）。动画应开始的时间（以毫秒为单位），相对于整个效果。
  • "end_time" -- 可选字段（默认为动画的自然持续时间）。动画应结束的时间（以毫秒为单位）。如果有其他更长的轨道，最后一帧将被保持直到整个效果结束。
  • "speed" -- 可选字段（默认为 1，最小值为 0.01）。动画应播放的速度。
  • "blend_time" -- 可选字段（默认为 200）。动画混合将持续的时间（以毫秒为单位）。
  • "response_frame" -- 可选字段。仅用于将在炉民对话中播放的动画。它是此动画中下一个说话者开始说话的帧。与渲染动画没有真正关系，它只是 Lua 代码的标志，让另一个人开始动画。

隐藏骨骼和附加物品轨道

与动画轨道一起，我们有时会看到这些其他类型的轨道："hide_bone" 和 "attach_item_effect"。

隐藏骨骼轨道的示例：

  "hide_carrying" : {
     "type": "hide_bone",
     "start_time": 0,
     "end_time": 0,
     "bone": "carry"
  }

我们有这些属性：

  • "type" : "hide_bone" -- 轨道的类型是隐藏骨骼。当播放此动画时，此轨道将隐藏实体骨骼中的一个骨骼。用于在吃/睡觉动画等中隐藏武器等物品。
  • "bone" -- 要隐藏的骨骼的名称。如果骨骼是不可见的（辅助骨骼）并且在此动画运行时没有附加任何东西，我们不需要为其向效果添加 "hide_bone" 轨道。
  • "start_time" -- 可选字段（默认为 0）。骨骼应开始保持隐藏的时间（以毫秒为单位）。
  • "end_time" -- 可选字段（默认为效果的持续时间）。骨骼应再次可见的时间（以毫秒为单位）。如果效果循环，请不要使用。

附加物品轨道的示例：

  "grab_axe" : {
     "type": "attach_item_effect",
     "start_time": 0,
     "end_time": 0,
     "item": "stonehearth:worker_axe",
     "bone": "mainHand"
  },

  "grab_tool" : {
     "type": "attach_item_effect",
     "start_time": 0,
     "end_time": 0,
     "item_in_slot": "mainhand",
     "bone": "mainHand"
  }

我们有这些属性：

  • "type" : "attach_item_effect" -- 轨道的类型是附加物品。当播放此动画时，此轨道会将物品附加到实体骨骼的一个骨骼上。

  • "bone" -- 要将物品附加到的骨骼的名称。

  • "start_time" / "end_time" -- 可选字段（将默认为效果的持续时间）。与 hide_bone 轨道相同。

  • "item" / "item_in_slot" -- 我们必须使用其中一个字段来确定要附加的物品：

    • 对于 "item"，我们只需指定一个 URI。物品将仅在动画持续时间内附加到骨骼。

      您可能会看到对于 promote.json 效果，"item" 是 "{{talisman}}"。如果我们通过 AI 为其设置值，我们可以使用这种用双大括号包围变量名称的语法（在这种情况下，您可以在 stonehearth/ai/actions/change_job_using_talisman_action.lua 文件中看到它）。

    • 对于 "item_in_slot"，我们必须指定一个 装备槽。当前装备在该槽位中的任何物品都将显示为在动画持续时间内附加到指定的骨骼。

动画文件

现在让我们讨论制作动画涉及的不同文件。

在 stonehearth mod 中，它们组织在 stonehearth/data/rigs 的子文件夹中。例如，对于炉民，它们在 stonehearth/data/rigs/entities/humans 中。我们通常有 2 个文件夹（effects 和 animations），然后每个 骨骼 有一个 JSON 文件，每个 姿态 有一个 JSON 文件。

对于炉民的情况，我们制作了一个 skeletons 文件夹来包含男性和女性骨骼文件，并且除了 postures.json 之外还有几个 JSON 文件来存储有关对话和幸福感（情绪）的数据。

骨骼

骨骼或 rig 是一个 JSON 文件，定义将移动实体每个部分的骨骼。

小型 rig 文件的示例：

  {
     "type" : "rig",
     "skeleton": {
        "ATTOVERCOG": [15.0,-15.8389449183,50.7871090782],
        "door": [7.0,15.0,19.0],
        "frame": [15.5,15.0,0.0],
        "head": [15.0,-14.9748535817,23.3160510152],
        "root": [15.0,15.0,0.0]
     },
     "animation_root" : "file(animations)",
     "effects_root": "file(effects)"
  }

我们可以看到这些属性：

  • "type" : "rig" -- 此文件代表一个骨骼。

  • "skeleton" : {} -- 这里我们定义实体的骨骼。每个键是 QB 模型中矩阵/层的名称（我们将每个可移动部分分离到不同的层），值是一个表示枢轴 X、Y 和 Z 坐标的数组。模型的该部分将以该枢轴为原点进行移动/旋转。

    我们还可以包括不是模型矩阵名称的骨骼名称。例如，大多数骨骼将有一个与地板和实体中心对齐的 "root" 骨骼。它代表实体在地面上的位置，因此很少移动，但通常在您的动画程序中所有其他骨骼都父级连接到它，以便更容易制作动画。

    对于炉民，我们还有一个 "bodyPosition" 骨骼，用于移动整个身体而不移动 "root"。"ATTOVERCOG" 骨骼用于将图像附加到它上面，例如健康指示器或对话气泡。还有 "mainHand" 和 "offHand" 骨骼，用于将武器或其他物品附加到炉民的手上。

    通常，您可以拥有任意数量的辅助骨骼（但骨骼 + 辅助骨骼的总数不能超过 50），并且它们不需要存在于所有动画中（例如，"carry" 骨骼并非在所有动画中都使用）。除非我们将东西附加到辅助骨骼上，否则它们在游戏中是不可见的。

  • "animation_root" -- 指向我们保存此骨骼所有动画的文件夹。

  • "effects_root" -- 指向我们保存此骨骼所有效果（带有动画轨道）的文件夹。

  • "postures" -- 可选字段。指向 postures.json 文件以定义实体的 姿态。

  • "conversation_effects" -- 可选字段。指向 conversation_effects.json 文件，定义对话的效果（仅当实体可以进行对话时使用 - 如炉民或宠物）。查看现有的作为示例。

  • "moods" -- 可选字段。指向 moods.json 文件，根据实体的情绪应用动画（仅当它们可以像炉民一样高兴/悲伤时使用）。它包含类似于姿态文件中的动画映射。

您可能会在一些 rigs 中看到 "weighted_effects" 字段。它不再使用，相反，我们在 entity_data 中有一个 "stonehearth:idle_effects" 字段，用于根据权重随机化空闲效果。

您可能还会在一些 rigs 中看到 "collision_shape" : {} 字段。它也不使用，我们在实体的 mob 组件中定义 碰撞类型。

重要！要将骨骼关联到实体，我们需要在实体 JSON 文件的 render_info 组件中定义 animation table。例如：

  "components": {
     "render_info": {
        "animation_table": "stonehearth:skeletons:humanoid:male"
     }
  }

"animation_table" 属性必须指向骨骼 JSON 文件。

姿态

姿态是为某些情况定义不同动画的一种方式。它们通过 posture 组件进行管理。

人类 postures.json 的简化示例

  {
     "stonehearth:carrying" : {
        "idle_breathe" : "carry_idle",
        "run" : "carry_walk"
     },
     "stonehearth:sitting" : {
        "idle_breathe" : "sitting_idle",
        "idle_sway" : "sitting_idle",
        "idle_look_around" : "sitting_idle"
     },
     "stonehearth:sitting_on_chair" : {
        "eat" : "sitting_on_chair_eat"
     },
     "stonehearth:patrol" : {
        "run" : "patrol_walk"
     },
     "stonehearth:have_shield" : {
        "combat_1h_idle" : "combat_1h_idle"
     },
     "stonehearth:swimming" : {
        "idle_breathe" : "idle_swim",
        "idle_sway" : "idle_swim",
        "idle_look_around" : "idle_swim",
        "run" : "run_swim"
     }
  }

在此文件中，我们定义姿态，并将动画映射到其中的其他动画。在 Lua 代码中，我们调用 radiant.entities.set_posture(entity, posture)（或使用 'stonehearth:set_posture' AI 动作）将姿态更改为此处定义的键之一。

然后，当我们在实体处于该姿态时播放为该姿态定义的任何动画时，实际播放的动画将是映射中分配的动画。

例如，当炉民游泳时，他们会获得一个将其姿态更改为 "stonehearth:swimming" 的增益效果。如果我们然后告诉他们播放 "idle_breathe" 动画，他们将改为播放 "idle_swim" 动画。

动画

最后，让我们描述动画文件。像其他一切一样，它们只是 JSON 文件。我们将它们全部放在相应 rig 的 animations 文件夹中，以便可以在 骨骼 中引用该文件夹（没有索引可以 mixinto 动画）。通常它们与引用它们的效果文件具有相同的名称，即使效果文件有其他轨道（如声音、粒子效果等）。

每个动画文件包含 3 个字段：

  • "type" : "animation" -- 此文件代表一个动画。

  • "total_frames" -- 可选字段。包含帧数的字符串。似乎在任何地方都没有使用，但您可以使用它来记住动画有多少帧。

  • "frames" : [] -- 包含动画帧的数组。需要注意的一些要点：

    • 每个动画必须至少有 2 帧。

    • 每帧是一个 JSON 对象，包含骨骼中每个骨骼的位置、旋转和缩放比例。门 rig 的一帧示例：

      {
   "root": {
      "pos": [0,0,0],
      "rot": [1.0,0.0,0.0,0.0]
   },
   "frame": {
      "pos": [0,-5.5,0],
      "rot": [1.0,0.0,0.0,0.0]
   },
   "rightDoor": {
      "pos": [-16,0,10],
      "rot": [1.0,0.0,0.0,0.0]
   },
   "leftDoor": {
      "pos": [16,0,10],
      "rot": [1.0,0.0,0.0,0.0]
   },
   "head": {
      "pos": [3,-22,27],
      "rot": [1.0,0.0,0.0,0.0]
   }
}

    • "pos" 数组表示 X、Y 和 Z 坐标。

    • "rot" 数组表示作为四元数 (w, x, y, z) 的旋转。

    • 缩放支持是在游戏后期引入的（用于动画化炉民的眼睛），因此一些现有动画可能缺少 "sca" 数组（包含 x, y, z 的缩放比例）。

为了在 Stonehearth 中为实体创建动画，我们需要一个可以导出为游戏使用的 JSON 格式的 3D 建模/动画程序。为此，人们创建了可与已知程序一起使用的插件或附加组件。您也可以创建自己的。

这些是应该适用于为 Stonehearth 制作动画的一般步骤。如果您对 Qubicle / Blender 更熟练，请随意调整您的工作流程。只要它们可以导入/导出 Stonehearth 用于 rigs 和动画的格式，您也可以使用其他 3D 应用程序。

动画工具

有几种插件可用于创建 Stonehearth 格式的动画。如果您想要更多功能，您可以为您选择的动画程序制作自己的插件。需要导入/导出的 JSON 文件格式很容易理解。

  • 3ds Max 脚本 -- 这些脚本最初由开发团队用于导出 rigs 和动画。没有关于如何使用它们的说明，并且它们可能有错误或已过时，但如果您选择的程序是 3ds Max，您可能想尝试它们。
  • Maya 脚本 -- 这些脚本在开发团队切换到 Maya 制作动画后使用。使用前请阅读 此 Readme 文件！很久以前还有一位社区成员为他们自己的 Maya 创建了脚本，但目前大多数人使用 Blender 插件。
  • Blender 插件 -- 社区成员 voxel_pirate 创建了此插件以与 Blender 一起使用。我们将在本指南中提供如何使用它的说明。

准备模型

首先，无论您选择了什么工具，都应该准备 QB 模型并导出为您的动画程序可以读取的格式。

通常，打算制作动画的模型在 Qubicle / 您的建模程序中不会居中于 (0, 0, 0)（至少对于 mobs 的模型是这样）。如有疑问，加载它们的 QB 模型并查看它是否居中。

  1. 如前所述，模型的每个可移动部分应在 .qb 文件中分割到单独的矩阵/层中。

    我们通常将所有层保持相同的大小，直到我们完成模型的迭代，然后在最后优化它们。如果在制作动画时它们看起来很奇怪，我们可能需要将更多部分分离到不同的层，并且在 Qubicle 中，当我们粘贴它们时，它们会移动到原点，因此保持相同的大小将确保我们在处理模型时它们彼此正确对齐。

  2. 将每个层命名为您将在骨骼中使用的骨骼名称（例如 head、body、leftArm 等）。我们通常使用不带下划线的小写名称，尽管我们有时使用数字。查看此页面以查看炉民身体部位是如何命名的。

    如果您打算重用现有骨骼（如炉民的 rig），请确保骨骼名称与骨骼文件中的名称匹配（不包括辅助骨骼）。Qubicle 可能在其名称中附加了额外的 _0 文本，这稍后会给我们带来一些麻烦。

    在 Qubicle 中，您可以通过选择矩阵并按 F2 来重命名它：

    对于炉民，身体、头部、眼睛（男性的眉毛）在单独的文件中，因此您需要在导出为 .obj 之前将它们合并在一起（或者分别导出然后在 Blender 中导入所有它们）。

  3. 一旦所有部分都被分割并正确命名，就该导出它们了。选择它们全部（或不选）以便考虑整个模型，并将其导出为 .obj（或您的动画程序支持的其他格式，例如 .dae）。确保在导出选项中层保持为单独的对象。

    对于 Qubicle 1，.obj 导出设置为：

    • Export As : One Object Per Matrix（每个矩阵一个对象）
    • Material : SimpleTex

    SimpleTex 设置：

    • Colormap : Use colors of selected matrices（使用选定矩阵的颜色）
    • Pixels per color : 1 pixel（每种颜色 1 像素）
    • Faces per color : 1（每种颜色 1 面）
    • Padding : unchecked（未选中）
    • Draw outlines to texture : unchecked（未选中）
    • Outline color and outline opacity : (indifferent as we didn't check the setting above)（轮廓颜色和轮廓不透明度：无关，因为我们未选中上述设置）
    • Don't overwrite existing texture map : unchecked（未选中）

    常规设置：

    • Scale factor : 1.00（缩放因子）
    • Remove lamina faces : checked（选中）
    • Remove completely surrounded cubes : checked（选中）
    • Move center to world origin : unchecked（未选中）

最后，一旦您完成了模型/rig 的迭代，您可以优化矩阵/层并将模型重新导出为 .qb，以节省 mod 中的大小。在 Qubicle 1 中，这是通过选择矩阵并点击'Modify > Optimize Size'菜单来完成的。体素周围的所有空白空间将缩小，同时保持立方层作为边界。

在其他工具中，导出准备模型的流程可能完全不同。例如，Voxelshop 不能导出为 .obj，但可以导出为 .dae（Blender 可以读取）。但是，模型部分可能在导入/导出期间移动、翻转或自行缩放，或者可能出现其他问题，因为它是不同的格式，因此您需要调整您的动画工作流程。

Blender 插件设置

我们将解释如何使用开源程序 Blender 设置动画环境。

Blender 插件由社区成员 voxel_pirate 开发。它支持导入现有 rigs、使用元文件（这是插件创建的自定义文件以反映骨骼层次结构，游戏中不使用）、导出骨骼和动画，以及偏移动画，以便我们不必从头开始为男性和女性骨骼制作相同的动画。它也可以导入现有动画，但由于 JSON 文件描述了动画的每一帧，因此很难使用它们。

  • 从这里下载最新版本的 Blender。

  • 从这里下载 Stonehearth 插件。

安装插件的说明：

  1. 打开 Blender，转到 File-> User Preferences... 并确保顶部选择了"Add-ons"，左侧菜单选择了"Import-Export"：

  2. 点击窗口底部的"Install Add-on from File..."按钮。

  3. 选择插件的 zip 文件夹，无需解压缩它（或者如果您已经解压缩它，则选择 init.py 文件）。点击右上角的"Install from File..."。

  4. 搜索'Import-Export: Stonehearth Add-On'（列表应已刷新以仅显示该内容），并选中它以启用它。

  5. 点击底部的"Save User Settings"，以便每次打开 Blender 时插件都保持启用状态，然后关闭窗口。

  6. 验证插件已加载。转到'File->Import...'和'File->Export...'并检查那里是否列出了名为"Stonehearth (.json)"的新类型。

    如果以这种方式安装插件不起作用（由于权限或其他原因），请尝试解压缩它并将文件夹粘贴到 Blender Foundation/Blender/<version_number>/scripts/addons 目录中（可能是隐藏的）。

    如果您正在更新插件，那么您可能希望从插件列表中取消选中它并从硬盘中删除它，以防止任何混淆。
