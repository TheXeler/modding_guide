粒子效果

Cubemitters 是粒子效果。按照惯例，stonehearth 的粒子效果文件（只是 JSON！）的名称以".cubemitter.json"结尾。这样我们可以将它们与可能具有相同名称的效果文件区分开来。所有现有的 cubemitters 都位于 stonehearth\data\horde\particles 目录内。

如果需要，您可以为同一个效果拥有多个 cubemitter 轨道（例如，同时播放大小粒子，如灯的效果）。

效果文件

效果 JSON 文件中 cubemitter 轨道的示例：

  "cubemitter": {
     "type" : "cubemitter",
     "start_time" : 0,
     "end_time" : 30,
     "cubemitter": "particles/sparks/hit_spark.json",
     "transforms": {
        "x" : 0,
        "y" : 1.25,
        "z" : 0,
        "rx" : 0,
        "ry" : 0,
        "rz" : 0
     }
  }

这些是我们可以在 cubemitter 轨道中找到的字段：

  • "type" : "cubemitter" -- 轨道的类型。粒子效果（"立方体发射器"）。

  • "cubemitter" -- 定义粒子设置的 JSON 文件的路径。默认情况下，它们将从 stonehearth/data/horde 读取。即，"particles/dust_poof/purple_dust_poof.cubemitter.json" 实际上将被读取为 stonehearth/data/horde/particles/dust_poof/purple_dust_poof.cubemitter.json。

    为了让游戏从我们的 mod 文件夹中读取我们的自定义效果，我们需要使用从我们 mod 的命名空间开始的绝对路径（例如 /candyland/data/horde/particles/sprinkles_rain/sprinkles_rain.cubemitter.json）。否则，游戏将尝试从 stonehearth/data/horde 目录开始读取路径。这也适用于渲染材质的路径。

  • "transforms" : {} -- 包含 "x"、"y"、"z"、"rx"、"ry" 和 "rz" 字段。这些是粒子发射器的偏移量（以世界单位为单位）和旋转（以度为单位）。这样，粒子可以从相应实体的不同位置/旋转生成。如果实体移动（例如炉民），发射器将随其移动，它是附加的。

  • "start_time" -- 可选字段（默认为 0）。此轨道应开始的时间（以毫秒为单位）（相对于整个效果）。

  • "end_time" -- 可选字段（默认为 0）。此轨道应停止的时间（以毫秒为单位）。它确定轨道的总持续时间（如果您未设置 "end_time"，cubemitter 轨道将循环）。

  • "bone" -- 可选字段。它是我们要附加发射器的实体骨骼的名称。如果我们省略此字段，发射器起始位置将是实体的根部（通常在实体的中心和地面级别）。然后根据我们在 "transforms" 字段中定义的参数移动和旋转它。

创建 cubemitter 文件

现在来看我们的 cubemitter JSON 文件的内容。请记住，您可以从 stonehearth\data\horde\particles 复制一个现有文件以开始（尝试一次更改一个值，看看它们如何影响粒子）。

每个 cubemitter 文件都有这些字段：

  • "name" -- cubemitter 效果的标识符。

  • "duration" -- cubemitter 的持续时间（以实时毫秒为单位）。

  • "material" -- cubemitter 的材质图路径（默认为 "materials/cubemitter.material.json"）。

  • "loops" -- 定义 cubemitter 是否循环的布尔值。

  • "mesh" : {} -- 可选字段。包含 "file"（引用 QB 文件）、"matrix"（QB 模型内矩阵的名称，如果只有一个可以为空）和 "offset" : {}（包含 "x" / "y" / "z" 点以偏移 QB 模型的枢轴 - 如果您的 QB 已经居中，则保留为 (0,0,0)）。这将使 cubemitter 粒子具有 QB 模型的形状。

    QB 模型在渲染时只有一种颜色，因为粒子一次只能有一种颜色。您可以在一些炉民表情、炉火效果和天气效果中看到此示例。

  • "max_distance" -- 可选字段。只有当我们离发射器的距离小于此距离（以方块数为单位）时，粒子才会生成。一旦生成，即使我们将相机移开，它们也将被渲染直到其生命周期结束（但如果我们距离更远，新粒子将不会生成）。

  • "max_particles" -- 可选字段。用于限制此 cubemitter 可以同时存活的最大粒子数。

  • "unculled" -- 可选字段（布尔值）。当为 true 时，将防止生成率随相机距离变化。

    如果相机距离发射器超过 200 个方块，默认情况下速率减半，如果我们设置了 "max_distance"，它将不再依赖于此，因此使用此字段，我们可以使速率无论距离如何都保持不变。

    当我们的效果必须从远处可见时很有用（在原版游戏中，它用于某些天气效果和泰坦杀手）。请记住，它可能会影响性能（因为即使效果不在视野中也会渲染）。

  • "realtime" -- 可选字段（布尔值）。当为 true 时，粒子将不会随游戏时间动画化，而是随实时动画化。这意味着即使游戏暂停，它们也会继续生成和移动，并且即使玩家以更高的速度游玩，它们的速度也将保持恒定。目前仅用于多人游戏中的 ping 效果（尝试在多人游戏中按住 Alt 并点击地形以查看它）。

  • "emitter_space_tiling" -- 可选字段。它使 cubemitter 在 3D 空间中“包裹”。重要的是，创建的粒子体积要与空间平铺对齐（因此直径为 80 单位的发射器需要 80 单位的包裹 - 否则会有间隙，事物不会被发射）。这用于附加到相机的天气效果。

对于 cubemitter 文件的所有以下字段，除非另有说明，否则假设 "kind" 属性的可能值为 "CONSTANT" / "CURVE" / "RANDOM_BETWEEN" / "RANDOM_BETWEEN_CURVES"。您可以在此表中检查如何为每个值定义 "values" 字段，尽管对于某些字段，它们可能略有不同（例如颜色）：

  • "emission" : {} -- 与粒子发射相关的属性：

    • "rate" : {} -- 粒子的生成率。包含一个 "kind" 字段（除了上述类型外，还可以是 "BURST"）和一个 "values" 字段。

    • "angle" : {} -- 粒子发射的角度。包含一个 "kind" 字段和一个 "values" 字段。角度必须以度表示。

    • "origin" : {} -- 发射器表面的形状，粒子将沿其随机生成。包含一个 "surface" 字段（必须是 "POINT" / "RECTANGLE" / "CUBOID" / "SPHERE" / "CYLINDER" / "CONE" 之一）和一个 "values" 字段（根据 "surface" 类型参考此表获取示例值）。您可以写 "shape" 而不是 "surface"，如果这对您来说更直观的话。

      通常，原点表面的尺寸以方块/世界单位表示。

    • "translation_x" / "translation_y" / "translation_z" -- 可选字段。粒子将根据这些字段生成到与发射器原点不同的位置。每个都包含 "kind" 和 "values" 字段。实际值（如果我们使用曲线，则不是时间的值）可以是负数，如果我们想向相反方向平移粒子（请记住，在 Stonehearth 中 Y 是上/下）。

    • "rotation_x" / "rotation_y" / "rotation_z" -- 可选字段。与平移相同，但用于不同轴的旋转。实际值以度为单位。

    • "scale_x" / "scale_y" / "scale_z" -- 可选字段。与前一个相同，但用于粒子生成时的缩放比例。粒子的缩放比例为 1 时，与一个方块/世界单位一样大。

  • "particle" : {} -- 与每个粒子相关的属性，如果您不使用它们，可以省略任何属性。请注意，对于名为 "start" 的字段，"kind" 的唯一可能值是 "CONSTANT" 和 "RANDOM_BETWEEN"。对于其余的，我们可以使用上面提到的四个值中的任何一个。

    • "lifetime" : {} -- 每个粒子应该存活多长时间，以毫秒为单位。它有一个 "start": {} 字段，里面有 "kind" 和 "values"。

    • "speed" : {} -- 可选字段。每个粒子的速度（粒子每秒移动的方块数，无论方向如何）。默认情况下，即使我们不设置任何速度，粒子也会向发射器外部移动。它有一个 "start" : {} 字段和/或一个 "over_lifetime" : {} 字段。在每个里面我们有 "kind" 和 "values"，可能的值如上所述。

      "start" 代表粒子生成时的速度，"over_lifetime" 用于在其生命周期的不同时刻改变速度。

    • "velocity" -- 可选字段。它类似于速度，但在特定方向上（例如，如果您的起始速度为 0，速度为 1，粒子将在该方向上每秒移动 1 个方块）。它具有任何 "over_lifetime_x" / "over_lifetime_y" / "over_lifetime_z"，每个都包含如上所述的 "kind" 和 "values" 字段。负值将使它们向相反方向移动。

    • "acceleration" -- 可选字段。每个粒子的速度随时间变化的程度。它具有任何 "over_lifetime_x" / "over_lifetime_y" / "over_lifetime_z"，每个都包含如上所述的 "kind" 和 "values" 字段。负值将使它们向相反方向移动。

    • "rotation" -- 可选字段。它具有任何 "over_lifetime_x" / "over_lifetime_y" / "over_lifetime_z"，每个都包含如上所述的 "kind" 和 "values" 字段。旋转的值以度为单位。

    • "scale" -- 可选字段。每个粒子的缩放比例。它有 "start"（但也可以有 "start_x" / "start_y" / "start_z"）和/或 "over_lifetime"（但也可以有 "over_lifetime_x" / "over_lifetime_y" / "over_lifetime_z"）字段。在每个里面我们有 "kind" 和 "values"，可能的值如上所述。

    • "color" -- 粒子的颜色。它有 "start"、"over_lifetime" 和/或 "over_lifetime_a" 字段，每个都有如上所述的 "kind" 和 "values"。

      颜色有一个特殊性。数组将有 3 个值，代表 RGB 颜色。它们必须以十进制格式编写（因此从 0 到 1，请查看此转换器以帮助转换它们，或使用 SHED）。

      "start" 的值包括第四个值，用于粒子的 alpha 或透明度。但对于 "over_lifetime"，它们没有（曲线中的第一个值像其他属性一样是时间）。"over_lifetime_a" 用于随时间设置 alpha。

测试粒子效果

有一种快速迭代粒子效果的方法，无需每次重新启动游戏。

将 "enable_renderer_file_watcher" : true, 添加到您的 user_settings.json 文件中，与"user_id"处于同一级别。 这将允许您在每次保存 cubemitter 的 JSON 文件时在游戏中查看更改（根据粒子的生命周期，更新可能需要一段时间）。

因此，我们可以开始新游戏，使用调试工具启动我们的效果，编辑 JSON 文件，保存它，它将更新游戏中的效果，无需重新打开它。

如何在 SHED 中创建粒子效果

在编写本指南时，如果效果不在上述目录内，SHED 将无法检测到它们。 因此，请确保您使用与 stonehearth mod 相同的目录结构（your_mod_namespace/data/horde/particles）。

转到 Effects Editor，然后选择'Cubemitters'选项卡。您可以右键单击现有的 cubemitter，点击'Clone'，选择您的 mod 作为目标 mod 并提供替换文本。一旦树刷新，您应该会看到列出的 cubemitter，如果没有，您可能需要重新加载 SHED。

当您选择 cubemitter 文件时，您应该在中间看到 JSON，在右侧看到一个 Web 编辑器：

在 Web 编辑器中编辑更容易，因为它有曲线编辑器和颜色选择器，然后使用顶部的蓝色'SAVE'按钮保存它，以便修改 JSON 文件，然后也保存 JSON 文件以确认更改。或者如果您愿意，您可以直接编辑 JSON 文件。

这里有一些解决方法，如果 Web 编辑器无法正常工作。

您会看到许多字段右侧有一个 + 按钮来添加或展开它们，以及一个 x 按钮来删除它们。

当您在下拉菜单中选择 "CURVE" 或 "RANDOM_BETWEEN_CURVES" 时，您首先需要添加一条曲线，然后您将能够在曲线编辑器中编辑它。工作方式如下：

  • 双击图表以添加点。
  • 您可以通过单击点来选择它们，然后按住并拖动鼠标中键（滚轮）来拖动点。
  • 要删除点，请右键单击它。
  • 图表右下角有撤销/重做按钮。还有一个重新 framing 按钮，用于如果您将点拖出视图，图表会移动比例，以便您可以同时看到所有点。
  • 水平线代表时间，因此它将从 0 到 1。如果您想为点设置特定值，可以在图表下方的文本框中进行（"T:" 代表时间，"V:" 代表值）。
  • 当图表表示多条曲线时，曲线之间的区域将显示为蓝色。

对于颜色，会有颜色选择器。单击它们，您将能够轻松选择不同的颜色以及透明度（但请记住上面提到的关于如何设置透明度字段的规则）。
