音效

Stonehearth 中的音效采用 .ogg 格式（以节省文件大小）。

我们可以在 Lua 中播放声音（通过运行它们所在的效果）或通过 Javascript（例如，当我们通过 UI 在商店中购买物品时）。有关如何播放音乐和环境声音，请查看此页面。

效果除了光照、动画或粒子外，还可以有音轨。效果 JSON 文件的示例音轨如下所示：

  "sound": {
     "type": "sound_effect",
     "start_time": 150,
     "min_distance": 27,
     "falloff" : 5,
     "volume": 60,
     "track" : "stonehearth:sounds:death_poof"
  }

这些是效果文件的字段：

  • "type" : "sound_effect" - 此效果轨道是音效。

  • "loop" -- 循环声音的布尔值（默认为 false）。

  • "start_time" -- 可选字段（默认为 0）。音轨应开始的时间（以毫秒为单位）（相对于整个效果）。某些声音在效果接近尾声时播放，因此这可能大于 0。

  • "end_time" -- 可选字段（默认为声音的自然持续时间）。音轨应结束的时间（以毫秒为单位）。

  • "duration" -- 可选字段。音轨的持续时间，以毫秒为单位。它不需要与声音的自然持续时间匹配。

    这四个字段的交互方式是：

    • 声音在 "start_time" 开始。
    • 如果设置了 "loop"，声音将循环播放 "duration" 时长。
    • 如果设置了 "end_time"，声音将在 "end_time" 停止播放，即使 "loop" 为 true。在这种情况下，"loop" 只是循环 "start_time" 和 "end_time" 之间的间隔（相对于效果的整个持续时间，而不是连续循环）。
    • 如果未设置 "end_time"，当声音的自然持续时间结束时，它将标记轨道为完成。

  • "min_distance" -- 可选字段（默认为 20）。声音开始可听的最小距离（以方块数为单位）。如果我们将相机移动到离播放效果的实体比此距离更远，我们将不再听到声音。

  • "max_distance" -- 可选字段（默认为 60）。声音可听的最大距离。

    注意：对于这种基于距离/位置的声音类型，请确保您的声音文件导出为单声道而不是立体声，否则会出现奇怪的回声。如果我们希望声音从远处也能听到，我们可以使用较高的值。

  • "falloff" -- 可选字段（默认为 1）。声音的衰减（在远处，声音的可听度会降低）。可以包含小数。

  • "volume" -- 可选字段（默认为 0.1）。声音的音量（从 0.1 到 100），以防我们需要调整它，因为我们的文件与游戏中的其他声音相比太响或太轻。将根据游戏声音设置中的效果音量进行缩放。

  • "pitch" -- 可选字段。应用于音轨的音高。

  • "max_instances" -- 可选字段。当许多此声音的实例可以同时运行时，这很有用。例如，如果我们希望无论附近有多少炉民在睡觉都只听到一首摇篮曲，我们会将其设置为 1。

    请注意，对于这个特定示例，当第一个炉民（开始播放摇篮曲的人）醒来时，声音将停止，无论附近是否有其他炉民仍在睡觉（他们取消了播放他们的声音，因为我们已经达到了 "max_instances"）。如果另一个炉民在第一个炉民醒来后去睡觉，那么声音将再次播放。

  • "repeat" -- 可选字段（默认为 false）。如果您的效果至少包含一个动画轨道和一个音轨，当动画循环时，我们会重新启动声音。通常情况下这是真的，无需此字段，但在极少数情况下，如果音效的持续时间比动画长，您可能希望使用它来更改其播放方式。这取决于您为轨道设置的 duration / loop / start_time 和 end_time。

  • "fade_out_time" -- 可选字段（默认为 0 / 无淡出）。声音将停止可听的毫秒数（它将在末尾淡出）。如果您的声音循环，请不要使用。

  • "track" -- 必填字段。我们要播放的声音文件的 URI，或者这个：

    "track" : {
   "type": "one_of",
   "items": [
      "stonehearth:sounds:jobs_trapper_arm_01",
      "stonehearth:sounds:jobs_trapper_arm_02",
      "stonehearth:sounds:jobs_trapper_arm_03"
   ]
}

    在这种情况下，将播放数组中的随机声音。

在 UI 中播放声音

这是在 Javascript 文件中播放声音的语法：

  radiant.call('radiant:play_sound', {'track' : 'stonehearth:sounds:ui:start_menu:shop_buy'} )

我们调用 'radiant:play_sound' 函数并传递一个包含我们要播放的轨道的 URI 的对象。

单位状态效果

炉民说话时出现在他们头顶的气泡，或者炉民失去生命值时出现的心形，都是单位状态效果。

它们是通过将始终面向相机的图像附加到炉民 rig 的骨骼（或任何实体中心的特定距离）来实现的。

这是单位状态效果的效果轨道：

  "unit_status_effect": {
     "type": "unit_status_effect",
     "material": {
        "name": "materials/billboard.material.json",
        "samplers": [
           {
              "name": "albedoMap",
              "map": "materials/thoughts/health/dead/dead_.png",
              "numAnimationFrames": 25,
              "frameRate": 20
           }
        ]
     },
     "width": 2,
     "height": 3.4,
     "xOffset": 0,
     "yOffset": 1.6,
     "duration": 990,
     "bone": "head"
  }

我们可以看到以下属性：

  • "type" : "unit_status_effect" -- 此轨道是单位状态效果。

  • "duration" -- 我们希望此效果轨道持续的持续时间，以毫秒为单位。如果您不指定持续时间，根据效果的不同，它可能会永远循环，所以要小心。

  • "material" : {} -- 必填字段。用于渲染图像的渲染材质。通常它将指向 billboard 材质（渲染始终面向相机的图像），并在"samplers"字段中传递要显示的图像（参见上面的示例）。

    如果您希望图像动画化，您可以像示例中那样设置它，除了在 "map" 中的图像外，还添加 "numAnimationFrames" 和 "frameRate" 字段（"name" 必须是 "albedoMap"）。注意图像名称以下划线结尾：该文件夹中有 25 张图像（每帧一张），从 dead_00.png 到 dead_24.png。您可以使用该约定来创建动画图像。

    如果不能使用该约定，另一种动画化方法如下：

    "samplers" : [
   {
      "name" : "albedoMap",
      "map" : "",
      "animation_frames": [
         "materials/thoughts/health/hearts/heart_4_8.png",
         "materials/thoughts/health/hearts/heart_4_8.png",
         "materials/thoughts/health/hearts/heart_5_8.png",
         "materials/thoughts/health/hearts/heart_4_8.png",
         "materials/thoughts/health/hearts/heart_5_8.png",
         "materials/thoughts/health/hearts/heart_4_8.png",
         "materials/thoughts/health/hearts/heart_5_8.png"
      ],
      "numAnimationFrames" : 7,
      "frameRate" : 8,
      "numLoops" : 1
   }
]

    在 "animation_frames" : [] 字段中指定每帧图像的路径，并使用 "numLoops" 字段指定它应该运行的次数。请记住，除非您使用绝对路径，否则这里的路径将从 stonehearth/data/horde 读取。

  • "bone" -- 可选字段，默认为 "head"。单位状态效果旨在用于动画实体，因此它们可能已经有一个"head"骨骼。您可以在此处定义不同的骨骼来附加图像。对于某些姿势实体，这指向"center"骨骼（确保骨骼存在于您实体的 rig 中）。

  • "width" -- 可选字段，默认为 3.0。宽度的缩放比例（例如，2 是图像宽度的两倍）。

  • "height" -- 可选字段，默认为 3.0。高度的缩放比例（例如，2 是图像高度的两倍）。请记住，当您缩小视图时，图像看起来会更小。如果您的图像不是完美的正方形，"height" 可能需要略大于 "width"，反之亦然，以保持比例。

  • "xOffset" -- 可选字段，默认为 0（我们将从中减去一半的宽度，以便原点位于图像的中心）。水平坐标的偏移量，用于移动图像（可以为负）。

  • "yOffset" -- 可选字段，默认为 0（如果实体被缩放，它将被缩放以调整其高度）。垂直坐标的偏移量，用于移动图像（可以为负）。与宽度和高度类似，偏移量以单位定义，而不是像素。

以下字段仅在 stonehearth/data/effects/thoughts/thought_bubble_base.json 中使用，但如果您愿意，也可以为您的效果使用它们：

  • "xScalePivot" -- 可选字段，默认为 0.5。水平缩放图像的枢轴。
  • "yScalePivot" -- 可选字段，默认为 0.5。垂直缩放图像的枢轴。这些是相对于图像的位置，因此提到的默认值指向图像的中心。
  • "detach" -- 可选字段，默认为 false。当为 true 时，图像将在创建时与骨骼分离，然后在那之后附加。默认情况下，图像直接附加到骨骼上创建。
  • "entryAnimation" : {} -- 可选字段。图像出现时的动画参数。
  • "exitAnimation" : {} -- 可选字段。图像消失时的动画参数。查看 thought_bubble_base.json 以检查用于进入/退出动画的字段。

您可以在 stonehearth/data/effects 中找到更多单位状态效果的示例，尽管那里的效果是混合的。还有一些具有动画轨道的单位状态效果，它们位于 stonehearth/data/rigs 文件夹下，但找到它们比较困难。

活动覆盖效果

当我们标记实体进行收获、掠夺等时，悬停在实体上方的 toast 通知是活动覆盖效果。

这主要用于非动画实体。活动覆盖轨道的示例：

  "activity_overlay_effect": {
     "type" : "activity_overlay_effect",
     "material" : {
        "name": "materials/overlay.material.json",
        "samplers" : [
           {
              "name" : "foregroundMap",
              "map": "materials/harvest_plant_overlay/harvest.png"
           },
           {
              "name" : "albedoMap",
              "map": "materials/thoughts/default_overlay.png"
           }
        ]
     },
     "y_offset" : -10
 }

这些是我们可以在活动覆盖效果轨道中找到的字段：

  • "type" : "activity_overlay_effect" -- 这是一个活动覆盖效果轨道。

  • "material" : {} -- 用于渲染图像的渲染材质。通常它将指向 overlay 材质，并在"samplers"字段中传递要显示的图像（参见上面的示例）。

    通常我们会有一个用于主图像的 "foregroundMap" 条目（使用透明背景制作），以及一个用于背景图像的 "albedoMap" 变量条目（通常指向 default_overlay.png，即方形气泡图像，在多人游戏中将用玩家颜色着色）。

    请记住，除非我们使用绝对路径，否则图像的路径将默认从 stonehearth/data/horde 读取。

  • "width" -- 可选字段（默认为 64）。图像的宽度。

  • "height" -- 可选字段（默认为 64）。图像的高度。与单位状态效果不同，覆盖效果的 "width" 和 "height" 以像素定义。

  • "y_offset" -- 可选字段。将图像垂直放置在实体中心上方的偏移量。与单位状态效果不同，覆盖效果的 "y_offset" 以像素定义。默认为 0，但仍会出现在实体上方，因此如果您想降低它，可以使用负值 - 小心，因为较高的负值可能会使其在某些相机位置/旋转下沉到地形中。

触发效果

在上一节中，我们讨论了 attach item 轨道的一些语法，这些语法只能在 promote 效果中看到，并且依赖于 AI 动作。promote 效果还有几个 "trigger_effect" 轨道，这些轨道在代码的其他地方没有使用（可能是因为我们可以更容易地从 Lua 触发和监听事件）。此效果除了几个触发效果轨道外，还有动画、声音和 cubemitter 轨道。

您很少会使用这个，因为可能有更简单/其他的方法来做这件事，但这里是示例：

  "promote_trigger" : {
     "type": "trigger_effect",
     "start_frame": 145,
     "info": {
        "event": "change_outfit",
        "talisman": "{{talisman}}"
     }
  }

属性是：

  • "type" : "trigger_effect" -- 此轨道是触发效果。
  • "start_frame" -- 轨道应触发的帧。
  • "info" : {} -- 关于要触发的事件的信息。它有一个带有事件键的 "event" 字段，以及一个 "talisman" 字段（这是自定义数据，也从传递给 'stonehearth:run_effect' 动作的参数中检索）。在 change_job_using_talisman_action.lua 中，我们创建一个触发函数（将其视为回调）以在运行 promote 效果时传递给它。然后根据 "event" 运行代码以提升炉民，移除护身符并通过 job 组件分配他们的工具。

使用 SHED 创建其他效果

我们已经提到了一种使用 SHED 创建效果的方法。还有另一种选择：转到 Effects Editor，在'Effects'选项卡中找到与您想要创建的类似的效果。右键单击它，点击'Clone'，将目标 mod 更改为您的 mod 并提供替换名称。

然后确认，树视图将刷新以添加您的效果。编辑它，因为替换文本可能指向不存在的文件，并将其余属性更改为您想要的。

使用此方法，您需要单独为效果添加别名，但您可以在 Manifest 视图中执行此操作：右键单击您的 mod 名称，点击'Add New Alias'，选择创建的文件，然后将出现一个弹出窗口供您编辑别名。

使用另一种方法，克隆效果文件时将生成别名，因此可能更快。您需要单独创建任何其他文件，或重用 stonehearth mod 中的一些文件。
