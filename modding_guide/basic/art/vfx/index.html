创建效果

Stonehearth 中的效果指任何视觉效果，如动画、粒子效果、光照等，也包括音效。有关颜色图和材质图（使用着色器），请参见此页面。

对于程序员，效果代码位于 radiant/modules/effects.lua，我们可以拥有的不同类型效果在 radiant/modules/effects/effect_tracks.lua 中内联，但效果的实际实现主要在 C++ 中。

在 stonehearth mod 中，我们可以在 stonehearth/data/effects 内找到游戏使用的大多数效果。对于动画效果，它们存储在 stonehearth/data/rigs 的某些子目录中。

这里有一个方便的颜色转换器，因为对于效果，它们被指定为 0 到 1 的范围，因此您必须将其十六进制/RGB 值转换为十进制（游戏的其余部分大多数时候使用十六进制值）：http://www.corecoding.com/utilities/rgb-or-hex-to-float.php

效果文件

此文件将确定位置和效果轨道。 让我们看看 stonehearth/data/poof_effect/poof_effect.json： 

  {
     "type" : "effect",
     "tracks": {
        "dust": {
           "type": "cubemitter",
           "cubemitter": "particles/dust_poof/purple_dust_poof.cubemitter.json",
           "loop": false,
           "start_time" : 0,
           "end_time" : 2000,
           "transforms": {
              "x":0,
              "y":0,
              "z":0,
              "rx":90,
              "ry":0,
              "rz":0
           }
        },
        "sound": {
           "type": "sound_effect",
           "start_time": 150,
           "min_distance": 27,
           "falloff" : 5,
           "volume": 60,
           "track" : "stonehearth:sounds:death_poof"
        }
     }
  }

效果文件是由两个键组成的 JSON 文件：

  • "type" : "effect"：我们告诉游戏这是一个 VFX。
  • "mixins" -- 可选字段，指向 mixin 或 mixin 数组，如果您想在效果文件之间重用一些信息。
  • "tracks" : {...}：这里我们声明 VFX 将具有的所有轨道。它们可以是不同的类型。 您可以随意命名每个轨道（为清晰起见，保持名称唯一）。在每个轨道内，我们声明轨道的类型和轨道的参数。在此示例中，我们有一个粒子效果和一个音效。每当游戏运行此效果时，两者都将同时播放（取决于它们的 start_time）。

设置效果轨道的 loop 时要小心。如果您在 AI 动作中播放动画并且将 loop 设置为 true，AI 动作可能无法完成并将无限期地卡住重复效果。

有不同类型的效果轨道，在本指南中讨论。最常见的是 "cubemitter"、"animation_effect"、"sound_effect" 和 "light"。

粒子和光照效果的值的类型

对于 cubemitter 或 animatedlight 文件中的每个字段，我们可以在不同类型的值之间进行选择。因此，我们将在此处为每种值类型及其约束提供一个示例，以用作指南其他部分的参考。

类型 示例值 注释
"kind" : "CONSTANT" "values" : [3] 数组内的单个值。
"kind" : "RANDOM_BETWEEN" "values" : [0, 0.1] 这里我们只能在数组中指定 2 个值。第一个必须小于第二个。
"kind" : "CURVE" "values": [ [0, 0.8],
[0.5, 0.6],
[1, 0.6] ]		 对于此类型，我们必须至少指定 2 个点。点的第一个值表示效果期间应用值的时刻（时间），从 0 到 1（因此通常第一个点是 [0, N]，最后一个点是 [1, M]）。
"kind" : "RANDOM_BETWEEN_CURVES" "values": [
[
[0, 0.4],
[0.5, 0.2],
[1, 0.6]
],
[
[0, 0.8],
[0.3, 0.6],
[1, 0.7]
]
]		 在数组内，我们必须有 2 个数组，每个数组包含一个曲线（定义类似于"CURVE"类型的值）。它们不需要具有相同数量的点。
"kind" : "BURST" "values": [ [ 250, 1, 1 ] ] 曲线数组。它将一次性生成指定数量的粒子。第一个值是实时秒数。第二个值是最小粒子数，第三个值是最大粒子数（将在最小值和最大值之间生成随机数量的粒子）。我们可以在此处为值指定多个曲线，以在指定时间爆发粒子，在它们之间创建延迟。
"surface" : "POINT" （POINT 表面不需要"values"字段，将使用效果文件中定义的位置）。
"surface" : "RECTANGLE" "values" : [1, 1, 0, 0] 数组中的值按顺序表示宽度、长度、内部宽度、内部长度。内部宽度/长度大于 0 将使粒子不是从实心矩形的任何地方发射，而是仅沿矩形的边缘发射，即在宽度/长度和内部宽度/长度形状之间。
"surface" : "CUBOID" "values" : [1, 1, 1, 0, 0, 0] 数组中的值表示宽度、长度、深度、内部宽度、内部长度、内部深度。与 RECTANGLE 一样，内部值大于 0 将使粒子在立方体的外边缘和内边缘之间的区域生成。如果更容易记住，您可以将其写为"CUBE"而不是"CUBOID"。
"surface" : "SPHERE" "values" : [1, 0, 360] 数组中的值按顺序表示半径、内部半径和角度。内部半径的含义与其他表面相同。角度（以度为单位）将使粒子不是在整个球体中生成，而仅在半球体（180）、四分之一球体（90）等形状中生成。用于创建彩虹之类的弧线很有用。
"surface" : "CYLINDER" "values" : [1, 2, 0, 360] 数组中的值表示高度、半径、内部半径和角度。含义与 SPHERE 表面类似。
"surface" : "CONE" "values" : [1, 0, 45, 0] 数组中的值按顺序表示半径、内部半径、角度、内部角度。此表面将从圆锥形状（默认向下指向）发射粒子。内部角度将使平坦区域不发射粒子（当您对 CYLINDER 表面使用内部半径时也会发生同样的情况）。角度值越高，圆锥体越扁平/短/圆（形状越不明显），因此尝试使用大半径和锐角来查看形状。

如何将默认效果附加到实体

如果您希望特定实体永久运行效果，请将其添加到实体 JSON 文件的 "components" 部分（来自 stonehearth/jobs/geomancer/geomancer_golem_workbench/geomancer_golem_workbench_ghost.json 的示例）：

  "effect_list": {
     "default": "file(/data/rigs/jobs/geomancer/golem_workbench/effects/idle.json)"
  }

如果这不起作用，请改用此语法：

  "effect_list": {
     "effects": [
        "stonehearth:effects:corruption:beetle"
     ]
  }

某些实体，如灯笼，使用 lamp 组件来指定将永久运行或仅在夜间运行的效果。

要从 Lua 运行效果，我们将执行：

  radiant.effects.run_effect(self._entity, 'stonehearth:effects:undeploy_overlay_effect', optional_parameters)

测试效果

要快速测试效果，我们可以在游戏中选择一个实体，打开默认控制台，输入 spawn_effect uri_of_the_effect 并按 Enter。

如果效果有别名，我们可以使用它（前面加上 mod 的命名空间）。对于动画实体，我们可以简单地使用具有动画的效果的名称（例如：sleep）。请注意，以这种方式测试时，某些效果可能不会按预期运行。

您还可以设置测试世界来测试您的效果。stonehearth_tests mod 中有一些您可以重用的示例。
