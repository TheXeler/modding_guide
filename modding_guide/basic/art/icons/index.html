图像大小

对于图标和配方图像，我们通常使用体素编辑器截图，并可能在插图程序中添加一些较深的轮廓（6px 黑色，40%-60% 不透明度）。这些图像的大小通常为 300x300 像素，但可以变化（确保它们是正方形）。

以下是在 Qubicle 1 中截图的方法：

  • 打开您的 QB 模型，不要触摸相机。

  • 转到顶部菜单，点击 Render，然后点击'Render Settings...'

  • 将右侧小文本框中的分辨率设置为 300x300，然后点击 OK 应用更改。

  • 在场景区域正上方的菜单栏中，点击 RES. GATE 以启用可视化分辨率门。

  • 平移相机，使模型位于其中，尝试使其尽可能大而不超出框架，但留出一些余量以使图像平衡。对于某些模型，您可能还想稍微旋转视图。

  • 现在再次转到 Render 菜单，点击'Render As...'并保存您的图像。

  • 您可能希望在 Photoshop 或类似程序中编辑它以添加边框。请注意，图像具有透明背景，如果您保存为实心背景，游戏将显示白色背景。

其他体素编辑器将有不同的截图选项。

动态肖像

对于动态肖像，例如炉民和宠物的肖像（由游戏内部拍摄），我们需要在实体的 JSON 文件中添加一些信息。 这是来自 rabbit.json，在其"entity_data"部分内： 

  "stonehearth:portrait": {
     "portrait_types": {
        "headshot": {
           "camera": {
              "pos": [
                 17,
                 0.8,
                 -27
              ],
              "look_at": [
                 0,
                 0.6,
                 0
              ],
              "fov": 45
           }
        }
     }
  }

我们添加 "stonehearth:portrait" : {}。内部信息由 portrait_renderer_service.lua 使用。目前肖像只有 "portrait_types" : { "headshot": {} }，但您可以自定义要在 UI 中显示的肖像（参见 portrait_renderer_service.lua 和使用肖像服务的 Javascript 文件，如 citizens.js、show_team_workshop.js、unit_frame.js 和 select_roster.js）。

头像的属性包括相机位置、look_at 和 fov（视野，距离相机的远近）。如果您的实体太小或太大，并且肖像在单位框架中看起来不好，您可以调整这些值。

生成的图标

游戏还会自动为思想气泡中的物品生成图标，并将它们缓存在 saved_objects/generated_icons 目录中。炉民可以谈论的一些主题没有 3D 表示，它们作为已存在的 2D 静态图像传递（例如，天气图标）。

这些图标通常取自实体的标志性形式，因为它是炉民与之交互的物品（通过携带它）。然而，已知它们会在应用了材质图的物品上显示空心表面。 如果您需要调整思想的生成肖像的相机（有时它可能会穿过物品），请为您的实体数据使用这些字段，并调整值直到获得所需的结果： 

  "stonehearth:portrait": {
      "portrait_types": {
         "icon": {
            "center": { "x": 0, "y": 0.3, "z": 0.8 },
            "rotation": { "x": 0, "y": 15, "z": 45 },
            "extra_distance": 0.5,
            "fov": 30,
            "animation": "custom.json",
            "animation_time_ms": 42
         }
      }
   }

由于图标被缓存，您可能必须清除 generated_icons 文件夹才能重新拍摄肖像。

静态肖像

对于战役对话框的肖像，我们使用体素编辑器制作的静态图像，这些不是自动生成的。 它们的大小范围从 200x200 像素到 400x400 像素，大多数为 300x300 像素。但这取决于您的选择。您还可以创建自己的遭遇战类型，使用自定义 UI 和自定义 CSS (Less)，如果图像太大或太小，可以重新缩放它。

UI 图标

如果您从 stonehearth mod 复制现有文件，您将已经拥有这些图标，但以防万一您想从头开始制作，以下是它们的大小：

  • 增益效果图标的大小为 46 x 51 像素。
  • 特性图标的大小为 60 x 40 像素。
  • 职业 perk 图标的大小为 74 x 74 像素。
  • 天气图标的大小为 100 x 100 像素。
  • 开始菜单图标的大小为 74 x 74 像素（有一些较窄的例外）。

对于制作工作坊的 UI，选择一个与您的制作者主题相似的并编辑图像。所有图像都是 .png 格式，因此很容易编辑它们。确保使用支持透明度的应用程序。

自定义游戏的其余 UI 元素也是如此。
