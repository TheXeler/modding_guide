杂项 JSON

以下是关于我们在 JSON 文件中的某些属性和组件中可以找到的一些表达式的解释：

  • 时间表达式
  • 战利品表
  • 加权集

时间表达式

在许多指南中，您会遇到具有时间表达式作为值的属性。

这些时间表达式是表示游戏内时间的字符串，将由代码解析。示例：

  "duration" : "3d"

代表 3 天。

  "duration" : "40h"

代表 40 小时。

  "duration" : "15m"

代表 15 分钟。

  "duration" : "1s"

代表 1 秒。

  "duration": "3d40h15m1s"

代表 3 天加上 40 小时、15 分钟和 1 秒。

我们可以使用"+"将这些时间表达式附加到这些时间上，如下所示：

  "delay" : "3d+40h"

这意味着“在'3 天’和'3 天 40 小时’之间选择一个随机时间量”。 我们使用这种类型的表达式为游戏增加更多的随机性。这样，事件和遭遇战不需要总是在同一时间发生，这也有助于性能，因为它们在时间上更加分散。

有些属性将以不同的方式定义时间，例如：

  "time" : "6:00"

代表早上 6 点，或

  "end_time" : 1200

代表实时毫秒数（请注意，这不是一个字符串）。

在编辑从 stonehearth mod 复制的文件时，请始终确保您为每个字段使用正确类型的值。

战利品表

当您杀死、收获或挖掘某物时获得任何东西的几率由战利品表给出。战利品表基于称为权重的东西工作。

loot_drops 组件具有以下字段：

  • "force_auto_loot": true -- 可选字段，将标记掉落物为自动拾取，无论游戏设置中的自动拾取选项如何。它仅在掉落物具有正确的 player_id 时才有效。

  • "entries" -- 一个或多个战利品表条目的字典（条目名称可以是任何字符串）。每个条目具有以下属性：

    • "roll_type"：要么是 "each_of"，要么是 "some_of"。如果省略，将默认为"some_of"。

      • "each_of" 将导致此点下方的所有条目和物品添加到列表中，作为战利品给予玩家。
      • "some_of" 将根据 "num_rolls" 值导致列出的单个或多个随机条目或物品添加到列表中，作为战利品给予玩家。

    • "num_rolls"：

      • 可以是一个包含键 "min" 和 "max" 的表
      • 可以是一个数字
      • 可以省略（将默认为 1）
      • 当使用"each_of"类型时，此值适用于其中每个物品的掉落数量。

    • "items"：将唯一字符串名称（编造一个！）映射到具有键 "type"、"uri"、"weight" 和 "num_rolls" 的条目的字典。

      • "type" 可以是 "item" 或 "bag"。如果省略，默认为"item"。
        • "item" 是物品的 uri（椅子、床、武器）。
        • "bag" 是另一个物品/袋子列表的 uri（就像这个一样的另一个战利品表）。
      • 如果唯一字符串名称是 "none" 或 "uri" 是空字符串，如果滚动到它，则不会创建掉落物。
      • 如果省略 "weight"，默认为 1。非整数值是可以的。
      • "num_rolls"：与上面解释的可能值相同。

来自 entling.json 的示例：

  "stonehearth:loot_drops": {
   "entries": {
      "default": {
         "num_rolls": {
            "min": 1,
            "max": 1
         },
         "items": {
            "ore": {
               "uri": "stonehearth:resources:wood:oak_log",
               "weight": 10
            }
         }
      }
   }
}

来自 goblin_tent.json 的示例：

  "stonehearth:loot_drops": {
     "force_auto_loot": true,
     "entries": {
        "default": {
           "items": {
              "skin": {
                 "uri": "stonehearth:loot_bag:goblin_tent",
                 "weight": 1,
                 "type": "bag"
              }
           }
        }
     }
  }

您可以在 stonehearth/lib/loot_table/loot_table.lua 中找到更长的示例。

其他文件即使不使用 loot_drops 组件也会使用战利品表语法。例如，我们可以在 small_oak_tree.json 中找到一个名为"resource_loot_table"的字段，用于除了木材之外还有几率掉落种子：

  "stonehearth:resource_node": {
     "durability": 6,
     "resource": "stonehearth:resources:wood:oak_log",
     "resource_loot_table": {
        "entries": {
           "optional": {
              "num_rolls": {
                 "min": 1,
                 "max": 1
              },
              "items": {
                 "none": {
                    "uri": "",
                    "weight": 8
                 },
                 "pelt": {
                    "uri": "stonehearth:trees:oak:seed",
                    "weight": 1
                 }
              }
           }
        }
     }
  }

加权集

我们在许多 JSON 文件中使用加权集。通过它们，当我们抽取随机条目时（例如，选择随机天气、随机遭遇战、随机战利品...），我们可以赋予某个条目更高的优先级或更多的被选中机会。

权重是一种分配某事概率的便捷方法，而无需分配严格的概率。概率是介于 0（从不发生）和 1（总是发生）之间的值。因此，如果您使用概率，表中的数字必须始终加起来为 1。因此，如果您添加新内容或删除某些内容，您必须重新计算您的数字。

权重有点不同！权重仅仅说明某事相对于其他事情的可能性有多大。

假设您希望 A 的掉落频率是 B 的 5 倍。B 的任意权重为 2。因此 A 的权重必须为 10，是 B 的 5 倍。所以目前，我们的总权重为 12。我们有 2/12 的机会看到 B，有 10/12 的机会看到 A。

如果我们想添加权重为 8 的物品 C（比 B 常见 4 倍）怎么办？现在我们的总权重为 20，给予 10/20 的机会看到 A，2/20 的机会看到 B，8/20 的机会看到 C。每个物品掉落的相对几率保持不变，但 A 和 B 掉落的几率现在比以前少了。这很重要。

在调整此权重之前，请检查您要 mixinto 的文件中的数字。如果您的数字太低，您的物品将不会掉落太多（如果有的话）。如果您的数字太高，它将超过所有其他物品，而其他任何东西都不会掉落！
