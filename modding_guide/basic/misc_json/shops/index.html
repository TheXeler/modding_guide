负载

其中一个出发屏幕是可供选择的负载列表。您可以使用 mixintos 修改此列表，以删除一些现有负载或添加您自己的组合。

除了每个负载外，每个王国开始时都有一个默认的 crafting talisman 和来自默认资源的几件物品，这两者都在其人口 JSON 文件中定义。

要添加新负载：

  1. 对包含负载的文件进行 mixinto：

    "mixintos": {
   "stonehearth:loadouts": "file(data/loadouts/loadouts.json)"
}

  2. 在 mixinto 文件中添加您的自定义负载：

    {
   "loadouts": {
      "colorful_garden": {
         "ordinal": 5,
         "display_name": "i18n(my_mod:data.loadouts.colorful_garden.display_name)",
         "description": "i18n(my_mod:data.loadouts.colorful_garden.description)",
         "tooltip": "i18n(my_mod:data.loadouts.colorful_garden.tooltip)",
         "splash": "file(images/colorful_garden/splash.png)",
         "seal": "file(images/colorful_garden/seal.png)",
         "content": {
            "blue_tulip": {
               "ordinal": 1,
               "display_name": "i18n(my_mod:data.loadouts.items.blue_tulip)",
               "type": "entity",
               "uri": "stonehearth:plants:cactus:tulip:blue",
               "icon": "/stonehearth/entities/plants/cactus_tulip_plant/blue/cactus_tulip_blue_plant.png",
               "amount": 10
            },
            "red_tulip": {
               "ordinal": 2,
               "display_name": "i18n(my_mod:data.loadouts.items.red_tulip)",
               "type": "entity",
               "uri": "stonehearth:plants:cactus:tulip:red",
               "icon": "/stonehearth/entities/plants/cactus_tulip_plant/red/cactus_tulip_red_plant.png",
               "amount": 10
            },
            "yellow_tulip": {
               "ordinal": 3,
               "display_name": "i18n(my_mod:data.loadouts.items.yellow_tulip)",
               "type": "entity",
               "uri": "stonehearth:plants:cactus:tulip:yellow",
               "icon": "/stonehearth/entities/plants/cactus_tulip_plant/yellow/cactus_tulip_yellow_plant.png",
               "amount": 10
            },
            "gold": {
               "ordinal": 4,
               "display_name": "i18n(stonehearth:data.loadouts.items.gold)",
               "type": "gold",
               "icon": "/stonehearth/data/loadouts/images/items/gold.png",
               "uri": "stonehearth:loot:gold",
               "amount": 100
            }
         }
      }
   }
}

我们在 "loadouts" 内添加一个具有自定义标识符的条目。这些属性是不言自明的。"splash" 和 "seal" 是当我们选择负载时出现在右侧的图像。"tooltip" 是出现在右侧选定负载底部的文本。

如果我们想添加一定数量的金币，我们使用 "type" : "gold"，而不是 "type" : "entity"。

要删除现有负载，请使用 mixintypes。

商店

游戏中的商店是通过遭遇战管理的。我们还可以防止物品被购买或出售。

例如，这些是 stonehearth/entities/furniture/wooden_chair/wooden_chair.json 的"entity_data"内的净值属性：

  "stonehearth:net_worth": {
     "value_in_gold": 4,
     "rarity": "common",
     "shop_info": {
        "buyable": true,
        "sellable": true,
        "shopkeeper_level": 1,
        "shopkeeper_type": "caravan"
     }
  }

"value_in_gold" 是我们在商店中出售此物品时收到的金钱（根据物品质量或城镇奖励可能会更高）。它也是对城镇净值的贡献（非资源和非食用物品，加上建筑物、作物和牧场 contributes 到您城镇的总净值）。

默认情况下，店主将以 value_in_gold * SALE_MULTIPLIER 的价格出售物品。此乘数定义在 constants.json 中，位于"score" : { "shop" : {} } 常量下，因此我们可以更改它。

"rarity" 用于确定商店中物品的稀有度。可能的值是：

    "junk" / "common" / "uncommon" / "rare"

在 "shop_info" 内，我们可以确定此物品是否应该 "buyable"（可购买）、"sellable"（可出售），如果是，则可以出售它的商店的 "shopkeeper_level"（店主等级）。"shopkeeper_type" 是一个不再使用的遗留字段。可以省略它。

所有商店都有很小的几率出售精良品质的物品（但不是卓越或大师品质），如果这些物品具有以下属性：

    "stonehearth:item_quality": {
       "variable_quality": true
    }

在它们的 entity_data 内（此属性还允许制作者制作更高质量的物品版本，如果他们解锁了该职业特权）。

当我们达到新的城镇层级时，新的商店将变得可用（基于 shopkeeper_level），但旧商店仍然可能出现（几率较小）。 更高层级的商店将包含 shopkeeper level 相同或低于为商店指定等级的物品。

让我们向第 1 层商店列表中添加一个新商店，以便它可以随机出现在游戏中：

  1. 复制 stonehearth/data/gm/campaigns/trader/arcs/encounters/tier_1_shops/stonecutter_shop_1.json 到您的 mod。

  2. 将其重命名为 custom_shop_1.json，例如，并让我们编辑字段：

    • "type": "encounter" -- 此文件代表一个遭遇战。

    • "encounter_type": "shop" -- 遭遇战类型是"shop"。

    • "unique": true -- 此遭遇战是唯一的，同一时间只能运行一个此类遭遇战。

    • "in_edge": "custom_shop_1" -- 这里我们使用将在"trader"弧线的遭遇战列表中 mixinto 的遭遇战名称（见下一步）。

    • "shop_info": {} -- 商店的参数：

      • "title" -- 将显示为商店通知的标题。

      • "name" -- 商店的名称，它将出现在商店窗口的顶部。

      • "inventory": {} -- 商店的内容：

        • "shopkeeper_level": 1 -- 此商店的物品在其 entity_data 的净值信息中具有此 shopkeeper level，如上所述。

        • "shopkeeper_gold": 600 -- 店主从您那里购买物品的最大金币数量。它可能会根据城镇奖励等增加。通常，要使您的物品出现在商店的“出售”选项卡中，它们必须在存储或堆放中。

        • "overall_price_multiplier": 2 -- 可选字段。如果我们包含它，此商店中的所有物品将以 (value_in_gold * SALE_MULTIPLIER) * overall_price_multiplier 的价格出售。

        • "entries": [] -- 描述商店内容的对象数组。有两种类型的条目，我们可以根据需要添加任意数量：

          {
   "items_matching": {
      "clay_stuff": {
         "category": [
            "furniture",
            "storage"
         ],
         "material": [
            "clay"
         ]
      }
   },
   "quantity": {
      "min": 0,
      "max": 6
   }
}

          在这个条目中，我们有一个 "items_matching" 键，里面有一个标识符。在标识符内，我们有 "category"，这是来自目录信息的类别列表，以及可选的 "material"，这是来自物品材质标签的标签列表。

          然后，我们有另一个带有 "quantity" 的条目，它将在 "min" 和 "max" 值之间随机生成每个匹配上述条件的物品的数量。

          商店将拥有游戏中符合这些类别之一 AND 在其 material_tags 中具有该材质的所有物品（如果我们声明了该键）。目前还没有游戏中使用的所有类别的列表，但您可以检查不同的物品以找出它们的类别。

          另一种类型的条目是这样的：

          {
   "items": [
      "stonehearth:cleric:talisman",
      "stonehearth:herbalist:talisman"
   ],
   "quantity": {
      "min": 5,
      "max": 10
   }
}

          一个 "items" 列表，其中包括物品的别名，以及如上定义的 "quantity"。这样我们可以确保某些物品将出现在我们的商店中，无论它们的类别如何（但请确保它们是可购买的）。

          可选地，我们可以为任何条目在"quantity"下方包含此键：

          "price_multiplier": 3

          这将使此特定条目中的物品以 (value_in_gold * SALE_MULTIPLIER) * price_multiplier 的价格出售。

  3. 对 caravan_shops_arc.json 添加 mixinto：

    "mixintos" : {
   "stonehearth/data/gm/campaigns/trader/arcs/caravan_shops_arc.json" : "file(data/gm/campaigns/trader/arcs/caravan_shops_arc.json)"
}

    mixinto 将看起来像这样：

    {
   "encounters": {
      "custom_shop_1": "file(encounters/tier_1_shops/custom_shop_1.json)"
   }
}

  4. 为您的清单中的 tier_1_shops.json 添加另一个 mixinto：

    "mixintos" : {
   "stonehearth/data/gm/campaigns/trader/arcs/caravan_shops_arc.json" : "file(data/gm/campaigns/trader/arcs/caravan_shops_arc.json)",
   "stonehearth/data/gm/campaigns/trader/arcs/encounters/tier_1_shops/tier_1_shops.json" : "file(data/gm/campaigns/trader/arcs/encounters/tier_1_shops/tier_1_shops.json)"
}

    mixinto 将看起来像这样：

    {
   "out_edge": {
      "out_edges": [
         {
            "out_edge": "custom_shop_1",
            "type": "weighted_edge",
            "weight": 1
         }
      ]
   }
}

  5. 最后，让我们检查我们的商店是否可以出现在游戏中。

    首先确保您的城镇经历了旗帜庆典，以确保我们处于第 1 层。

    然后，打开战役浏览器，点击商人战役遭遇战的节点名称以打开小窗口，然后点击“立即触发”，直到您到达“生成商店”遭遇战。

    由于商店是随机的，并且我们为自定义商店使用了相同的权重，我们需要多次触发遭遇战并关闭其他商店，直到我们看到它出现：

    Rayya's children 有一个不同的商人战役，因此我们需要在他们的弧线中添加另一个 mixinto，以便也为他们包含我们的自定义商店。 Northern alliance 重用 Ascendancy 的战役。
