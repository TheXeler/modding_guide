颜色和材质图

颜色图是我们可以用来为物品添加颜色多样性的东西。它们与材质图一起使用。现有的颜色图和材质图可以在 stonehearth/data/materials 内找到。

这些图可用于将调色板或渲染材质应用于 QB 模型或其部分，可以通过 JSON 或在 Lua 脚本中动态应用。

一些背景：对象在创建时会在其 entity_data 中查找默认的"stonehearth:render_materials"材质（参见 item_properties.json mixin 示例）。我们可以通过 mixinto 更改此默认材质，以便物品具有自定义默认材质，或者我们可以使用材质图为其应用不同的材质和着色器。

随机化颜色

颜色图的用途之一是允许相同的 .qb 模型每次在游戏中生成时具有随机颜色（这也可以通过使用不同的 .qb 模型来实现）。

  1. 让我们从创建一个 color_map 开始并为其在清单中添加别名：

    "aliases" : {
   "color_map:flower" : "file(data/materials/color_maps/flower_color_map.json)"
}

    flower_color_map.json 文件如下所示：

    {
   "#FC6EFE" : "silent_woods:flower:hilight",
   "#A438A5" : "silent_woods:flower:midtone",
   "#761B76" : "silent_woods:flower:shadows"
}

    JSON 键以十六进制格式引用我们 QB 模型中的颜色。您可以根据需要添加任意数量的键，但如果您稍后修改模型，请确保也更新此处的颜色条目，否则颜色图将不生效。

    color_map 内键的值可以命名为任何名称。我们通常将它们写为 URI，以 mod 的名称为前缀，但它们只是我们将在材质图中使用的标识符。如果您不想对模型的某些部分应用随机颜色，只需不要将该部分的颜色添加到您的颜色图中即可。

    QB 模型具有什么颜色并不重要，因为我们将用此重新映射它们。例如，炉民不同发型的模型是紫色和粉色的，但您不会在游戏中看到这些颜色，因为会对其应用调色板。

  2. 现在让我们创建一个将这些颜色区域映射到其他颜色的材质图。让我们将此文件命名为 flower_red_material_map.json，例如：

    {
   "silent_woods:flower:hilight" : {
      "color" : "#DB6D5C"
   },
   "silent_woods:flower:midtone" : {
      "color" : "#A94839"
   },
   "silent_woods:flower:shadows" : {
      "color" : "#5F241B"
   }
}

    这里我们将标记为 "silent_woods:flower:hilight"（"#FC6EFE"）的颜色更改为"#DB6D5C"。其他颜色也是如此。 让我们复制此文件，将其命名为 flower_blue_material_map.json，并为每个条目分配不同的颜色。

  3. 最后，让我们将颜色和材质图分配给我们的实体。为此，在我们的实体 ghost JSON 文件中，我们将它们添加到 render_info 组件（它也可以在 iconic 的 JSON 文件中，或在您声明要更改的 QB 模型的任何位置）：

    ...
"components": {
   "render_info": {
      "color_map": "silent_woods:color_map:flower",
      "material_maps": [
         {
            "type": "one_of",
            "items" : [
               "/silent_woods/data/materials/material_maps/flower_red_material_map.json",
               "/silent_woods/data/materials/material_maps/flower_blue_material_map.json"
            ]
         }
      ]
   },
   ...
},
...

    使用 "type" : "one_of"，将只选择其中一个材质图。这样，每当我们在游戏中生成我们的花时，它将是红色或蓝色，而不是 QB 模型的原始颜色。

    一旦花生成，即使我们移动它或取消部署它，它也将保持选择的颜色。但是对于 ghosts，它们每次都会有不同的随机颜色（因为每次我们移动或放置物品时都会生成一个新的 ghost）。但是，您可以将一个材质图应用于 ghost，将另一个材质图应用于实体，这样您就可以控制在每种情况下将应用哪些颜色。

"one_of"类型也可用于随机化 QB 模型。

在测试时，如果您注意到无论您生成物品多少次，它总是具有相同的颜色，请尝试在您声明颜色/材质图的"render_info"组件内添加以下内容：

    "cache_model_geometry": false

出于性能原因，默认情况下会缓存模型几何体，但在 Lua 文件中的某些地方我们禁用了它，因此对于某些实体，即使它们的 JSON 文件中没有将"cache_model_geometry"设置为 false，随机颜色也会起作用。

应用渲染材质

我们还可以使用材质图应用不同的渲染材质。它将使颜色以不同方式渲染，例如使其闪亮、半透明或发光。在尝试制作自己的材质之前，您可以使用 stonehearth mod 中的现有材质。

  1. 首先，让我们像以前一样制作一个颜色图。但这次让我们重用现有材质图中的条目。这是 silent_woods/data/materials/color_maps/nightdew_color_map.json：

    {
   "#204C7A": "stonehearth:glowing",
   "#20598D": "stonehearth:glowing",
   "#24619A": "stonehearth:glowing",
   "#8895AE": "stonehearth:glowing"
}

    "stonehearth:glowing" 引用了 stonehearth/data/materials/material_maps/mob_material_map.json 和 stonehearth/data/materials/material_maps/natural_material_map.json 中的条目。

  2. 现在我们将颜色图和匹配的材质图添加到我们的实体。因此，在 nightdew_ghost.json 内，我们有这样的 render_info：

    "render_info": {
   "scale": 0.15,
   "color_map": "stonehearth:color_map:natural",
   "material_maps": [
      "/stonehearth/data/materials/material_maps/natural_material_map.json"
   ]
},

    注意"material_maps"数组仅包含条目，而不是像前面随机化颜色示例中的对象。如果我们希望应用所有材质图，我们可以以这种方式添加多个材质图（例如，如果我们的颜色图的条目在不同的材质图中）。

    让我们看看 natural_material_map.json：

    {
   "stonehearth:glowing" : {
      "render_material" : "materials/glowy_voxel.material.json"
   },
   "stonehearth:shiny" : {
      "render_material" : "materials/shiny.material.json"
   },
   "stonehearth:voxel" : {
      "render_material" : "materials/voxel.material.json"
   },
   "stonehearth:transparent_glow" : {
      "render_material" : "materials/transparent_with_depth_bloom.material.json"
   }
}

    这里我们在顶部有 "stonehearth:glowing" 条目。它不是指向颜色，而是指向渲染材质。这里还有其他条目，因此如果我们在颜色图的某个条目中使用它们，它们也将应用于这些颜色。

    "render_material" 中定义的路径指向 stonehearth/data/horde 内的文件。即使我们在我们的 mod 中使用了自己的材质图文件，如果我们这样声明，游戏将从 stonehearth mod 中读取材质文件：

    "render_material" : "materials/glowy_voxel.material.json"

    如果我们希望游戏从我们的 mod 中读取材质文件，我们必须这样编写完整路径：

    "render_material" : "/silent_woods/data/horde/materials/glowy_voxel.material.json"

  3. 让我们测试渲染材质是否应用于我们的实体（如果存在任何问题，stonehearth.log 中将出现一条消息，通知游戏无法加载材质）：

您可以在此处阅读更多关于渲染材质和着色器的信息。
