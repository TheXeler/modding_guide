植物和树木

我们在上一节中学习了如何创建可再生和不可再生资源。在这里，我们将了解如何将它们添加到现有生物群落中，以便它们在世界生成时生成。有关如何从头开始创建您自己的生物群落，请遵循此指南。

向现有生物群落添加新植物

  1. 决定您希望您的植物/树木在哪个生物群落中生成。对于我们的示例，我们将使来自 silent_woods mod 的 Nightdew 植物出现在温带生物群落中。

  2. 创建对生物群落 generation_data 文件的 mixinto：

    "mixintos" : {
   "stonehearth/data/biome/temperate_generation_data.json" : "file(data/biome/temperate_generation_data.json)"
}

  3. 将您的植物包含在生物群落生成数据的适当位置。这是我们的 mixinto 的样子：

    {
   "landscape": {
      "placement_table": {
         "silent_woods:plants:nightdew": {
            "placement_type": "dense",
            "parameters": {
               "grid_multiple": 2,
               "item_density": 0.2,
               "exclusion_radius": 1
            }
         },
      },
      "scattered": {
         "plants": {
            "weights": {
               "plains": {
                  "1": {
                     "silent_woods:plants:nightdew": 1
                  }
               },
               "foothills": {
                  "1": {
                     "silent_woods:plants:nightdew": 2
                  },
                  "2": {
                     "silent_woods:plants:nightdew": 1
                  }
               },
               "mountains": {
                  "1": {
                     "silent_woods:plants:nightdew": 0.1
                  }
               }
            }
         }
      }
   }
}

    在 "landscape" 部分内，我们在 "placement_table" 和 "scattered" : { "plants" : {} } 组内添加了我们的植物。

    "placement_table" 内的条目是我们植物的别名，并包含放置属性（以孤立或集群方式生成它们）。

    然后，在 "scattered" : { "plants" : {} } 部分中，我们有不同地形海拔的 "weights"。"plains"、"foothills" 和 "mountains" 内的每个数字键代表一个地形海拔。在它们内部，我们再次声明我们植物的别名并为其分配一个权重（权重是根据该条目内的所有植物计算的）。

    这样，我们确定了与每种植物相比，我们的植物在每个地形海拔上出现的频率。

关于树木

树木在生物群落文件中有一个特殊性。如果我们按照 stonehearth mod 中的树木的方式组织它们，我们可以将它们包含在我们生物群落的 generation_data 文件的 "trees" 部分中。这取决于树木别名的结构。

如果我们的树木别名以":small"、":medium"、":large"或":ancient"结尾，这些后缀可用于在生成数据中确定大小。 例如： 

  "trees": {
     "sizes": {
        "stonehearth:trees:oak": {
           "small": 0,
           "medium": 8,
           "large": 18,
           "ancient": {
              "percentage": 0
           }
        }
     },
     "weights": {
        "plains": {
           "1": {
              "stonehearth:trees:oak": 1
           }
        },
        "foothills": {
           "1": {
              "stonehearth:trees:oak": 4
           },
           "2": {
              "stonehearth:trees:oak": 2
           }
        }
     }
  }

请注意，"sizes" 和 "weights" 内的键不是树木的完整别名，只是最后一个冒号之前的公共部分。

然后，在 sizes 部分中，我们可以为每棵树指定大小（与它们别名的后缀匹配）。这些被指定为阈值（值需要为 0 或更高），除了"ancient"变体，它有一个百分比。这些阈值用于噪声函数中，根据地形特征（水、山丘等）确定树木的放置。

我们可以为我们的树木大小使用其他名称（只要我们遵循此模式），或者拥有较少的不同大小。

不要忘记也在 "placement_table" 部分中添加树木的各个完整别名！

始终使用 stonehearth/data/biome/temperate_generation_data.json 文件作为示例，它是每个参数注释最多的文件。您还可以使用 desert_generation_data.json 或 arctic_generation_data.json 文件作为模板来启动您自己的生物群落。

季节性变体

生物群落每个季节可以有不同的颜色和天气（您也可以发明您自己的季节）。 除此之外，如果您愿意，植物和树木每个季节可以有不同的模型。

以下是使任何实体每个季节具有不同模型的步骤：

  1. 在定义其 QB 模型的 JSON 文件中，我们必须为每个季节添加一个模型变体。

    "model_variants": {
   "default": {
      "models": [
         "file(potted_flower_summer.qb)"
      ]
   },
   "autumn": {
      "models": [
         "file(potted_flower_autumn.qb)"
      ]
   },
   "winter": {
      "models": [
         "file(potted_flower_winter.qb)"
      ]
   },
   "spring": {
      "models": [
         "file(potted_flower_spring.qb)"
      ]
   }
}

  2. 然后，我们还必须在"components"内添加 "stonehearth:seasonal_model_switcher"：

    "stonehearth:seasonal_model_switcher": {
   "stonehearth:biome:temperate": {
      "spring": "spring",
      "summer": "default",
      "autumn": "autumn",
      "winter": "winter"
   }
}

    在此组件中，我们可以为拥有我们植物的每个生物群落添加一个条目。键是生物群落别名，值将季节（如每个生物群落 JSON 文件中定义）与我们在上面定义的模型变体名称映射。在我们的示例中，我们将"summer"季节映射到"default"模型变体。

    如果我们使用 "*" 作为键而不是生物群落别名，季节性模型变体将应用于所有定义了这些季节的生物群落，甚至是修改过的生物群落，这很有用：

    "stonehearth:seasonal_model_switcher": {
   "*": {
      "spring": "spring",
      "summer": "default",
      "autumn": "autumn",
      "winter": "winter"
   }
}

    当季节变化时，新季节的前几天将是过渡期。在此期间，地形将逐渐改变其调色板，具有季节性变体的实体将在过渡期间的随机时刻交换其模型。它们还将优先在夜间切换模型（因此有些夜晚不会有模型更改，而其他夜晚会有多个模型被交换）。

    为了测试，只需选择不同的起始季节并检查植物是否以正确的季节性变体生成会更快，因为即使以更高的速度播放，季节也需要很长时间才能过去。
