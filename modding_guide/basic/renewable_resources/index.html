可再生资源

可再生资源是指任何像浆果丛或绵羊这样的物品，一旦收获，将在一段时间后重新生长，我们将能够再次收获它。

创建可再生资源

这是我们如何制作来自"Silent Woods"教程 mod的生物发光蘑菇（"Nightdew"）：

  1. 我们将现有的可再生资源（浆果丛）复制到我们的 mod 中。

  2. 我们重命名了文件夹和文件。

  3. 我们编辑了 QB 模型并重新导出它们。

  4. 我们编辑了 JSON 文件并将本地化字符串添加到我们的区域设置文件中。 这些是我们在 nightdew.json 中需要为 renewable_resource_node 组件填写的字段： 

    ...
"stonehearth:renewable_resource_node": {
   "resource": "silent_woods:resources:biolumin_essence:essence",
   "category": "harvest",
   "renewal_time": "22h+2h",
   "unripe_description": "i18n(silent_woods:entities.plants.nightdew.unripe_description)",
   "harvest_overlay_effect": "stonehearth:effects:harvest_plant_overlay_effect",
   "resource_loot_table": {
      "entries": {
         "optional": {
            "num_rolls": {
               "min": 1,
               "max": 1
            },
            "items": {
               "none": {
                  "uri": "",
                  "weight": 4
               },
               "seed": {
                  "uri": "silentwood:plants:nightdew:seed",
                  "weight": 1
               }
            }
         }
      }
   }
},
...

    • "resource"：我们收获此实体时要生成的物品的完整别名。它可以是您 mod 中的物品，也可以是另一个 mod 中的物品。

    • "category"：大多数情况下它将是"harvest"，但您可能希望为其使用不同的值。例如，对于剪羊毛，类别是"herding"。这使得只有牧羊人可以收获它（因为只有牧羊人的 AI 有 herding 任务组，这允许收获具有"herding"类别的可再生资源）。

    • "renewal_time"：资源自我更新所需的时间。时间表达式示例："1d"（一天）、"36h"（36 小时）、"6m"（6 分钟）、"22h+2h"（22 小时加上 0 到 2 小时之间的随机时间量）。

    • "unripe_description"：本地化描述，将在实体被收获后直到资源更新之前出现在单位框架中。

    • "harvest_overlay_effect"：当实体被标记为收获时播放的效果的完整别名。在这种情况下，我们重用了 stonehearth mod 中的收获植物覆盖效果，它在实体上方显示一个小植物图标。

    • "resource_loot_table"：此字段是可选的。我们可以使用它在每次生成主要资源时有几率生成其他额外资源。其值是一个战利品表。

    此外，我们确保在此文件中包含一个名为 "depleted" 的模型变体（默认模型在 ghost 的 JSON 文件中声明）。一旦实体被收获，其模型将自动交换为 depleted 变体：

    "model_variants": {
   "depleted": {
      "models": [
         "file(nightdew_empty.qb)"
      ]
   }
}

    如果我们在 renewable_resource_node 组件中添加 "durability" 字段，我们将能够收获实体 N 次，之后它将被销毁。如果我们不包含它，我们将能够在每次重新生长时无限期地继续收获它。

  5. 我们在 mod 的清单中为我们的蘑菇添加了别名：

    "plants:nightdew": "file(entities/plants/nightdew/nightdew.json)"

  6. 我们通过复制和编辑 stonehearth/entities/resources 中的现有资源来创建它掉落的资源，并将别名添加到我们的清单中：

    "resources:biolumin_essence:essence": "file(entities/resources/biolumin_essence/essence)"

  7. 最后，我们将植物添加到生物群落中（如这里所示。如何创建您自己的生物群落在这里解释），并包含一些材质映射，以便蘑菇发光。确保测试它可以正确收获并且可以更新其资源。

创建不可再生资源

我们还可以创建不会更新其资源并在收获后消失的物品（例如树木、巨石和一些现有植物）。为此，我们将在实体的 JSON 文件中使用 resource_node 组件，而不是 renewable_resource_node 组件。

通常，不可再生资源有一个 "durability" 字段，这是在消失之前将被收获的次数。来自 small_oak_tree.json 的示例：

  "stonehearth:resource_node": {
     "resource": "stonehearth:resources:wood:oak_log",
     "durability": 6
  }

树木重用 tree.json mixin，它具有 resource_node 组件的这些其他字段：

  "stonehearth:resource_node": {
     "harvest_overlay_effect": "stonehearth:effects:chop_overlay_effect",
     "harvester_effect": "chop",
     "description": "chop_tree"
  }

它包括一个 "harvest_overlay_effect"（与可再生资源相同），以及一个 "harvester_effect" 字段，以便炉民在收获它们时播放砍伐动画。"description" 字段出于遗留原因存在，因此不要更改其值。

我们还可以使用 resource_loot_table 来实现掉落数量，而不是 durability。这是来自 wintermoss.json 的示例：

  ...
  "stonehearth:resource_node": {
     "resource": "stonehearth:resources:fiber:wintermoss_bundle",
     "category": "harvest",
     "harvest_overlay_effect": "stonehearth:effects:harvest_plant_overlay_effect",
     "resource_spawn_effect": "stonehearth:effects:harvest_plants_effect",
     "description": "harvest_plant",
     "resource_loot_table": {
        "entries": {
           "mandatory": {
              "num_rolls": {
                 "min": 1,
                 "max": 2
              },
              "items": {
                 "none": {
                    "uri": "stonehearth:resources:fiber:wintermoss_bundle",
                    "weight": 1
                 }
              }
           },
           "optional": {
              "num_rolls": {
                 "min": 1,
                 "max": 1
              },
              "items": {
                 "none": {
                    "uri": "",
                    "weight": 3
                 },
                 "pelt": {
                    "uri": "stonehearth:plants:wintermoss:seed",
                    "weight": 2
                 }
              }
           }
        }
     }
  },
  ...

资源战利品表将掉落 1 或 2 束 wintermoss，如果幸运的话，有时还会掉落 1 颗种子。之后，即使我们只收获了一次，wintermoss 也会消失。

为植物创建种子

如何为作物创建种子在这里解释。 野生植物的种子将具有生长阶段，类似于作物的生长阶段。在游戏中，您可以移动刚种植的种子，但如果它们是树木或某些其他植物的种子，一旦它们第一次生长，您将无法再移动它们。当然，您可以在 modded 实体的组件中更改此行为。

让我们看看如何创建种子。它涉及许多文件，具体取决于我们希望在种子和完全生长的植物之间有多少个生长阶段，但它们只是 JSON 和 QB 模型，过程与任何其他实体完全相同。

我们将使用 stonehearth mod 中的 wintermoss 作为示例：

  1. 首先，我们找到我们要从中复制种子的现有植物。在这种情况下，请注意我们将有 4 个生长阶段加上成熟的 wintermoss：

  2. 首先让我们修改"seed"文件夹内的文件： 我们重命名它们以引用我们植物的名称，保持名称的后缀不变，以便我们知道每个文件引用什么。

    wintermoss_plant_seed_iconic.json：

    {
   "mixins": "stonehearth:mixins:item_properties",
   "type": "entity",
   "components": {
      "model_variants": {
         "default": {
            "models": [
               "file(wintermoss_plant_seed_iconic.qb)"
            ]
         }
      },
      "mob": {
         "model_origin": { "x": 0.05, "y": 0, "z": 0.05 }
      }
   },
   "entity_data": {
      "stonehearth:catalog": {
         "is_item": true,
         "category": "seed",
         "material_tags": "stockpile_plant plant resource seed seed_hard"
      },
      "stonehearth:food_decay": {
         "initial_decay": {
            "min": 168,
            "max": 168
         }
      }
   }
}

    这里我们引用种子的图标模型、存储过滤器的材质标签以及其他用途的类别。我们还在"entity_data"中添加 "stonehearth:food_decay" 数据，以便未在一周内种植的种子会自动销毁（这由 food_decay 服务控制 - 腐烂计算为"min"和"max"值之间的范围，这些值是表示游戏内小时的数字，在本例中为 24h * 7d = 168）。

    wintermoss_plant_seed_ghost.json：

    {
   "mixins": "stonehearth:mixins:placed_object",
   "type": "entity",
   "components": {
      "render_info": {
         "scale": 0.15
      },
      "model_variants": {
         "default": {
            "models": [
               {
                  "type": "one_of",
                  "items": [
                     "file(../wintermoss_1.qb)"
                  ]
               }
            ]
         }
      },
      "mob": {
         "model_origin": { "x": -0.075, "y": 0, "z": 0.075 },
         "region_origin": { "x": 0.5, "y": 0, "z": 0.5 }
      }
   },
   "entity_data": {
      "stonehearth:catalog": {
         "display_name": "i18n(stonehearth:entities.plants.wintermoss.seed.wintermoss_plant_seed_ghost.display_name)",
         "description": "i18n(stonehearth:entities.plants.wintermoss.seed.wintermoss_plant_seed_ghost.description)",
         "icon": "file(wintermoss_plant_seed.png)",
         "category": "seed"
      }
   }
}

    这里我们调整 ghost 的模型（我们使用完全生长植物的模型，以便我们知道它在完成生长时将占用多少空间）、比例和模型/区域原点（如果需要），以及目录的数据。

    wintermoss_plant_seed.json：

    {
   "type": "entity",
   "mixins": "stonehearth:mixins:placed_object",
   "components": {
      "render_info": {
         "scale": 0.1
      },
      "stonehearth:evolve": {},
      "stonehearth:entity_forms": {
         "iconic_form": "file(wintermoss_plant_seed_iconic.json)",
         "ghost_form": "file(wintermoss_plant_seed_ghost.json)",
         "placeable_on_ground": true,
         "hide_undeploy_ui": true
      },
      "model_variants": {
         "default": {
            "models": [
               {
                  "type": "one_of",
                  "items": [
                     "file(../../dirt_pile/dirt_pile_1.qb)",
                     "file(../../dirt_pile/dirt_pile_2.qb)",
                     "file(../../dirt_pile/dirt_pile_3.qb)"
                  ]
               }
            ]
         }
      },
      "mob": {
         "model_origin": { "x": -0.075, "y": 0, "z": 0.075 },
         "region_origin": { "x": 0.5, "y": 0, "z": 0.5 },
         "mob_collision_type": "clutter"
      }
   },
   "entity_data": {
      "stonehearth:evolve_data": {
         "next_stage": "stonehearth:plants:wintermoss:sprout",
         "evolve_time": "2h+2h"
      },
      "stonehearth:net_worth": {
         "value_in_gold": 2,
         "rarity": "common",
         "shop_info": {
            "buyable": true,
            "sellable": true,
            "shopkeeper_type": "caravan",
            "shopkeeper_level": 1
         }
      },
      "stonehearth:catalog": {
         "display_name": "i18n(stonehearth:entities.plants.wintermoss.seed.wintermoss_plant_seed.display_name)",
         "description": "i18n(stonehearth:entities.plants.wintermoss.seed.wintermoss_plant_seed.description)",
         "icon": "file(wintermoss_plant_seed.png)",
         "category": "seed",
         "material_tags": "stockpile_plant plant resource seed seed_hard"
      }
   }
}

    这是种子的根形态。这些字段中的大多数应该已经很熟悉了。让我们解释一下新的字段：

    • "stonehearth:evolve": {}" -- 此组件负责将种子生长为植物，以及将植物阶段生长为下一个阶段。它没有属性，它们是在"entity_data"的 "stonehearth:evolve_data" 部分中定义的，因为它们可重用于此类植物的所有种子。

      演化数据的属性是："next_stage"，指向下一个生长阶段的完整别名，在本例中为"stonehearth:plants:wintermoss:sprout"，以及 "evolve_time"，这是一个时间表达式，表示种子种植后生长到下一个阶段所需的时间。如果尚未添加别名，请记住将它们添加到您的清单中。

      我们还可以为 "next_stage" 属性指定 URI 数组。这样实体将随机演化为这些实体之一，例如，可以演化为 poyo 或恐龙的蛋。

    • "stonehearth:entity_forms" : { ... } -- 提醒一下，这里我们添加了两个可选字段："placeable_on_ground"，允许将此物品放置在地面上，以及 "hide_undeploy_ui"，这将在实体放置后从单位框架中隐藏取消部署按钮。

    • "mob" : { ... } -- "mob_collision_type" : "clutter" 意味着此实体可以与其他实体重叠，炉民可以穿过它。

  3. 一旦我们完成了种子文件，我们可以对下一个生长/演化阶段执行相同的操作，在本例中为"sprout"。

    如您所见，对于种子和完全生长植物之间的阶段，我们只需要一个文件，因为植物在生长过程中将无法取消部署，所以我们不需要为其提供图标/ghost 文件。所以让我们只更改我们需要指向我们文件的字段：

    {
   "type": "entity",
   "mixins": "stonehearth:mixins:placed_object",
   "components": {
      "stonehearth:evolve": {},
      "model_variants": {
         "default": {
            "models": [
               "file(wintermoss_sprout.qb)"
            ]
         }
      },
      "mob": {
         "model_origin": { "x": 0, "y": 0, "z": 0 },
         "region_origin": { "x": 0.5, "y": 0, "z": 0.5 },
         "mob_collision_type": "clutter"
      }
   },
   "entity_data": {
      "stonehearth:evolve_data": {
         "next_stage": "stonehearth:plants:wintermoss:stem",
         "evolve_time": "7h+2h"
      },
      "stonehearth:catalog": {
         "material_tags": "stockpile_plant",
         "display_name": "i18n(stonehearth:entities.plants.wintermoss.sprout.wintermoss_plant_sprout.display_name)",
         "description": "i18n(stonehearth:entities.plants.wintermoss.sprout.wintermoss_plant_sprout.description)",
         "category": "plants"
      },
      "stonehearth:appeal": {
         "appeal": 1
      }
   }
}

    它们类似于种子根形态的字段。我们声明下一个阶段的别名（记住将其添加到您的清单中）。

  4. 对后续阶段重复此操作，在本例中为"stem"和"bud"。

    Bud 然后将指向生长的 wintermoss 植物（wintermoss.json）的完整别名，完成生长周期：

    ...
"stonehearth:evolve_data": {
   "next_stage": "stonehearth:plants:wintermoss",
   "evolve_time": "7h+2h"
},
...

最后说明

evolve 组件和 decay 服务可用于需要在一段时间后更改的任何内容，例如 poyos 和绵羊使用 evolve 组件在出生后生长：

  ...
  "stonehearth:evolve_data": {
     "current_stage": "young",
     "next_stage": "stonehearth:sheep",
     "evolve_time": "24h",
     "evolve_effect": "stonehearth:effects:fursplosion_effect"
  },
  ...

在这种情况下，我们还指定了"current_stage"和"evolve_effect"，它们在其他 Lua 文件中使用。

growing 组件主要用于作物。
