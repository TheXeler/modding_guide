使用 SHED 添加物品

免责声明：如果 SHED 更新，这里的说明可能会过时或更改。SHED 有一些错误，因此可能不会按预期工作。

为了在 SHED 中创建物品，此页面中列出的步骤也适用。

首先，确保您已经设置好 SHED，并且它读取了您的 mod 和 stonehearth mod。我们也将在这里使用 startermod_basic 进行说明。

  1. 假设我们要添加一个新桌子，其大小与 stonehearth mod 中的单人桌大致相同。

  2. 首先，我们需要使用搜索栏找到此物品。 现在，我们将把它克隆到我们的 mod 中。

    如果我们计划为此物品创建配方，我们可以改为克隆配方，因为这将尝试克隆生成的物品。但是，如果它是 stonehearth mod 中现有制作者的新物品，我们需要在 SHED 之外编辑配方文件，并在 SHED 之外设置对配方索引的 mixinto。

    在树视图中右键单击它并点击"Clone"。

  3. 首先，让我们将目标 mod 更改为我们的 mod。 现在我们将输入一些内容以替换文件夹和文件名中的所有"table_for_one"。在我们的例子中，我们希望物品称为"custom_table"： 如果我们愿意，我们可以使用右上角的"+"按钮添加更多替换文本。现在点击"Clone!"按钮。

  4. 将出现一个包含所有将创建的文件和别名的列表： 花几秒钟检查以确保一切正确（您可以取消选中那些您不想创建的），然后点击"Accept"。刷新视图可能需要一点时间，请等待直到 SHED 恢复。

  5. 让我们在树视图中点击我们克隆的物品，并检查是否有错误。

    首先，您会注意到 ghost JSON 文件中 entity_data 的文本已被取消本地化。这对于自动本地化正常工作是必要的。

    只需点击"Localize This File"按钮，它将使用与 stonehearth 中相同的约定将相应的键添加到您 mod 的 en.json 文件中，并调整我们桌子的 JSON 文件以指向这些键（它应该自动刷新，等待几秒钟）。

    您现在可以 Ctrl+点击本地化字符串，将出现一个弹出窗口，让我们自动更改文本。这是 SHED 的一个非常方便的功能。对于那些无法使用它的人，他们始终可以使用文本编辑器中的搜索功能（或在文件中搜索）。

    现在来看模型。它将克隆桌子的模型并重命名它。如果您安装了体素编辑器，您可以点击带有 Qubicle 图标的按钮（在选项卡上方）以用它打开 QB 模型，并开始编辑。确保使用正确的设置和名称重新导出它。

  6. 在游戏中测试您的 mod，以确保物品看起来和行为正确。

在 SHED 中添加别名

您还可以通过在 SHED 中右键单击您的 mod 名称并点击"Add alias"来向您的 mod 添加物品。这将提示您选择一个要用该别名引用的文件，因此您需要事先已经创建了该文件。当您注意到 mod 中的某些现有文件将被多个文件引用并且您想为其创建别名时，这会很有用。

您还可以使用右键菜单中的选项删除别名，但它们指向的文件不会从您的 mod 中删除。如果您需要删除它们，请手动执行。
