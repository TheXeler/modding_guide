手动添加物品

准备您自己的 mod（例如此部分中的基本空 mod 示例），或为此示例重用 startermod_basic 或 startermod_items。

在继续之前，请确保将您的 stonehearth.smod 重命名为 stonehearth.zip，并正确解压缩，以便它仍然有效并且我们可以查看文件。

添加新物品的标准过程是：

  1. 在 stonehearth mod（或其他 mod）中找到与您想要创建的物品相似的物品。
  2. 将该实体的文件复制到您的 mod 文件夹中。
  3. 修改复制的 JSON 文件的名称和内容。
  4. 将物品的别名添加到您的 mod 清单中。
  5. 检查文件中的所有路径和别名，以确保它们指向现有文件。
  6. 在游戏中测试您的 mod，以确保物品看起来和行为正确。

例如，要向 startermod_basic mod 添加新物品，这些步骤的工作方式如下：

  1. 假设我们要添加一个新桌子，其大小与 stonehearth mod 中的"table for one"（单人桌）大致相同。

  2. 游戏中的所有实体（除了工作台、职业制服和一些其他东西）都位于 stonehearth/entities 文件夹内。

    为了方便起见，模组作者会重建 stonehearth 目录的结构，以便以后更容易找到文件，但这不是必需的。您可以将 mod 的所有文件放在与 manifest.json 文件相同的级别，或仅放在一个文件夹内，但当 mod 变得更大时，这是一种不好的做法。

    单人桌文件位于 stonehearth/entities/furniture/table_for_one 中。如果您的 mod 文件夹中没有 entities/furniture 子文件夹，请创建它们。然后，复制 stonehearth mod 中的 table_for_one 文件夹，并将其粘贴到您 mod 的 furniture 文件夹中。

  3. 将 table_for_one 文件夹重命名为您要添加的实体的名称。为了避免拼写错误和路径问题，文件夹和文件命名只使用小写字母、数字和下划线。在这种情况下，让我们将文件夹重命名为类似"custom_table"的名称。 现在来看文件。重命名文件，将"table_for_one"替换为"custom_table"。 然后，我们将在体素编辑器中打开 QB 文件，将模型更改为我们喜欢的样子，并将其重新导出为具有相应名称的 QB 文件。有关这方面的更多说明，请参阅指南的艺术部分。也不要忘记对模型进行截图，以替换复制的图标。 现在我们必须打开每个 JSON 文件并确保一切正确。打开 custom_table.json。

    将所有显示 table_for_one 的文本替换为 custom_table，以便路径指向我们的文件。 如果需要，调整 region_collision_shape 值（更多信息见这里）。 根据需要修改 entity_data。在这种情况下，我们将增加金币价值和吸引力。 接下来，打开 custom_table_ghost.json 文件。再次，将 table_for_one 更改为 custom_table，并在需要时调整 model_origin 和 region_origin。
    在 entity_data 中，将名称和描述更改为其他文本并对其进行本地化（本地化指南见这里，记得在您的清单中添加 "default_locale" : "en"！），或者只是保留纯文本而不进行本地化。更改图标的名称以指向我们的图像，并在需要时编辑材质标签（在这种情况下，我们将'wood'更改为'stone'）。

    最后，打开 custom_table_iconic.json 文件，并按上述说明调整值。

  4. 打开您的 mod 清单。如果其中不存在"aliases"部分，请添加它（必要时添加逗号）。在"aliases"部分内，让我们为新物品添加别名。同样，为了安全起见以防止拼写错误，我们只使用小写字母、数字和下划线。为了更好地组织别名，我们可以使用冒号（":"）。

    因此，在这种情况下，添加新别名后的 startermod_basic 清单将如下所示：

    {
   "info" : {
      "name" : "startermod_basic",
      "namespace": "startermod_basic",
      "version" : 3
   },
   "aliases" : {
      "sample_object" : "file(entities/sample_object/sample_object.json)",
      "piggy_bank"    : "file(entities/piggy_bank/piggy_bank.json)",
      "furniture:custom_table":"file(entities/furniture/custom_table/custom_table.json)"
   },
   "mixintos" : {
      "stonehearth/jobs/carpenter/recipes/recipes.json" : "file(recipes/recipes.json)"
   }
}

  5. 仔细检查您的文件以确保路径引用正确（有关如何在 stonehearth 文件中使用绝对/相对路径，请参见此页面）。另外，为了确保我们没有缺少或多余的逗号，让我们在 JSONLint 中检查我们的 JSON 文件。

  6. 打开游戏，确保您的 mod / startermod_basic mod 以及 debugtools mod 已启用。

    开始新游戏或加载存档。使用物品印章工具，输入 custom_table 并按 Enter 或在 dropdown 中点击物品以选择它。

    选中"iconic"复选框，点击印章工具并点击地形以生成它。右键单击以退出物品印章工具。

    点击生成的物品以验证名称和描述。

    下令将其放置在某个地方，以验证它在 3 种实体形式中看起来是否正确（您可能需要先创建一个储藏室并等待物品补货，以便放置菜单检测到它）。
