更改比例

当您建模时，对象与游戏中其他对象相比的大小并不重要，因为我们可以从其 JSON 文件中更改其比例。

如果您未指定物品的比例，则默认为 0.1。

树木和巨石的比例为 1（在其 mixins 中定义），这意味着它们模型中的每个体素占据 1 个地形方块（您可以通过打开区域菜单或建筑菜单来查看地形方块的大小，地形上将显示网格）。

Iconics（标志性物品）通常以 10x10x10 体素的大小建模，并且由于默认比例为 0.1，它们在游戏中完美地适应 1 个地形方块，这使它们在储藏室中很好地对齐，并且也适合炉民的手。炉民本身也有 0.1 的比例。他们的脸宽 13 个体素，您可以在游戏中看到他们几乎只占据一个地形方块的宽度。

要更改对象的比例，我们需要在其 JSON 文件中添加 "render_info" 组件。例如，这是 wheat_pile_ghost.json 文件，它决定了放置后小麦堆的外观：

  {
     "mixins": "stonehearth:mixins:placed_object",
     "components": {
        "model_variants": {
           "default": {
              "models": [
                 "file(wheat_pile.qb)"
              ]
           }
        },
        "render_info": {
           "scale": 0.12
        },
        "mob": {
           "model_origin": { "x": -0.55, "y": 0, "z": 0.5 },
           "region_origin": { "x": 0.5, "y": 0, "z": 0.5 }
        }
     },
     "entity_data": {
        "stonehearth:catalog": {
           "display_name": "i18n(stonehearth:entities.containers.wheat_pile.display_name)",
           "description": "i18n(stonehearth:entities.containers.wheat_pile.description)",
           "icon": "file(wheat_pile.png)",
           "category": "storage",
           "material_tags": "stockpile_cooking_ingredient crafted wheat_pile resource_pile"
        }
     }
  }

在"render_info"组件内部，我们说 "scale" : 0.12，这将使对象看起来比默认比例稍大。 有了这个，我们可以使物品更小（例如："scale" : 0.05 - 默认比例的一半大小）： 或更大（例如："scale" : 2 - 比默认比例大 20 倍）：

render_info 组件还用于定义其他属性，如动画表、颜色图和材质图。

我们还可以在游戏中测试这一点。选择一个实体，打开 Lua 控制台，然后输入以下内容：

  e:get_component('render_info'):set_scale(0.2)

然后按 Enter。一旦您知道所需的确切比例值，请在 JSON 文件中编辑它并保存更改。
