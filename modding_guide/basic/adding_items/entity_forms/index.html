实体形式

Entity forms 是游戏中所有可以放置在世界中、再次拾取和移动的物品使用的组件。在为您的 mod 创建新物品时，您应该了解它。

有些物品实际上不需要它，因此它们没有这个特定组件（例如，在世界生成时创建的不可移动物品，如树木/巨石，木材或其他不可放置的资源，食物篮子，炉民等）。

每个可放置物品必须有 3 种形式，由不同的 JSON 文件表示：

译者注：实际上这三种形式分别指的是已放置方块（Root）、未放置方块（Ghost）和物品（Iconic）形式的组件。

Root（根）

这代表已放置的物品。其 JSON 文件包含 entity_forms 组件，因此引用了另外两种形式的文件。

这是代表 Piggy Bank 实体的 piggy_bank.json 文件：

  {
     "type": "entity",
     "mixins" : "file(piggy_bank_ghost.json)",
     "components" : {
        "stonehearth:entity_forms" : {
           "iconic_form" : "file(piggy_bank_iconic.json)",
           "ghost_form" : "file(piggy_bank_ghost.json)",
           "placeable_on_ground" : true
        },
        "region_collision_shape" : {
           "region": [
              {
                 "min" : { "x" : 0, "y" : 0, "z" : 0 },
                 "max" : { "x" : 1, "y" : 0.4, "z" : 1 }
              }
           ]
        }
     },
     "entity_data" : {
        "stonehearth:net_worth" : {
           "value_in_gold" : 10,
           "rarity" : "common",
           "shop_info" : {
              "buyable" : false,
              "sellable" : true,
              "shopkeeper_level" : 1,
              "shopkeeper_type" : "caravan"
           }
        }
     }
  }

如您所见，此文件还将 ghost 形式文件引用为 mixin。这是因为 ghost 和最终放置的物品共享一些属性，例如 QB 模型、名称和描述，以及存储类别。这样我们就不必在 piggy_bank.json 文件中重复这些属性。

entity_forms 组件将读取我们在此声明的 ghost 和 iconic 形式的文件，因此游戏将知道当此实体在储藏室中、当我们下令放置它以及当它已经放置时如何表示它。

Ghost（幽灵）

这代表当我们下令将其放置在世界中但尚未放置时物品具有的半透明形式。

这是 piggy_bank_ghost.json 文件：

  {
     "mixins": "stonehearth:mixins:placed_object",
     "components": {
        "model_variants": {
           "default": {
              "models": [
                 "file(wooden_piggy_bank.qb)"
              ]
           }
        },
        "mob": {
           "model_origin": {
              "x": -0.05,
              "y": 0,
              "z": 0.05
           },
           "region_origin": {
              "x": 0.5,
              "y": 0,
              "z": 0.5
           }
        }
     },
     "entity_data": {
        "stonehearth:catalog": {
           "description": "i18n(startermod_basic:entities.piggy_bank.piggy_bank_ghost.description)",
           "icon": "file(piggy_bank.png)",
           "display_name": "i18n(startermod_basic:entities.piggy_bank.piggy_bank_ghost.display_name)",
           "material_tags": "toy decoration crafted stockpile_decoration"
        }
     }
  }

请注意，它没有"type"字段，它不是真正的实体，它只是被其他实体和组件重用的数据块。它只有一些我们需要表示物品"ghost"形式的 mixins、组件和 entity_data。我们还在 piggy_bank.json 文件中将其重用为 mixin，因此所有这些属性也被拉入该文件。

Iconic（标志性）

这代表物品的小型/简化形式，当我们制作/获得该物品以及炉民携带它或将其移动到储藏室时会得到这种形式。

这是 piggy_bank_iconic.json 文件：

  {
     "mixins": "stonehearth:mixins:item_properties",
     "type": "entity",
     "components": {
        "model_variants": {
           "default": {
              "models": [
                 "file(wooden_piggy_bank.qb)"
              ]
           }
        },
        "mob" : {
           "model_origin" : { "x": -0.05, "y": 0, "z": 0.05 }
        }
     }
  }

此文件代表物品的"iconic"形式，请注意它本身就是一个实体（"type": "entity"）。它也有自己的 mixins 和组件（这样我们可以为其定义不同的 QB 模型，例如）。Iconic 将与 root 形式具有相同的名称/描述/图标。
