目录

目录服务负责根据实体数据中的属性对实体进行分类。

这是目录服务使用的 entity_data 示例，它位于 berry_bush_ghost.json 内，因此 berry_bush.json 中描述的根实体也将具有此名称/描述/图标等（因为它将其作为 mixin 包含）：

  "entity_data": {
    "stonehearth:catalog": {
       "display_name": "i18n(stonehearth:entities.plants.berry_bush.display_name)",
       "description": "i18n(stonehearth:entities.plants.berry_bush.description)",
       "icon": "file(berry_bush.png)",
       "category": "plants",
       "material_tags": "plant stockpile_plant"
    }
  }

由于它在"entity_data"内，此信息将由该类型的每个物品共享。通常的属性是 "display_name"、"description" 和 "icon"，当您在游戏中选择此物品（在其任何实体形式中）时，它们将显示在单位框架中。如果我们愿意，我们可以为iconic 版本设置不同的属性，因为它本身是一个独立的实体。

除了这些，我们还有 "category"、"material_tags" 和 "subjects"。
这三个属性的值不应本地化（它们不在"i18n()"内），游戏使用它们在内部指定类别。

"material_tags" 是由空格分隔的材质字符串。或者，我们可以将其声明为字符串数组，这将使其与其他 mod 更兼容：

    "material_tags": [ "plant", "stockpile_plant" ]

这样，mod 可以向此物品添加标签而无需更改整个字符串，这可能会覆盖其他 mod 添加的标签。

如果您需要向以空格分隔的字符串形式拥有材质标签的物品添加材质标签，您的 mixinto 也需要有一个以空格分隔的字符串，覆盖整个值。如果您想将其转换为数组（或反之亦然），您需要使用 mixintype 来覆盖 JSON 节点。

在创建您自己的物品时正确填写此字段非常重要，因为它们会根据其中一个或多个标签补货到存储中。在这种情况下，"stockpile_plant"标签将使物品补货到启用了植物过滤器的堆放和箱子中。

当您在堆放中勾选"All"复选框时，它将接受每个物品，而不检查标签。当您勾选"None"复选框时，堆放将拒绝任何物品，无论它们的标签是什么。如果您勾选每个过滤器，"All"框将自动勾选，但它不会接受所有物品，只接受至少匹配其中一个过滤器的每个物品。

此处定义的其他标签用于编目物品，并且可以像任何其他 entity_data 一样查询。例如，我们可以将"flammable"作为材质标签之一，并在某个 Lua 文件中，检查具有该标签的所有实体并点燃它们（运行 VFX，一段时间后销毁实体，或更改其模型变体以使其具有烧焦的外观）。

您可能仍然在一些游戏文件中看到"material"组件。这已被弃用，并已由 entity_data 中"stonehearth:catalog"键内的"material_tags"取代。

现有存储过滤器的列表在 stonehearth/ui/data/stockpile_filters.json 中，输入框过滤器的列表在 stonehearth/ui/data/input_box_filters.json 中。它们在那里定义， along with 用于在 UI 中显示其名称的本地化字符串、其图标、顺序等。您可以使用 mixintos 向这些文件添加您自己的过滤器。

"category" 是一个通用类别，用于某些特质和组件，以及编目您的库存（用于食品、财富和在某些菜单中显示物品组）。您可以像使用材质标签一样使用它，并根据需要使用您自己的类别。没有现有类别的列表，但如果您从 stonehearth mod 复制类似物品，它将已经具有适当的类别。

它的值是单个单词，不是由空格分隔的几个单词，这与"material_tags"不同。

"stonehearth:catalog" entity_data 内的另一个常见键是

  "is_item": true

这通常用于 iconics 的 JSON 文件中，以便炉民知道这些是可以拾取的物品。

目录服务还读取实体数据内的 "stonehearth:net_worth" 和 "stonehearth:appeal" 条目，以编目可购买/可出售物品以及计入吸引力功能/物品偏好的物品。

单位信息

单位信息是一个以前用于管理实体所有名称和描述的组件，但现在应仅用于 可以具有自定义名称和描述的事物（例如宠物、炉民、哥布林...）。

在游戏中，当我们想要更改实体的自定义数据时，会从 Lua 添加此组件。在 radiant/modules/entities.lua 中，您可以找到从其他 Lua 文件甚至 UI 的 JavaScript 文件设置自定义名称、描述等的函数。

这样我们确保仅在确实有可以更改的自定义数据时才添加此组件，这与 entity_data 内的 display_name 或 description 相反，后者被该类型的所有物品重用。
